# 东南亚 CRM AI 销售闭环设计方案

> 面向东南亚 B2B 市场的 AI 驱动 CRM 销售全链路设计，覆盖从获客到交接实施的八个阶段。

---

## 一、市场背景与设计前提

### 1.1 东南亚 B2B 销售的现实特征

| 特征 | 说明 |
|------|------|
| 沟通渠道 | WhatsApp/Line/Zalo 是主战场，不是邮件和电话 |
| 销售模式 | 关系型销售为主，决策链长，周期 2-5 个月 |
| 数据现状 | CRM 数据完整率普遍 < 30%，销售抗拒录入 |
| 语言环境 | 多语言混用（印尼语、泰语、越南语、英语） |
| 人员流动 | 销售团队年流失率 30-40%，培训成本高 |
| 付款习惯 | 回款周期长，应收账款管理是痛点 |

### 1.2 核心设计原则

1. **IM 优先**：销售在 WhatsApp/Line 中完成所有操作，CRM 在后台自动运转
2. **零录入**：AI 从对话中自动提取数据，销售只需确认，不需手动录入
3. **阶段闭环**：每个阶段有明确的输入、AI 处理、输出和验证指标
4. **数据飞轮**：所有阶段的数据回流，持续优化 AI 策略

### 1.3 典型客户画像

- 印尼 SaaS 公司，销售团队 20 人
- 客单价 $10K-$80K
- 销售周期 2-5 个月
- 同时覆盖印尼、泰国、越南市场

---

## 二、销售全链路总览

```
获客 → 首次触达 → 需求挖掘 → 方案呈现 → 报价谈判 → 决策推进 → 签约回款 → 交接实施
 │        │          │          │          │          │          │          │
 ▼        ▼          ▼          ▼          ▼          ▼          ▼          ▼
AI线索    AI破冰     AI对话     AI方案     AI定价     AI决策     AI合同     AI交接
富化+评分  话术生成   实时辅助   自动生成   策略引擎   地图分析   自动生成   文档生成
```

每个阶段的 AI 输出是下一个阶段的输入，形成完整数据链路。

## 三、阶段一：获客 — AI 线索发现与富化

### 3.1 业务痛点

销售自己找客户，靠参展、转介绍、LinkedIn 手动搜。市场部给的线索信息只有名字和邮箱，销售不知道该不该花时间跟进。

### 3.2 AI 闭环链路

```
触发：市场活动 / 官网注册 / WhatsApp 广告带来新线索
    ↓
┌─────────────────────────────────────────┐
│ AI 线索富化引擎                           │
│                                          │
│ 输入：姓名 + 公司名 + 手机号/邮箱          │
│                                          │
│ 自动补全：                                │
│ ├→ 公司信息：行业、规模、注册资本           │
│ │   （对接印尼 AHU、泰国 DBD 等工商数据源） │
│ ├→ 联系人职位：从 LinkedIn 匹配            │
│ ├→ 公司技术栈：是否已用竞品                │
│ │   （从招聘信息、官网技术痕迹推断）         │
│ ├→ 近期动态：是否在扩张/融资/换系统         │
│ └→ 匹配度评分：和 ICP 的吻合度             │
└─────────────────────────────────────────┘
    ↓
AI 线索评分（0-100）：
    90+  → 热线索，立即分配给 Top Sales
    60-89 → 温线索，进入 SDR 跟进队列
    <60  → 冷线索，进入自动培育流程
```

### 3.3 智能分配规则

| 维度 | 规则 |
|------|------|
| 行业匹配 | 卖过制造业的销售接制造业线索 |
| 语言匹配 | 会泰语的销售接泰国线索 |
| 负载均衡 | 当前商机数少的优先分配 |
| 地理位置 | 雅加达的销售接雅加达客户 |

### 3.4 销售接收体验

销售收到 WhatsApp 通知：

> 新线索：PT Sentosa Jaya，制造业，200人，评分 87
> 关键信息：正在招 IT Manager（可能要换系统），去年营收增长 30%
> 建议话术：[AI 生成的首次触达模板]
> 一键拨打 / 一键发 WhatsApp

### 3.5 闭环指标

| 指标 | 目标 |
|------|------|
| 线索富化完整率 | > 80% 字段自动填充 |
| 线索到首次触达时间 | < 2 小时 |
| 线索评分准确率 | 高分线索转化率 > 低分线索 3 倍 |

## 四、阶段二：首次触达 — AI 辅助破冰

### 4.1 业务痛点

销售打电话或发 WhatsApp，客户不回。新销售不知道怎么开口，发的消息千篇一律，客户一看就是群发。

### 4.2 AI 闭环链路

```
销售准备联系客户
    ↓
AI 生成个性化触达方案：
┌──────────────────────────────────────────────┐
│ 客户：PT Sentosa Jaya                         │
│ 推荐触达渠道：WhatsApp（该行业 80% 偏好 WA）    │
│ 推荐时间：周二上午 10:00（该行业最佳响应时段）   │
│                                               │
│ 个性化话术（印尼语）：                          │
│ "Pak Budi，看到贵司最近在扩产线，我们帮过         │
│  同行业的 PT Maju 把生产排程效率提升了 35%，      │
│  方便这周聊 15 分钟吗？"                        │
│                                               │
│ 话术逻辑：                                     │
│ ├→ 提到客户近期动态（扩产线）→ 表明做过功课      │
│ ├→ 提到同行业案例 → 建立可信度                  │
│ ├→ 具体数字（35%）→ 量化价值                   │
│ └→ 低承诺请求（15分钟）→ 降低决策门槛           │
└──────────────────────────────────────────────┘
```

### 4.3 未响应自动跟进策略

```
销售发送消息
    ↓
AI 跟踪响应：
    ├→ 客户回复了 → 提取意图，建议下一步
    ├→ 已读未回 → 48h 后建议换个角度跟进
    └→ 未读 → 3天后建议换渠道（电话/邮件/LinkedIn）
    ↓
连续 3 次未响应 → AI 建议：
    1. 尝试联系公司其他人（AI 找到 IT Director 联系方式）
    2. 通过共同联系人引荐
    3. 降级到自动培育，3 个月后再试
```

### 4.4 闭环指标

| 指标 | 目标 |
|------|------|
| 首次触达回复率 | > 25%（行业平均 10-15%） |
| 从触达到有效对话的转化率 | > 15% |
| 平均触达次数到建联 | < 3 次 |

## 五、阶段三：需求挖掘 — AI 实时对话辅助

### 5.1 业务痛点

销售和客户聊了一小时，只聊了表面需求，没挖到预算、决策流程、时间线、竞品情况。新销售尤其不会提问。

### 5.2 AI 闭环链路

```
销售与客户进行视频会议/电话/WhatsApp 语音通话
    ↓
AI 实时转录 + 分析（不打断对话）
    ↓
实时侧边栏提示（只有销售能看到）：
┌──────────────────────────────────────────┐
│ 🎯 已获取信息          ❓ 还需挖掘         │
│ ✅ 业务痛点：手工排程   ❌ 预算范围          │
│ ✅ 现有方案：用 Excel   ❌ 决策人是谁        │
│ ✅ 团队规模：50 人      ❌ 决策流程和周期     │
│                        ❌ 是否在看竞品       │
│                        ❌ 上线时间期望       │
│                                           │
│ 💡 建议下一个问题：                         │
│ "Pak Budi，这个项目最终是谁来拍板决定？      │
│  通常你们采购这类系统需要走什么流程？"        │
│                                           │
│ ⚠️ 风险信号：                              │
│ 客户提到"先看看"出现 3 次 → 购买意愿可能不强 │
│ 建议：用案例制造紧迫感                      │
└──────────────────────────────────────────┘
```

### 5.3 通话后自动输出

| 输出项 | 说明 |
|--------|------|
| 会议摘要 | 按 BANT/MEDDIC 框架结构化 |
| 客户需求清单 | 已确认 vs 待确认 |
| 行动项 | 双方承诺的事项及截止时间 |
| 商机阶段建议 | 是否可以推进到下一阶段 |
| 风险标记 | 如"决策人未识别" |

所有内容自动写入 CRM，销售确认即可。

### 5.4 闭环指标

| 指标 | 目标 |
|------|------|
| BANT 四要素覆盖率 | > 80% |
| 会议摘要自动生成准确率 | > 90% |
| 商机推进速度 | 缩短 20% |

## 六、阶段四：方案呈现 — AI 生成定制化方案

### 6.1 业务痛点

售前/销售做方案靠复制粘贴历史 PPT，改改客户名字就发出去。客户看到的方案千篇一律，感受不到针对性。做一个定制方案要 2-3 天，售前忙不过来。

### 6.2 AI 闭环链路

```
输入：前几轮沟通中 AI 沉淀的客户信息
    ↓
AI 自动生成定制方案框架：
┌──────────────────────────────────────────────┐
│ PT Sentosa Jaya 解决方案                       │
│                                               │
│ 1. 客户现状与痛点（从对话记录中提取）            │
│    - 50 人团队用 Excel 管理生产排程              │
│    - 每月因排程错误导致 3-5 次延期交付            │
│    - 老板希望 3 个月内上线                       │
│                                               │
│ 2. 推荐方案（基于客户规模和行业匹配）             │
│    - 生产管理模块 + 排程优化模块                  │
│    - 50 用户授权                                │
│    - 建议分两期实施                              │
│                                               │
│ 3. 同行业案例（AI 从案例库中匹配最相似的）        │
│    - PT Maju：同行业同规模，排程效率 +35%        │
│    - CV Abadi：类似痛点，ROI 6 个月回本          │
│                                               │
│ 4. 投资回报分析（基于客户数据自动计算）            │
│    - 当前损失：延期交付罚款约 $2K/月              │
│    - 预期节省：$2K/月 × 12 = $24K/年            │
│    - 系统投资：$35K → 18 个月回本                │
│                                               │
│ 5. 实施计划 + 报价概要                          │
└──────────────────────────────────────────────┘
```

### 6.3 方案发送后的行为追踪

```
方案发送给客户
    ↓
AI 跟踪客户查看行为：
    "客户打开了方案，在 ROI 分析页停留了 4 分钟，
     案例页停留了 3 分钟，价格页只看了 30 秒"
    ↓
AI 生成跟进策略：
    "客户对 ROI 很关注，下次沟通重点强化投资回报，
     可以安排参考客户做一次通话"
```

### 6.4 闭环指标

| 指标 | 目标 |
|------|------|
| 方案生成时间 | 从 2-3 天 → 30 分钟 |
| 方案到报价的转化率 | > 60% |
| 客户方案查看完成率 | > 70% |

## 七、阶段五：报价谈判 — AI 定价策略与谈判辅助

### 7.1 业务痛点

销售为了成单随意打折，不同销售给同行业客户报价差异大。管理层事后才发现利润被压太低。东南亚客户特别喜欢砍价，销售容易在压力下让步过多。

### 7.2 AI 定价引擎

```
销售发起报价请求
    ↓
AI 定价分析：
┌──────────────────────────────────────────────┐
│ 标准价：$48,000                               │
│                                               │
│ 参考因素：                                     │
│ ├→ 同行业成交价中位数：$42,000（-12.5%）       │
│ ├→ 该客户规模对应价格带：$38K-$45K             │
│ ├→ 当季度 pipeline 缺口：$120K（需要这单）      │
│ ├→ 竞对同类方案估价：$30K-$35K                 │
│ ├→ 客户预算信号："大概 $40K 左右"               │
│ └→ 客户战略价值：行业标杆，案例价值高            │
│                                               │
│ AI 建议：                                      │
│ 首次报价：$45,000（折扣 6.25%，在销售权限内）   │
│ 底线价格：$40,000（折扣 16.7%，需总监审批）     │
└──────────────────────────────────────────────┘
```

### 7.3 谈判场景预设

| 客户动作 | AI 建议应对 |
|----------|------------|
| 要求降到 $40K | 可以接受，但要求年付不月付 |
| 要求降到 $35K | 砍掉高级报表模块，保持利润率 |
| 拿竞对比价 | 强调实施成功率和本地服务差异 |
| 要求免费试用 | 提供 1 个月 POC，限定范围和成功标准 |

### 7.4 实时谈判辅助

```
谈判过程中，客户提出新条件：
"能不能 $38K，我们一次付清"
    ↓
销售输入客户条件 → AI 实时分析：
    "$38K 一次付清 vs $40K 分期：
     考虑资金时间价值，$38K 一次付清实际等价于 $39.5K
     在可接受范围内，但建议争取 $39K 一次付清
     需要总监审批（超出销售折扣权限 2%）"
    ↓
一键发起审批 → 总监手机端看到完整定价分析 → 审批
    ↓
生成最终报价单（支持印尼语/英语双语）→ 发送客户
```

### 7.5 闭环指标

| 指标 | 目标 |
|------|------|
| 平均折扣率 | 控制在 15% 以内 |
| 报价审批周期 | < 4 小时 |
| 利润率达标比例 | > 85% |
| 报价到签约转化率 | > 40% |

## 八、阶段六：决策推进 — AI 识别卡点并推动成交

### 8.1 业务痛点

方案和报价都给了，客户说"我们内部讨论一下"，然后就没消息了。销售不知道客户内部在发生什么，只能干等或者盲目催。

### 8.2 AI 决策地图

```
商机进入"决策阶段"后，AI 自动构建决策地图：
┌──────────────────────────────────────────────┐
│ PT Sentosa Jaya 决策地图                       │
│                                               │
│ 👤 Pak Budi（IT Manager）- 推荐者              │
│    态度：积极 ✅  影响力：中                    │
│    关注点：技术可行性、实施风险                   │
│                                               │
│ 👤 Ibu Sari（Finance Director）- 审批者         │
│    态度：未知 ❓  影响力：高                     │
│    关注点：ROI、预算合理性（推测）               │
│    ⚠️ 尚未直接接触                             │
│                                               │
│ 👤 Pak Hendra（CEO）- 最终决策者                │
│    态度：未知 ❓  影响力：最高                   │
│    关注点：战略价值、竞争优势（推测）             │
│    ⚠️ 尚未直接接触                             │
│                                               │
│ 🔴 卡点分析：                                  │
│ Finance Director 未参与，可能在预算审批环节卡住   │
└──────────────────────────────────────────────┘
```

### 8.3 每周推进建议

```
AI 每周自动检测决策进展并生成建议：
    ├→ 新的联系人加入沟通 → 更新决策地图
    ├→ 客户查看方案/合同的行为变化
    └→ 竞对动态信号
    ↓
推送给销售：
    "本周建议：
     1. 周二给 Pak Budi 发一个 ROI 计算器让他内部传阅
     2. 周四约 Ibu Sari 的 30 分钟 call
     3. 准备一个'快速启动'方案（先上核心模块，降低首期投入）"
```

### 8.4 闭环指标

| 指标 | 目标 |
|------|------|
| 决策阶段平均停留时间 | 缩短 30% |
| 关键决策人覆盖率 | > 80% 商机识别到 2+ 决策人 |
| 决策阶段赢单率 | > 50% |

## 九、阶段七：签约回款 — AI 合同与收款自动化

### 9.1 业务痛点

客户口头同意了，但合同来回改了两周。签完合同催款又是一个月。东南亚客户付款习惯差，应收账款周期长。

### 9.2 AI 合同生成与审核

```
客户口头确认成交
    ↓
AI 自动生成合同草案：
    ├→ 基于报价单 + 标准合同模板
    ├→ 自动适配客户所在国法律条款
    │   （印尼/泰国/越南合同法差异）
    ├→ 标记需要法务审核的非标条款
    └→ 双语版本（本地语言 + 英语）
    ↓
合同发送 → 客户修改 → AI 对比分析：
    "客户修改了 3 处：
     1. 付款条件从 30 天改为 60 天
        → ⚠️ 影响现金流，建议协商 45 天
     2. SLA 从 99.5% 改为 99.9%
        → ⚠️ 超出标准承诺，需技术确认
     3. 增加了提前终止条款
        → 🔴 需法务审核"
```

### 9.3 签约后自动触发

| 触发动作 | 说明 |
|----------|------|
| 财务 | 生成应收账款，设置回款提醒 |
| 实施 | 创建项目，分配实施顾问 |
| CSM | 创建客户成功计划 |
| 销售 | 更新业绩，计算佣金 |

### 9.4 AI 回款跟踪

```
到期前 7 天  → AI 自动发送友好提醒（WhatsApp）
到期未付     → 升级提醒频率
逾期 15 天   → 通知销售介入
逾期 30 天   → 升级到管理层
逾期 60 天   → 触发法务流程
```

### 9.5 闭环指标

| 指标 | 目标 |
|------|------|
| 合同签署周期 | < 7 个工作日 |
| 回款周期（DSO） | < 45 天 |
| 逾期率 | < 15% |

## 十、阶段八：交接实施 — AI 确保销售承诺落地

### 10.1 业务痛点

销售为了成单承诺了很多，交接给实施团队时信息丢失，实施团队不知道客户的真实期望，导致实施阶段客户不满意，影响续约。

### 10.2 AI 自动生成交接档案

```
签约完成，触发自动交接
    ↓
```
