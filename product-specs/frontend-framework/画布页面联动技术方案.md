# 画布页面联动技术方案

> 版本：1.0 | 日期：2026-04-20
> 目标：定义页面、弹框、组件之间的完整联动机制

---

## 一、核心问题

当前画布中的组件是孤立的——按钮不知道打开哪个弹框，弹框不知道数据写入哪里，表单字段不知道绑定什么数据。需要一套"页面上下文"把它们串起来。

## 二、设计方案

### 2.1 页面上下文（PageContext）

每个画布页面运行时自动创建一个 PageContext，包含：

```typescript
interface PageContext {
  /** 当前页面绑定的元模型（从 dataTable 组件推断） */
  metamodelApiKey: string;
  /** 当前页面的数据分组 */
  entityApiKey: string;
  /** 当前选中的行数据（表格行点击/编辑时设置） */
  selectedRecord: Record<string, unknown> | null;
  /** 表单数据（弹框中的字段组件读写） */
  formData: Record<string, string>;
  /** 触发动作 */
  dispatch: (action: ActionPayload) => void;
  /** 刷新计数器 */
  refreshKey: number;
}
```

### 2.2 联动链路

```
语言管理页面（pageSettingsLang）
│
├── dataTable 组件
│   props.metamodelApiKey = "globalPickOption"
│   props.entityApiKey = "language"
│   → 自动注入到 PageContext.metamodelApiKey / entityApiKey
│   → 行操作"编辑" → PageContext.selectedRecord = 当前行
│   → 行操作"删除" → dispatch({ action: 'delete', record })
│
├── buttonGroup 组件
│   按钮"新增语言" → action: "create"
│   → dispatch({ action: 'create' })
│   → ActionContext 读取 PageContext.metamodelApiKey
│   → 查找弹框页面: pageForm_{metamodelApiKey}
│   → 打开弹框
│
└── 弹框页面（pageForm_globalPickOption）
    │
    ├── fieldText (apiKey) → 绑定 formData.apiKey
    ├── fieldText (label) → 绑定 formData.label
    ├── fieldNumber (optionOrder) → 绑定 formData.optionOrder
    ├── fieldBoolean (defaultFlg) → 绑定 formData.defaultFlg
    ├── fieldBoolean (enableFlg) → 绑定 formData.enableFlg
    │
    └── buttonGroup
        ├── "取消" → action: "cancel" → 关闭弹框
        └── "保存" → action: "save"
            → 收集 formData
            → batchSaveMetadata(createMap/updateMap)
            → 关闭弹框
            → 通知父页面刷新 (refreshKey++)
```

### 2.3 按钮动作的完整参数

按钮的 action 不再只是一个字符串，而是一个结构化的动作定义：

```typescript
interface ButtonAction {
  /** 动作类型（对应 uiAction.apiKey） */
  type: string;
  /** 目标弹框页面（create/edit 时使用） */
  targetPageApiKey?: string;
  /** 如果不指定 targetPageApiKey，自动推断规则：pageForm_{metamodelApiKey} */
}
```

画布 JSON Schema 中的按钮定义：

```json
{
  "label": "新增语言",
  "type": "primary",
  "action": "create",
  "targetPageApiKey": "pageForm_globalPickOption"
}
```

如果不指定 targetPageApiKey，系统自动按约定查找 `pageForm_{metamodelApiKey}`。

### 2.4 弹框页面的数据绑定

弹框页面中的字段组件通过 `fieldKey` 属性绑定到 PageContext.formData：

- 新增模式：formData 初始为空，字段组件显示空值
- 编辑模式：formData 从 selectedRecord 预填，字段组件显示已有值
- 用户修改字段 → 更新 formData
- 点击"保存" → 收集 formData → 调用 API

### 2.5 页面间通信

```
父页面 PageContext
    ↓ dispatch('create')
弹框 PageContext（子）
    ↓ 用户填写表单
    ↓ dispatch('save')
    ↓ API 调用成功
    ↓ 关闭弹框
父页面 PageContext.refreshKey++
    ↓ dataTable 重新加载数据
```

## 三、实现要点

### 3.1 PageContext 自动推断

PageRuntime 渲染画布时，扫描所有组件：
- 找到 dataTable → 提取 metamodelApiKey / entityApiKey 作为页面级上下文
- 找到 buttonGroup → 注入 PageContext（按钮可以读取 metamodelApiKey）
- 找到 field* 组件 → 绑定 formData（弹框场景）

### 3.2 弹框页面约定

| 约定 | 规则 | 示例 |
|:---|:---|:---|
| 弹框页面 apiKey | `pageForm_{metamodelApiKey}` | pageForm_globalPickOption |
| 弹框 pageType | 2（弹框） | |
| 弹框画布 | 包含 field* 组件 + buttonGroup（保存/取消） | |
| 保存按钮 action | `save` | |
| 取消按钮 action | `cancel` | |

### 3.3 字段组件的表单绑定

字段组件在弹框中运行时，需要：
1. 从 PageContext.formData 读取当前值
2. 用户修改时更新 PageContext.formData
3. 保存时从 PageContext.formData 收集所有字段值

这通过 RenderContext 扩展实现：

```typescript
interface RenderContext {
  mode: 'edit' | 'preview';
  /** 弹框场景下的表单数据（字段组件读写） */
  formData?: Record<string, string>;
  onFieldChange?: (key: string, value: string) => void;
}
```
