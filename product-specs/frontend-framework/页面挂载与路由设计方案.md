# 页面挂载与路由设计方案

> 版本：1.0 | 日期：2026-04-20
> 目标：画布定义的页面如何被菜单、按钮、其他页面引用和打开

---

## 一、核心问题

画布中定义了页面（page + pageSection），但这些页面没有"入口"：
- 全屏页面（pageType=1）→ 需要挂载到菜单
- 弹框页面（pageType=2）→ 需要被按钮的 action 引用
- 抽屉页面（pageType=3）→ 同上
- 内嵌页面（pageType=4）→ 需要被其他页面的区域引用

## 二、页面的三种使用方式

```
┌─────────────────────────────────────────────────────────┐
│                    使用方式                               │
├──────────┬──────────────────┬───────────────────────────┤
│ 菜单挂载  │   按钮触发弹框    │    页面内嵌               │
│          │                  │                           │
│ menu     │  buttonGroup     │  iframe / pageEmbed       │
│ .pageApi │  .action=create  │  .targetPageApiKey        │
│ Key      │  .targetPage     │                           │
│          │  ApiKey          │                           │
│ ↓        │  ↓               │  ↓                        │
│ 全屏渲染  │  弹框/抽屉渲染    │  内嵌区域渲染              │
│ PageType │  PageType=2/3    │  PageType=4               │
│ =1       │                  │                           │
└──────────┴──────────────────┴───────────────────────────┘
```

### 2.1 菜单挂载（全屏页面）

menu 元数据的 `pageApiKey` 字段指向 page 的 apiKey：

```
menu(menuSettingsLang)
  .pageApiKey = "pageSettingsLang"
    → AdminConsolePage 路由到 pageSettingsLang
    → PAGE_COMPONENTS 有注册？用手写组件
    → 没有？用 PageRuntime 从画布渲染
```

**当前已实现**，但新建的页面需要手动去菜单管理中关联。

### 2.2 按钮触发弹框（弹框/抽屉页面）

buttonGroup 中按钮的 `targetPageApiKey` 指向弹框页面：

```
buttonGroup
  .buttons[0].action = "create"
  .buttons[0].targetPageApiKey = "pageForm_globalPickOption"
    → ActionContext.dispatch({ action:'create', targetPageApiKey })
    → CanvasModal 加载 pageForm_globalPickOption 的画布
    → 渲染弹框
```

**当前已实现**。

### 2.3 页面内嵌（内嵌页面）

一个页面的某个区域嵌入另一个页面：

```
pageSettingsLang 的画布中放一个 pageEmbed 组件
  .targetPageApiKey = "pageSettingsLangDetail"
    → PageRuntime 递归渲染子页面
```

**待实现**。

## 三、完整流程：从创建页面到用户可见

### 场景 A：新建一个全屏管理页面

```
Step 1: 页面管理 → 新增页面
  apiKey: pageCustomerList
  pageName: 客户管理
  pageType: 1（全屏）
  entityApiKey: customer

Step 2: 编辑画布
  拖入 title + filterBar + buttonGroup + dataTable
  或用"从元模型生成" → 选择 customer → CRUD 模式

Step 3: 菜单管理 → 新增菜单
  apiKey: menuCustomer
  label: 客户管理
  pageApiKey: pageCustomerList  ← 关联到页面
  menuGroup: 业务管理
  menuIcon: Users

Step 4: 用户刷新页面
  → 左侧菜单出现"客户管理"
  → 点击 → PageRuntime 渲染画布
```

### 场景 B：新建一个弹框页面

```
Step 1: 页面管理 → 新增页面
  apiKey: pageForm_customer
  pageName: 新增/编辑客户
  pageType: 2（弹框）
  entityApiKey: customer

Step 2: 编辑画布
  拖入 metaForm(metamodelApiKey=customer) + divider + buttonGroup(保存/取消)

Step 3: 回到客户管理页面的画布
  选中 buttonGroup → 编辑"新增客户"按钮
  → 点击后执行: 打开新增弹框
  → 目标弹框页面: 新增/编辑客户 (pageForm_customer)

Step 4: 用户点击"新增客户"
  → 弹出 pageForm_customer 的画布弹框
  → 填写表单 → 保存 → 列表刷新
```

## 四、页面管理中的改进

### 4.1 页面列表增加"使用情况"列

显示每个页面被哪些菜单/按钮引用：

| 页面 | 类型 | 使用情况 |
|:---|:---|:---|
| pageSettingsLang | 全屏 | 菜单: 语言管理 |
| pageForm_globalPickOption | 弹框 | 按钮: 语言管理.新增语言, 时区管理.新增时区 |
| pageCustomerList | 全屏 | 未挂载（无菜单引用） |

### 4.2 快捷操作

- 全屏页面：页面列表中直接"挂载到菜单"按钮
- 弹框页面：页面列表中显示"被 N 个按钮引用"

### 4.3 页面创建向导

新增页面时，根据 pageType 引导下一步：
- 全屏页面 → 创建后提示"是否立即挂载到菜单？"
- 弹框页面 → 创建后提示"此页面可在其他页面的按钮中选择"
