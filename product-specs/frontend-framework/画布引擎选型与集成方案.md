# 画布引擎选型与集成方案

> 版本：1.0 | 日期：2026-04-18
> 目标：选择一个开源画布引擎，集成到元数据驱动的页面设计器中

---

## 一、GitHub 高分开源项目对比

### 1.1 候选项目

| 项目 | Stars | 定位 | 技术栈 | 许可证 |
|:---|:---|:---|:---|:---|
| [Puck](https://github.com/puckeditor/puck) | 12.1k | React 可视化页面编辑器 | React, TypeScript | MIT |
| [craft.js](https://github.com/prevwong/craft.js) | 7.8k | React 拖拽页面编辑器框架 | React, TypeScript | MIT |
| [lowcode-engine](https://github.com/alibaba/lowcode-engine) | 14.8k | 阿里企业级低代码引擎 | React, TypeScript | MIT |
| [Formily](https://github.com/alibaba/formily) + [Designable](https://github.com/alibaba/designable) | 11.9k + 3.3k | 表单设计器 + 可视化设计引擎 | React, TypeScript | MIT |

### 1.2 深度对比

| 维度 | Puck | craft.js | lowcode-engine | Formily+Designable |
|:---|:---|:---|:---|:---|
| **核心能力** | 页面级拖拽编辑 | 组件级拖拽编辑 | 完整低代码平台 | 表单设计+渲染 |
| **组件注册** | Config 对象声明 | React 组件直接用 | 物料协议(JSON) | JSON Schema |
| **数据格式** | JSON（自定义） | 内部节点树 | 搭建协议(JSON) | JSON Schema |
| **属性面板** | 自动生成 | 需自建 | 内置 setter | 内置 |
| **拖拽排序** | ✅ 内置 | ✅ 内置 | ✅ 内置 | ✅ 内置 |
| **嵌套布局** | ✅ DropZone | ✅ Canvas | ✅ 容器组件 | ✅ 布局组件 |
| **预览模式** | ✅ Render 组件 | ✅ 分离渲染 | ✅ 预览模式 | ✅ 预览 |
| **包体积** | ~50KB | ~30KB | ~500KB+ | ~200KB+ |
| **学习成本** | 低 | 中 | 高 | 中高 |
| **定制性** | 高（组件自由） | 最高（框架级） | 中（协议约束） | 中（Schema 约束） |
| **与 Ant Design** | 无绑定 | 无绑定 | 可集成 | 深度集成 |
| **维护活跃度** | 非常活跃 | 活跃 | 活跃 | Designable 停更 |
| **适合场景** | CMS/营销页/管理后台 | 自定义编辑器 | 企业低代码平台 | 表单驱动应用 |

### 1.3 推荐：Puck

**理由：**

1. **最契合我们的场景** — 我们需要的是"用自己的组件搭建管理页面"，不是通用低代码平台。Puck 的设计理念就是"bring your own components"
2. **API 简洁** — 只需定义 Config（组件列表+属性）和 Data（页面结构），Puck 处理拖拽、排序、属性编辑、预览
3. **与元数据体系完美对接** — Config 可以从元数据动态生成，Data 可以存储在 pageSection 元数据中
4. **MIT 许可** — 无商业限制
5. **活跃维护** — 2024-2025 持续更新，社区活跃
6. **轻量** — ~50KB，不会显著增加打包体积
7. **React 原生** — 与我们的 React 19 + TypeScript 技术栈完全兼容

**不选其他的原因：**
- craft.js：更底层，需要自建属性面板和工具栏，工作量大
- lowcode-engine：太重，协议复杂，与我们的元模型体系有冲突
- Formily+Designable：只覆盖表单，不覆盖列表/详情/画布；Designable 已停更

## 二、Puck 核心概念

### 2.1 三个核心对象

```typescript
// 1. Config — 定义可用组件和属性
const config: Config = {
  components: {
    GlobalPickManager: {
      label: "选项集管理",
      fields: {
        globalPickApiKey: { type: "select", label: "选项集", options: [...] },
      },
      render: ({ globalPickApiKey }) => <SystemSettingsView globalPickApiKey={globalPickApiKey} />,
    },
    EntityGrid: {
      label: "数据表格",
      fields: {
        columns: { type: "array", label: "显示列", arrayFields: { field: { type: "text" } } },
        pageSize: { type: "number", label: "每页条数" },
      },
      render: ({ columns, pageSize }) => <EntityGridPreview columns={columns} pageSize={pageSize} />,
    },
  },
};

// 2. Data — 页面结构（存储在 pageSection 元数据中）
const data: Data = {
  root: { props: {} },
  content: [
    { type: "GlobalPickManager", props: { globalPickApiKey: "language" } },
  ],
};

// 3. Editor — 渲染编辑器
<Puck config={config} data={data} onPublish={handleSave} />
```

### 2.2 Puck 提供的能力

| 能力 | 说明 |
|:---|:---|
| 拖拽添加 | 从组件列表拖入画布 |
| 拖拽排序 | 画布内组件上下拖拽排序 |
| 属性面板 | 根据 fields 定义自动生成属性编辑器 |
| 嵌套布局 | DropZone 支持容器组件嵌套子组件 |
| 预览模式 | Render 组件只渲染不编辑 |
| 撤销/重做 | 内置 undo/redo |
| 响应式 | 支持多视口预览 |
| 自定义 UI | 可覆盖工具栏、侧边栏、属性面板 |

## 三、集成方案

### 3.1 架构

```
元数据层（p_meta_item / p_common_metadata）
    ↓ 动态生成
Puck Config（组件定义 + 属性字段）
    ↓
Puck Editor（拖拽 + 属性编辑 + 预览）
    ↓ 保存
Puck Data → 转换为 pageSection 元数据 → batchSaveMetadata
    ↓ 运行时
PageRuntime 从 pageSection 加载 → 渲染页面
```

### 3.2 Config 从元数据动态生成

```typescript
async function buildPuckConfig(pageApiKey: string): Promise<Config> {
  // 1. 加载页面的数据源信息
  const page = await getPageMeta(pageApiKey);

  // 2. 根据数据源类型决定可用组件
  const components: Config["components"] = {};

  // 通用组件（所有页面都可用）
  components.ActionBar = { ... };
  components.StatCard = { ... };

  // 全局选项集页面
  if (page.dataSourceType === 'globalPickOption') {
    const pickDefs = await listGlobalPickOptionDefs();
    components.GlobalPickManager = {
      label: "选项集管理",
      fields: {
        globalPickApiKey: {
          type: "select",
          label: "选项集",
          options: pickDefs.map(d => ({ label: d.label, value: d.apiKey })),
        },
      },
      render: ({ globalPickApiKey }) => <SystemSettingsView globalPickApiKey={globalPickApiKey} />,
    };
  }

  // 实体页面
  if (page.dataSourceType === 'entity') {
    const items = await listEntityItems(page.entityApiKey);
    const fieldOptions = items.map(i => ({ label: i.label, value: i.apiKey }));

    components.EntityGrid = {
      label: "数据表格",
      fields: {
        columns: {
          type: "array",
          label: "显示列",
          arrayFields: {
            field: { type: "select", label: "字段", options: fieldOptions },
          },
        },
        pageSize: { type: "number", label: "每页条数" },
      },
      render: (props) => <EntityGridPreview {...props} entityApiKey={page.entityApiKey} />,
    };

    components.EntityForm = {
      label: "实体表单",
      fields: {
        mode: { type: "select", label: "模式", options: [
          { label: "新建", value: "create" },
          { label: "编辑", value: "edit" },
        ]},
        columns: { type: "select", label: "列数", options: [
          { label: "1列", value: 1 },
          { label: "2列", value: 2 },
        ]},
        // 字段分组编辑器（自定义 field type）
        fieldSets: { type: "custom", label: "字段分组", render: FieldSetEditor },
      },
      render: (props) => <EntityFormPreview {...props} entityApiKey={page.entityApiKey} />,
    };

    components.FilterPanel = {
      label: "筛选面板",
      fields: {
        fields: {
          type: "array",
          label: "筛选字段",
          arrayFields: {
            field: { type: "select", label: "字段", options: fieldOptions },
          },
        },
      },
      render: (props) => <FilterPanelPreview {...props} />,
    };

    components.TreePanel = { ... };
  }

  return { components };
}
```

### 3.3 Data 与 pageSection 元数据互转

```typescript
// pageSection 元数据 → Puck Data
function sectionsToPuckData(sections: SectionMeta[]): Data {
  return {
    root: { props: {} },
    content: sections
      .sort((a, b) => a.sectionOrder - b.sectionOrder)
      .map(sec => ({
        type: sec.componentType,
        props: sec.componentProps,
      })),
  };
}

// Puck Data → pageSection 元数据
function puckDataToSections(data: Data, pageApiKey: string): SectionMeta[] {
  return data.content.map((item, idx) => ({
    apiKey: `${pageApiKey}_${item.type.toLowerCase()}_${idx}`,
    sectionKey: `${item.type.toLowerCase()}_${idx}`,
    sectionOrder: idx + 1,
    layoutMode: 1,
    componentType: item.type,
    componentProps: item.props,
  }));
}
```

### 3.4 字段级配置能力

Puck 的 fields 系统支持：

| field type | 用途 | 示例 |
|:---|:---|:---|
| text | 文本输入 | 标题、描述 |
| number | 数字输入 | 每页条数、列数 |
| select | 下拉选择 | 选项集、模式、字段 |
| radio | 单选 | 布局模式 |
| textarea | 多行文本 | JSON 配置 |
| array | 数组编辑器 | 显示列列表、字段分组 |
| object | 对象编辑器 | 嵌套配置 |
| custom | 自定义渲染 | 字段分组编辑器、可见性规则编辑器 |
| external | 外部数据源 | 从 API 加载选项 |

**关键能力 — 字段选择器：**

```typescript
// 自定义 field：从元数据加载字段列表，多选
const FieldSelector: CustomField = {
  type: "custom",
  render: ({ value, onChange, entityApiKey }) => {
    const [items, setItems] = useState([]);
    useEffect(() => { listEntityItems(entityApiKey).then(setItems); }, []);
    return (
      <CheckboxGroup
        options={items.map(i => ({ label: `${i.label} (${i.apiKey})`, value: i.apiKey }))}
        value={value}
        onChange={onChange}
      />
    );
  },
};
```

**关键能力 — 可见性规则编辑器：**

```typescript
// 自定义 field：配置字段的显示/隐藏规则
const VisibilityRuleEditor: CustomField = {
  type: "custom",
  render: ({ value, onChange }) => (
    <div>
      <select value={value?.field} onChange={...}>
        <option>当字段...</option>
        {fields.map(f => <option value={f.apiKey}>{f.label}</option>)}
      </select>
      <select value={value?.operator}>
        <option value="eq">等于</option>
        <option value="ne">不等于</option>
        <option value="empty">为空</option>
      </select>
      <input value={value?.value} onChange={...} />
    </div>
  ),
};
```

## 四、实现路径

### Phase 1：集成 Puck + 基础画布（2 周）

```bash
pnpm add @measured/puck
```

- 安装 Puck
- 实现 `buildPuckConfig` — 从元数据动态生成 Config
- 实现 `sectionsToPuckData` / `puckDataToSections` — 数据互转
- 替换自研 CanvasEditor 为 Puck Editor
- GlobalPickManager 组件接入

### Phase 2：实体页面组件（3 周）

- EntityGrid 组件 + 列选择器（从 item 元数据加载）
- EntityForm 组件 + 字段分组编辑器
- FilterPanel 组件 + 筛选字段选择器
- ActionBar 组件
- TreePanel 组件

### Phase 3：高级配置（2 周）

- 字段显示/隐藏规则编辑器
- 字段排序拖拽
- 布局容器（行/列/标签页）
- 条件渲染表达式

### Phase 4：运行时对接（1 周）

- PageRuntime 从 Puck Data 格式渲染
- 预览模式使用 Puck Render
- 保存/发布流程
