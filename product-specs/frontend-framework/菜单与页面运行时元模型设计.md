# 菜单与页面运行时元模型设计

> 版本：1.0 | 日期：2026-04-18
> 前置：[元数据驱动页面设计器方案](元数据驱动页面设计器方案.md)
> 目标：菜单、页面、弹框、按钮全部元数据化，消除前端硬编码

---

## 一、核心问题

当前前端的菜单（`navigation.ts`）、页面路由（`AdminConsolePage.tsx` 的 switch-case）、弹框触发（各页面的 `setModal('create')`）全部硬编码。新增一个管理页面需要改 3 个文件。

目标：**菜单 + 页面 + 弹框 + 按钮全部存储在元数据中**，前端运行时从元数据读取并渲染，新增页面零代码。

---

## 二、新增 4 个元模型

### 2.1 元模型总览

| 元模型 | api_key | 职责 | enable_common | enable_tenant |
|:---|:---|:---|:---|:---|
| 菜单 | `menu` | 定义导航菜单树 | 1 | 1 |
| 页面 | `page` | 定义画布配置（页面/弹框） | 1 | 1 |
| 页面区域 | `pageSection` | 定义画布中的区域和组件 | 1 | 1 |
| 页面动作 | `pageAction` | 定义按钮和触发行为 | 1 | 1 |

### 2.2 关系图

```
menu（菜单）
  └── pageApiKey → page（点击菜单打开哪个页面）

page（页面/弹框）
  ├── pageSection（区域列表）
  │     └── componentType + componentProps（放什么组件）
  └── pageAction（按钮列表）
        └── targetPageApiKey → page（按钮打开哪个页面/弹框）
```

| 字段 | 值 |
|:---|:---|

| label | 页面 |
| enable_common | 1 |
| enable_tenant | 1 |

### 4.2 字段定义

| api_key | db_column | label | 类型 | 说明 |
|:---|:---|:---|:---|:---|
| pageName | dbc_varchar1 | 页面名称 | TEXT | 如"用户管理"、"编辑用户" |
| pageType | dbc_smallint1 | 页面类型 | SELECT | 见下方枚举 |
| entityApiKey | dbc_varchar2 | 主实体 | TEXT | 页面绑定的实体，如 `user` |
| pageDescription | dbc_varchar3 | 描述 | TEXT | — |

| layoutColumns | dbc_smallint2 | 表单列数 | NUMBER | 1=单列 2=双列（默认2） |
| enableFlg | dbc_smallint3 | 是否启用 | BOOLEAN | — |

### 4.3 pageType 枚举

| code | 名称 | 说明 | 使用场景 |
|:---|:---|:---|:---|
| 1 | `PAGE` | 全屏页面 | 菜单点击后展示的主页面 |
| 2 | `DIALOG` | 模态弹框 | 新建/编辑表单 |
| 3 | `DRAWER` | 侧边抽屉 | 详情面板、配置面板 |
| 4 | `INLINE` | 内嵌区域 | 嵌入到其他页面的子区域 |

### 4.4 场景判定规则

| 场景 | pageType | 触发方式 | 示例 |
|:---|:---|:---|:---|
| 菜单点击进入 | PAGE(1) | menu.pageApiKey | 用户管理列表页 |
| 新建按钮 | DIALOG(2) | pageAction.targetPageApiKey | 新建用户弹框 |
| 编辑按钮 | DIALOG(2) | pageAction.targetPageApiKey + recordId | 编辑用户弹框 |
| 行点击查看详情 | DRAWER(3) | pageAction.targetPageApiKey + recordId | 用户详情抽屉 |
| 角色权限配置 | PAGE(1) | menu.pageApiKey | 独立全屏页 |
| 全局选项集管理 | PAGE(1) | menu.pageApiKey + entityApiKey 参数 | 语言/时区管理 |

## 五、pageSection 元模型 — 页面区域

### 5.1 字段定义

| api_key | db_column | label | 类型 | 说明 |
|:---|:---|:---|:---|:---|
| pageApiKey | entity_api_key | 所属页面 | TEXT | 关联 page.apiKey |

|:---|:---|:---|

## 六、pageAction 元模型 — 按钮与动作

### 6.1 字段定义

| api_key | db_column | label | 类型 | 说明 |
|:---|:---|:---|:---|:---|
| pageApiKey | entity_api_key | 所属页面 | TEXT | 关联 page.apiKey |
| actionName | dbc_varchar1 | 按钮文本 | TEXT | 如"新建"、"编辑"、"删除" |
| actionIcon | dbc_varchar2 | 图标 | TEXT | lucide 图标名 |
| actionType | dbc_smallint1 | 动作类型 | SELECT | 见下方枚举 |
| actionOrder | dbc_int1 | 排序 | NUMBER | — |
| targetPageApiKey | dbc_varchar3 | 目标页面 | TEXT | 打开哪个 page（弹框/抽屉） |
| actionScope | dbc_smallint2 | 作用范围 | SELECT | 1=全局 2=行级 3=批量 |
| confirmMessage | dbc_varchar4 | 确认提示 | TEXT | 如"确定删除？" |
| apiEndpoint | dbc_varchar5 | API 端点 | TEXT | 直接调用的后端接口 |
| apiMethod | dbc_varchar6 | HTTP 方法 | TEXT | GET/POST/PUT/DELETE |
| permissionApiKey | dbc_varchar7 | 权限标识 | TEXT | 控制按钮可见性 |
| visibleExpr | dbc_varchar8 | 可见性表达式 | TEXT | 动态控制 |
| enableFlg | dbc_smallint3 | 是否启用 | BOOLEAN | — |
| dangerFlg | dbc_smallint4 | 是否危险操作 | BOOLEAN | 1=红色按钮 |

### 6.2 actionType 枚举

| code | 名称 | 行为 |
|:---|:---|:---|
| 1 | `OPEN_PAGE` | 打开 targetPageApiKey 指定的页面/弹框/抽屉 |
| 2 | `API_CALL` | 直接调用 apiEndpoint（如删除、启用/禁用） |
| 3 | `NAVIGATE` | 跳转到 targetPageApiKey 指定的全屏页面 |
| 4 | `BATCH_API` | 批量操作（选中多条后调用 apiEndpoint） |
| 5 | `REFRESH` | 刷新当前页面数据 |
| 6 | `EXPORT` | 导出数据 |
| 7 | `IMPORT` | 导入数据 |

### 6.3 actionScope 枚举

| code | 名称 | 位置 | 说明 |
|:---|:---|:---|:---|
| 1 | `GLOBAL` | ActionBar（页面顶部） | 新建、批量删除、导入、导出 |
| 2 | `ROW` | 表格行操作列 | 编辑、删除、启用/禁用 |
| 3 | `BATCH` | ActionBar（选中后出现） | 批量删除、批量导出 |

## 八、前端运行时

### 8.1 渲染流程

```
1. 应用启动
   → 加载 menu 元数据（Common + Tenant 合并）
   → 渲染侧边栏菜单树

2. 用户点击菜单项 "用户管理"
   → menu.pageApiKey = "pageUserList"
   → 加载 page 元数据（pageType=PAGE）
   → 加载 pageSection 列表
   → 加载 pageAction 列表
   → PageCanvas 渲染各 Section

3. 用户点击 "新建用户" 按钮
   → pageAction.actionType = OPEN_PAGE
   → pageAction.targetPageApiKey = "dialogUserCreate"
   → 加载目标 page（pageType=DIALOG）
   → 渲染模态弹框 + 内部 EntityForm

4. 用户点击行 "编辑" 按钮
   → pageAction.actionType = OPEN_PAGE
   → pageAction.targetPageApiKey = "dialogUserEdit"
   → 传入 recordId
   → 渲染模态弹框 + EntityForm（编辑模式）

5. 用户点击行 "删除" 按钮
   → pageAction.actionType = API_CALL
   → pageAction.confirmMessage = "确定删除？"
   → 弹出确认框 → 调用 apiEndpoint → 刷新列表
```

### 8.2 核心组件

```typescript
// 页面运行时入口
function PageRuntime({ pageApiKey, recordId }: { pageApiKey: string; recordId?: string }) {
  const page = usePageMeta(pageApiKey)        // 加载 page 元数据
  const sections = usePageSections(pageApiKey) // 加载区域列表
  const actions = usePageActions(pageApiKey)   // 加载按钮列表

  if (page.pageType === 'DIALOG') {
    return <DialogRuntime page={page} sections={sections} actions={actions} recordId={recordId} />
  }
  if (page.pageType === 'DRAWER') {
    return <DrawerRuntime page={page} sections={sections} recordId={recordId} />
  }
  return <PageCanvas page={page} sections={sections} actions={actions} />
}

// 菜单运行时
function MenuRuntime() {
  const menus = useMenuMeta()  // 加载菜单元数据
  const [activePageApiKey, setActivePageApiKey] = useState<string>()

  return (
    <Layout>
      <Sidebar menus={menus} onSelect={m => setActivePageApiKey(m.pageApiKey)} />
      <Content>
        {activePageApiKey && <PageRuntime pageApiKey={activePageApiKey} />}
      </Content>
    </Layout>
  )
}

// 按钮执行器
function ActionExecutor({ action, record }: { action: PageAction; record?: Record<string, unknown> }) {
  const execute = useCallback(() => {
    switch (action.actionType) {
      case 1: // OPEN_PAGE
        openPage(action.targetPageApiKey, record?.id)
        break
      case 2: // API_CALL
        if (action.confirmMessage) {
          confirm(action.confirmMessage).then(() => callApi(action.apiEndpoint, action.apiMethod, record))
        } else {
          callApi(action.apiEndpoint, action.apiMethod, record)
        }
        break
      case 3: // NAVIGATE
        navigate(action.targetPageApiKey)
        break
    }
  }, [action, record])

  return <Button icon={action.actionIcon} danger={action.dangerFlg === 1} onClick={execute}>{action.actionName}</Button>
}
```

## 十、设计原则

| 原则 | 说明 |
|:---|:---|
| 菜单即元数据 | 菜单树存储在 p_common_metadata，租户可覆盖（隐藏/重排/新增） |
| 页面即配置 | 页面结构、区域、组件、按钮全部 JSON 配置，零代码 |
| 弹框即页面 | 弹框和页面是同一个 page 元模型，只是 pageType 不同 |
| 按钮即动作 | 按钮的行为（打开弹框/调用 API/跳转页面）全部声明式配置 |
| Common/Tenant 双层 | 系统出厂菜单和页面在 Common 级，租户可在 Tenant 级覆盖 |
| 自动推导 | 无页面配置时，从 entity + item 元数据自动生成默认页面 |
