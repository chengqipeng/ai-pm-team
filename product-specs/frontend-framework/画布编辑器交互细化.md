> 基于 Puck 集成后的实际体验反馈

## 一、当前问题

### 1.1 右侧属性面板

当前右侧显示 `Page > Page` + `title` 字段，这是 Puck 的 root 属性。
应该改为显示页面元数据属性：

| 属性 | 说明 |
|:---|:---|
| 页面标识 (apiKey) | 只读，如 pageSettingsLang |
| 页面名称 | 可编辑，如"语言管理" |
| 页面描述 | 可编辑 |
| 数据源类型 | 下拉：无/实体/全局选项集 |
| 数据源标识 | 如 language、user |

### 1.2 中间画布区域

组件预览太小，需要：
- 画布宽度自适应（不要 iframe 缩放）
- 组件预览更大更清晰
- 点击组件后右侧切换到组件属性面板

### 1.3 组件属性面板

点击画布中的组件后，右侧应显示该组件的所有可配置属性。
当前 GlobalPickManager 的属性：

| 属性分组 | 属性 | 控件 |
|:---|:---|:---|
| 数据源 | 选项集 | 下拉（从元数据加载） |
| 标题区 | 标题 | 文本输入 |
| | 描述 | 文本输入 |
| | 显示新增按钮 | 开关 |
| 表格配置 | 显示列 | 文本（逗号分隔） |
| | 显示序号列 | 开关 |
| | 显示默认标记列 | 开关 |
| | 显示状态列 | 开关 |
| 行操作 | 显示编辑按钮 | 开关 |
| | 显示删除按钮 | 开关 |
| | 显示启用/禁用按钮 | 开关 |

## 二、Puck root 配置方案

Puck 的 `root` 对应整个页面，可以配置页面级属性：

```typescript
const config: Config = {
  root: {
    fields: {
      pageApiKey: { type: 'text', label: '页面标识 (apiKey)' },
      pageName: { type: 'text', label: '页面名称' },
      pageDescription: { type: 'text', label: '描述' },
      dataSourceType: {
        type: 'select',
        label: '数据源类型',
        options: [
          { label: '无', value: 'none' },
          { label: '实体', value: 'entity' },
          { label: '全局选项集', value: 'globalPickOption' },
        ],
      },
      dataSourceKey: { type: 'text', label: '数据源标识' },
    },
    render: ({ children, puck }) => (
      <div style={{ padding: 16 }}>{children}</div>
    ),
  },
  components: { ... },
};
```

这样点击画布空白区域时，右侧显示页面属性；点击组件时，右侧显示组件属性。

## 三、画布缩放问题

Puck 默认使用 iframe 渲染画布，导致组件被缩放。可以通过 `iframe` 配置禁用：

```typescript
<Puck
  config={config}
  data={data}
  onPublish={handleSave}
  iframe={{ enabled: false }}  // 禁用 iframe，直接在页面中渲染
/>
```

或者设置 iframe 宽度为 100%：

```typescript
<Puck
  config={config}
  data={data}
  onPublish={handleSave}
  viewports={[{ width: '100%', label: '全宽', icon: <Monitor /> }]}
/>
```
