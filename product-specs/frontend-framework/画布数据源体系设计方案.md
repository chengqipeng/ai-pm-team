# 画布数据源体系设计方案

> 版本：1.0 | 日期：2026-04-22
> 定位：画布编辑器中组件数据加载的统一架构，覆盖元模型定义、元数据实例、业务数据三层

---

## 一、架构概览

画布组件通过声明式 `dataSource` 配置获取数据，运行时由 `useDataSource` hook 统一加载。数据源分为三个层级，全部由后端 `p_meta_model` 表的字段驱动，前端零硬编码。

```
┌─────────────────────────────────────────────────────────────┐
│                    画布数据源三级分类                          │
├─────────────────┬──────────────────┬────────────────────────┤
│   元模型（定义层）│   元数据（实例层） │     数据（业务数据层）   │
│   metamodelLayer │   metamodelLayer │   enableDataSource=1   │
│   = 1            │   = 0            │   的元模型的实例列表     │
├─────────────────┼──────────────────┼────────────────────────┤
│ 元模型列表       │ 角色(role)       │ 客户(account)          │
│ 字段定义         │ 部门(department) │ 商机(opportunity)      │
│ 关联关系         │ 业务对象(entity) │ 线索(lead)             │
│ 选项值           │ 字段(item)       │ 用户(user)             │
│                 │ 共享规则         │ ...                    │
│                 │ ...（50个）       │ （动态，从entity加载）   │
└─────────────────┴──────────────────┴────────────────────────┘
```

## 二、p_meta_model 控制字段

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `enable_ui_config` | SMALLINT | 1 | 是否在画布数据源选择器中可见。0=隐藏（UI组件类、系统内部类） |
| `metamodel_layer` | SMALLINT | 0 | 数据层级。0=元数据实例层，1=元模型定义层 |
| `enable_data_source` | SMALLINT | 0 | 该元模型的实例是否可作为数据层数据源。1=是（当前仅 entity） |

### 数据分布

| 分类 | 条件 | 数量 | 示例 |
|---|---|---|---|
| 元模型定义层 | `metamodel_layer=1 AND enable_ui_config=1` | 4 | metaModel, metaItem, metaLink, metaOption |
| 元数据实例层（可见） | `metamodel_layer=0 AND enable_ui_config=1` | 50 | role, department, entity, item, sharingRule... |
| 元数据实例层（隐藏） | `metamodel_layer=0 AND enable_ui_config=0` | 12 | uiComponent, uiIcon, metaOperateLog, excelTask... |
| 数据层来源 | `enable_data_source=1` | 1 | entity（其实例 account/opportunity 等作为数据源） |

## 三、DataSourceConfig 配置

```typescript
interface DataSourceConfig {
  type: DataSourceType;          // 数据源类型标识
  metamodelApiKey?: string;      // 元数据类型：元模型 apiKey（如 'role'）
  entityApiKey?: string;         // 数据类型：业务对象 apiKey（如 'account'）
  depends?: string[];            // 联动依赖的 PageContext state key
  stateKey?: string;             // 选中项写入 PageContext 的 key
  paramMapping?: Record<string, string>;  // state key → API 参数名映射
  lazy?: boolean;                // 懒加载
}
```

### DataSourceType 枚举

| type | 分类 | 数据来源 | 后端接口 |
|---|---|---|---|
| `metaModel` | 元模型 | `p_meta_model` | `GET /meta/metamodels` |
| `metaItem` | 元模型 | `p_meta_item` | `GET /meta/meta-items?metamodelApiKey=xxx` |
| `metaLink` | 元模型 | `p_meta_link` | `GET /meta/meta-links` |
| `metaOption` | 元模型 | `p_meta_option` | `GET /meta/meta-options?metamodelApiKey=xxx` |
| `metadata` | 元数据 | `p_common/tenant_metadata` 或快捷表 | `GET /meta/metadata?metamodelApiKey=xxx` |
| `entityData` | 数据 | `p_tenant_data_*` 分片表 | `GET /entity/data/{entityApiKey}` |
| `columnMapping` | （内部） | `p_meta_item` 衍生 | `GET /meta/column-mapping?metamodelApiKey=xxx` |
| `custom` | （预留） | 任意后端 API | `GET {apiPath}` |

## 四、组件间联动机制

### 4.1 数据流

```
左侧组件（ListPanel / Tree）
  ↓ 用户点击选中
  ↓ setState(stateKey, selectedApiKey)
  ↓
PageContext.state = { selectedRoleApiKey: 'systemAdmin' }
  ↓ state 变化
  ↓
右侧组件（DetailHeader / DataTable）
  ↓ useDataSource 检测 depends 值变化
  ↓ 重新加载数据
  ↓ 渲染更新
```

### 4.2 stateKey 推导优先级

```
1. dataSource.stateKey（显式配置，最高优先级）
2. props.stateKey（组件级配置）
3. 自动推导：
   - metaModel → selectedModelApiKey
   - metadata + metamodelApiKey='role' → selectedRoleApiKey
```

### 4.3 配置示例：角色与授权页面

```json
{
  "components": [
    { "type": "splitPanel", "props": { "leftWidth": 280 } },
    { "type": "listPanel", "props": {
      "layoutZone": "left",
      "dataSource": { "type": "metadata", "metamodelApiKey": "role", "stateKey": "selectedRoleApiKey" },
      "displayField": "label",
      "secondaryField": "apiKey"
    }},
    { "type": "detailHeader", "props": {
      "dataSource": { "type": "metadata", "metamodelApiKey": "role", "depends": ["selectedRoleApiKey"] },
      "titleField": "label"
    }},
    { "type": "dataTable", "props": {
      "dataSource": { "type": "entityData", "entityApiKey": "user", "depends": ["selectedRoleApiKey"] },
      "columns": ["name", "phone", "email"]
    }}
  ]
}
```

### 5.2 数据驱动逻辑

前端 `useCanvasMetadata` 启动时加载 `listMetaModels()`，按 `metamodel_layer` 和 `enable_ui_config` 分类：

```typescript
// 元模型分类：metamodel_layer=1 且 enable_ui_config=1
const metaModelTypes = mmList.filter(mm => mm.metamodelLayer === 1 && mm.enableUiConfig !== 0);

// 元数据分类：metamodel_layer=0 且 enable_ui_config=1
const metadataTypes = mmList.filter(mm => mm.metamodelLayer !== 1 && mm.enableUiConfig !== 0);

// 数据分类：enable_data_source=1 的元模型的实例列表
const dataSourceModels = mmList.filter(mm => mm.enableDataSource === 1);
for (const dsMm of dataSourceModels) {
  const instances = await listMetadata(dsMm.apiKey);  // 如 listMetadata('entity')
  // 每个实例（如 account、opportunity）生成一个数据源选项
}
```

## 六、useDataSource LOADERS 注册表

```typescript
const LOADERS = {
  metaModel:     () => listMetaModels(),
  metaItem:      (p) => listMetaItems(p.selectedModelApiKey),
  metaLink:      () => listMetaLinks(),
  metaOption:    (p) => listMetaOptions(p.selectedModelApiKey),
  metadata:      (p) => listMetadata(p.metamodelApiKey).then(filterByDepends),
  entityData:    (p) => httpClient.get(`/entity/data/${p.entityApiKey}`).then(unwrap),
  columnMapping: (p) => getColumnMapping(p.selectedModelApiKey).then(toArray),
  custom:        (p) => httpClient.get(p.apiPath, { params }).then(unwrap),
};
```
