# 画布编辑器详细设计（自研方案）

## 背景

### 兼容性验证结论（2026-04-18）

| 方案 | 问题 | 结论 |
|:---|:---|:---|
| amis-editor@6.13.0 | `amis-editor-core` 使用 `componentWillMount`/`componentWillReceiveProps`（React 19 已移除）；依赖 `mobx@4.15.7` + `mobx-react@6.3.1`（项目用 mobx 6）；CSS 文件 `dist/style.css` 不存在 | ❌ 不可用 |
| amis@6.13.0（渲染器） | 依赖 `mobx-react@6.3.1`（mobx 4 时代），React 19 下运行时崩溃 | ❌ 不可用 |
| @measured/puck@0.20.2 | DropZone 白屏、Slot API 不稳定、属性面板只显示 title、组件内部不可编辑 | ❌ 不可用 |
| amis-core@6.13.0（纯渲染） | 无废弃 API，peerDeps react>=16.8.6 | ✅ 兼容但无编辑器 |

**决策：基于 React 19 + Ant Design 6 + HTML5 Drag & Drop 自研轻量画布编辑器。**
已从 package.json 移除：amis、amis-core、amis-editor、amis-formula、amis-ui、@measured/puck。

## 核心设计原则

1. **JSON Schema 驱动**：页面配置以 JSON Schema 存储在 pageSection.componentProps 中
2. **原子组件 + 复合组件**：最小粒度是原子组件（标题、表格、按钮等），复合组件是原子组件的组合
3. **拖拽排序**：基于 HTML5 Drag & Drop API，不引入额外拖拽库
4. **属性面板**：选中组件后右侧显示属性编辑面板，属性定义由组件自身声明
5. **元数据绑定**：组件属性可绑定元模型字段，实现元数据驱动

## 组件体系

### 第一层：原子组件（Atom）

最小不可分割的 UI 单元，每个原子组件声明：
- `type`: 唯一标识（如 `title`, `dataTable`, `button`）
- `label`: 显示名称
- `icon`: 图标
- `category`: 分类（布局/数据/操作/表单）
- `defaultProps`: 默认属性
- `propSchema`: 属性定义（用于属性面板渲染）
- `render(props, context)`: 渲染函数

| type | 名称 | 分类 | 说明 |
|:---|:---|:---|:---|
| title | 标题 | 布局 | H1-H6 标题文本 |
| text | 文本 | 布局 | 段落/说明文本 |
| divider | 分割线 | 布局 | 水平分割线 |
| spacer | 间距 | 布局 | 垂直间距 |
| dataTable | 数据表格 | 数据 | 元数据驱动的表格，绑定 metamodelApiKey |
| dataForm | 数据表单 | 数据 | 元数据驱动的表单 |
| dataDetail | 数据详情 | 数据 | 只读详情展示 |
| button | 按钮 | 操作 | 单个按钮，可绑定动作 |
| buttonGroup | 按钮组 | 操作 | 多个按钮的组合 |
| filterBar | 筛选栏 | 数据 | 搜索/筛选条件 |
| statCard | 统计卡片 | 数据 | 数字统计展示 |
| tabs | 标签页 | 布局 | 多标签切换容器 |

### 第 1.5 层：字段组件（Field Atom）

与 ItemTypeEnum 一一对应，用于表单场景。每个字段组件绑定 `fieldKey`（meta_item api_key），
运行时自动从元数据读取选项、校验规则等。

| type | 名称 | ItemType code | 存储类型 | 说明 |
|:---|:---|:---:|:---|:---|
| fieldText | 文本输入 | 1 | VARCHAR | 单行文本 |
| fieldSelect | 单选下拉 | 2 | VARCHAR | 支持 pickOption/globalPickOption 选项源 |
| fieldMultiSelect | 多选下拉 | 3 | VARCHAR | 多选，值以逗号分隔 |
| fieldTextarea | 文本域 | 4 | TEXT | 多行文本 |
| fieldNumber | 整数输入 | 5/11 | BIGINT | 支持 min/max/step |
| fieldCurrency | 金额输入 | 6 | DECIMAL | 支持精度和前缀符号 |
| fieldDate | 日期选择 | 7 | BIGINT | 时间戳存储 |
| fieldAutoNumber | 自动编号 | 9 | VARCHAR | 只读，系统自动生成 |
| fieldRelation | 关联选择 | 10/34/41 | BIGINT | 弹窗选择关联记录 |
| fieldMultiTag | 多选标签 | 16 | VARCHAR | 自由输入标签 |
| fieldPhone | 电话输入 | 22/13 | VARCHAR | 电话格式校验 |
| fieldEmail | 邮箱输入 | 23 | VARCHAR | 邮箱格式校验 |
| fieldUrl | 网址输入 | 24 | VARCHAR | URL 格式校验 |
| fieldImage | 图片上传 | 29 | VARCHAR | 存储文件路径 |
| fieldBoolean | 布尔开关 | 31 | SMALLINT | 0/1 开关 |
| fieldPercent | 百分比 | 33 | DECIMAL | 带 % 后缀 |
| fieldDatetime | 日期时间 | 38/15 | BIGINT | 日期+时间选择 |
| fieldFile | 文件上传 | 39 | VARCHAR | 存储文件路径 |
| fieldRichtext | 富文本 | 40 | TEXT | 富文本编辑器 |

> 虚拟类型（8 LAYOUT_LINE / 26 JOIN / 27 FORMULA / 99 DIMENSION）不生成字段组件。

### 第二层：复合组件（Composite）

由多个原子组件预组合而成的业务模板：

| type | 名称 | 包含的原子组件 |
|:---|:---|:---|
| crudPage | CRUD 页面 | title + filterBar + buttonGroup + dataTable |
| formDialog | 表单弹框 | title + dataForm + buttonGroup |
| detailPage | 详情页面 | title + dataDetail + buttonGroup |
| settingsPage | 设置页面 | title + dataTable(选项集) + buttonGroup |

### 第三层：页面（Page）

由多个原子/复合组件在画布中排列组成，存储为 JSON Schema。

## JSON Schema 格式

```json
{
  "version": "1.0",
  "components": [
    {
      "id": "comp_1",
      "type": "title",
      "props": { "text": "语言管理", "level": 2 }
    },
    {
      "id": "comp_2",
      "type": "dataTable",
      "props": {
        "metamodelApiKey": "globalPickOption",
        "entityApiKey": "language",
        "columns": ["apiKey", "label", "defaultFlg", "enableFlg"],
        "showActions": true,
        "actions": ["edit", "delete"]
      }
    },
    {
      "id": "comp_3",
      "type": "buttonGroup",
      "props": {
        "buttons": [
          { "label": "新增", "action": "create", "type": "primary" },
          { "label": "批量删除", "action": "batchDelete", "type": "danger" }
        ]
      }
    }
  ]
}
```

## 属性面板 PropSchema 定义

每个组件通过 `propSchema` 声明可编辑的属性：

```typescript
interface PropField {
  key: string           // 属性路径，如 "text", "level", "columns[0]"
  label: string         // 显示名称
  type: 'string' | 'number' | 'boolean' | 'select' | 'multiSelect'
       | 'metamodelPicker' | 'fieldPicker' | 'json' | 'color'
  defaultValue?: unknown
  options?: { label: string; value: unknown }[]  // select/multiSelect 用
  required?: boolean
  group?: string        // 属性分组（基础/数据/样式）
}
```

## 元数据绑定

dataTable/dataForm 等数据组件通过 `metamodelApiKey` + `entityApiKey` 绑定元数据：
- 属性面板提供元模型选择器（metamodelPicker）
- 选择元模型后自动加载该元模型的 p_meta_item 字段列表
- 用户勾选要显示的字段（fieldPicker）
- 运行时通过 listMetadata API 加载数据并渲染

## 存储

画布 JSON Schema 存储在 pageSection 元数据的 componentProps 字段中：
- metamodelApiKey = `pageSection`
- entityApiKey = 所属 page 的 apiKey
- componentType = `CanvasSchema`
- componentProps = JSON.stringify(schema)

## 复合组件自动生成机制

### 核心流程

```
用户点击"从元模型生成"
    ↓
GenerateDialog 弹框
    ↓ 选择元模型 + 生成模式
listMetaItems(metamodelApiKey) → MetaItem[]
listMetaOptions(metamodelApiKey) → MetaOption[]
    ↓
compositeGenerator.generateComposite(opts)
    ↓ 按 itemType 映射到字段原子组件
CanvasNode[] 预览
    ↓ 确认
插入画布
```

### 生成模式

| 模式 | 生成的组件组合 | 适用场景 |
|:---|:---|:---|
| crud | 标题 + 筛选栏 + 操作按钮(新增/删除/导出) + 数据表格 + 分页 | 标准管理页面 |
| form | 标题 + 字段组件(按 itemType 自动映射) + 分割线 + 保存/取消 | 新建/编辑表单 |
| detail | 面包屑 + 标题 + 描述列表 + 分割线 + 编辑/删除/返回 | 详情查看页面 |
| list | 标题 + 数据表格(无筛选无按钮) | 简洁列表展示 |
| dashboard | 统计行(总数/新增/待处理) + 卡片 + 最近记录表格 | 仪表盘卡片 |

### 字段映射规则

`compositeGenerator.ts` 中的 `itemToFieldNode()` 函数：

1. 根据 `MetaItem.itemType` 查 `ITEM_TYPE_TO_FIELD` 映射表，得到字段组件 type
2. 从 `MetaItem` 提取 label/apiKey/requireFlg/description 等基础属性
3. 选择类字段（SELECT/MULTI_SELECT）：从 `MetaOption[]` 注入选项列表和默认值
4. 布尔字段：设置 trueLabel/falseLabel
5. 数字字段：设置 min/max
6. 文本字段：设置 maxLength/placeholder
7. 关联字段：标记 targetEntity（需从 MetaLink 推导）

### 字段过滤

- 排除虚拟字段（LAYOUT_LINE=8, JOIN=26, FORMULA=27, DIMENSION=99）
- 排除系统字段（id, deleteFlg, createdAt, createdBy, updatedAt, updatedBy, tenantId 等）
- 按 itemOrder 排序

```
components/GenerateDialog.tsx   # 生成弹框 UI（选择元模型 + 模式 + 预览 + 确认）
```
