## 一、现状

### 1.1 运行时

四个页面仍使用硬编码 React 组件（`UserManagementView` 等），在 `AdminConsolePage.PAGE_COMPONENTS` 中注册。功能完整。

### 1.2 画布 Schema

四个页面的画布 Schema 已定义（`seeds/orgAuthPageSchemas.ts`），已注册到种子数据（`seeds/index.ts`，SEED_VERSION=5）。可在页面设计器（pagePageConfig）中预览和编辑布局。

### 1.3 迁移前置条件

运行时迁移到 `PageRuntime` 画布驱动需要：
- ✅ 通用数据源体系（metadata/entityData/custom loader）
- ✅ 组件联动机制（stateKey + depends + PageContext）
- ✅ TreeAtom 运行时数据加载 + 选中联动
- ✅ ListPanelAtom stateKey 配置
- ⬜ entityData loader 分页/筛选参数透传
- ⬜ permissionMatrix 原子组件（角色权限配置页面）
- ⬜ userPicker 原子组件（角色成员添加弹框）

## 二、四页面 Schema 设计

### 2.1 用户管理（pageUserList）

```
splitPanel (左侧 240px)
├── 左侧: tree
│   dataSource: { type: 'metadata', metamodelApiKey: 'department', stateKey: 'selectedDeptApiKey' }
│   labelField: 'label', parentField: 'deptParentApiKey'
└── 右侧:
    ├── buttonGroup [新增用户]
    └── dataTable
        dataSource: { type: 'entityData', entityApiKey: 'user', depends: ['selectedDeptApiKey'],
                      paramMapping: { selectedDeptApiKey: 'departApiKey' } }
        columns: [name, phone, email, departApiKey, status]
```

数据源策略：
- 部门树 → `metadata` 类型（元数据库 `p_tenant_department`）
- 用户列表 → `entityData` 类型（业务数据 `p_user`，通过 `/entity/data/user` 加载）

### 2.2 部门树管理（pageDeptTree）

```
splitPanel (左侧 280px)
├── 左侧: tree
│   dataSource: { type: 'metadata', metamodelApiKey: 'department', stateKey: 'selectedDeptApiKey' }
└── 右侧:
    ├── detailHeader
    │   dataSource: { type: 'metadata', metamodelApiKey: 'department', depends: ['selectedDeptApiKey'] }
    ├── buttonGroup [新增子部门, 编辑, 删除]
    └── dataTable (部门成员)
        dataSource: { type: 'entityData', entityApiKey: 'user', depends: ['selectedDeptApiKey'],
                      paramMapping: { selectedDeptApiKey: 'departApiKey' } }
```

### 2.3 角色与授权（pageRoleList）

```
splitPanel (左侧 280px)
├── 左侧: listPanel
│   dataSource: { type: 'metadata', metamodelApiKey: 'role', stateKey: 'selectedRoleApiKey' }
│   displayField: 'label', secondaryField: 'apiKey'
└── 右侧:
    ├── detailHeader
    │   dataSource: { type: 'metadata', metamodelApiKey: 'role', depends: ['selectedRoleApiKey'] }
    ├── buttonGroup [添加用户]
    └── dataTable (角色成员)
        dataSource: { type: 'custom', apiPath: '/auth/roles/users', depends: ['selectedRoleApiKey'],
                      paramMapping: { selectedRoleApiKey: 'roleApiKey' } }
```

注：角色成员列表使用 `custom` 类型（`/auth/roles/users` 是自定义 API，不走 entityData），因为角色成员关联在 `p_user_role` 表中，不是 entity 数据。

### 2.4 角色权限配置（pageRoleAuth）

```
splitPanel (左侧 240px)
├── 左侧: listPanel
│   dataSource: { type: 'metadata', metamodelApiKey: 'role', stateKey: 'selectedRoleApiKey' }
└── 右侧:
    ├── detailHeader
    │   dataSource: { type: 'metadata', metamodelApiKey: 'role', depends: ['selectedRoleApiKey'] }
    └── dataTable (权限配置，待替换为 permissionMatrix)
        dataSource: { type: 'custom', apiPath: '/permission/data-permission', depends: ['selectedRoleApiKey'] }
```

## 三、数据源分类

| 数据 | 数据源类型 | 来源 |
|---|---|---|
| 部门列表 | `metadata` (metamodelApiKey='department') | 元数据库 `p_tenant_department` |
| 角色列表 | `metadata` (metamodelApiKey='role') | 元数据库 `p_tenant_role` |
| 用户列表 | `entityData` (entityApiKey='user') | 业务数据 `p_user` |
| 角色成员 | `custom` (apiPath='/auth/roles/users') | 自定义 API（`p_user_role` 关联查询） |
| 权限配置 | `custom` (apiPath='/permission/data-permission') | 自定义 API |
