# 元模型联动模式分析矩阵

> 版本：1.0 | 日期：2026-04-24
> 基于 p_meta_link（36 条）、entityLink（30+ 条）、globalPickDependency 依赖体系，
> 分析每对元模型之间适合哪种联动模式。

---

## 一、联动模式速查

| 模式 | 标识 | 数据来源 | 配置方式 | 后端接口 |
|:-----|:-----|:---------|:---------|:---------|
| A | 直接父子 | p_meta_link | depends 自动推导 | GET /meta/metadata |
| B | 同模型详情 | 同 metamodelApiKey | depends 自动推导 | GET /meta/metadata（前端过滤） |
| C | 跨模型 filterField | 无直接关系，手动配置 | filterField + filterFrom | GET /entity/data 或 custom |
| D | 同模型树形 | dbc 扩展列 | filters + $动态值 | GET /meta/metadata（前端过滤） |
| E | 跨中间表链式 | globalPickDependency 或 p_meta_link 多级 | chainQueryRef / chainQuery | POST /meta/chain-query |

## 九、汇总统计

### 9.1 按模式分布

| 模式 | 关联对数 | 占比 | 典型场景 |
|:-----|:---------|:-----|:---------|
| A（直接父子） | **36** | 60% | entity→item, item→pickOption, metaModel→metaItem |
| B（同模型详情） | **12** | 20% | role→role详情, entity→entity详情 |
| C（跨模型 filterField） | **5+** | 8% | role→user, department→user, account→contact |
| D（同模型树形） | **2** | 3% | department→子部门, role→子角色 |
| E（跨中间表链式） | **5+** | 8% | 省→市, 实体→所有选项值 |

### 9.2 按元模型分类

| 分类 | 元模型数 | 主要模式 | 说明 |
|:-----|:---------|:---------|:-----|
| type=1 实体元数据 | 26 | A 为主，E 跨级 | 树形父子结构，p_meta_link 完整覆盖 |
| type=2 组织权限 | 6 | B + C + D | 详情 + 跨模型关联用户 + 树形 |
| type=3 UI 页面 | 4 | A + B | page→section/action 父子 |
| type=4 UI 组件 | 4 | B 为主 | 无父子关系，只有详情 |
| type=5 系统运维 | 5 | B 为主 | 日志/任务，只有列表+详情 |
| type=6 认证安全 | 7 | B 为主 | 配置类，只有列表+详情 |
| type=7 数据基础设施 | 4 | B 为主 | 路由/统计，只有列表+详情 |
| type=8 业务扩展 | 6 | A + B | position→positionUser 父子 |
| type=9 全局配置 | 5 | A + E | 父子 + 省市区 chainQuery |
| 定义层 | 4 | A + B | metaModel→metaItem 父子 |

### 9.3 模式选择决策树

```
问：左右两侧的元模型之间是什么关系？

├── p_meta_link 中有直接的 parent→child 记录？
│   └── 是 → 模式 A（直接父子，零配置）
│
├── 左右是同一个 metamodelApiKey？
│   ├── 右侧是详情展示 → 模式 B（同模型详情）
│   └── 右侧是子级列表 → 模式 D（同模型树形，需要 dbc 层级字段）
│
├── entityLink 中有关联记录？
│   └── 是 → 模式 C（跨模型 filterField）
│
├── 需要经过中间表才能关联？
│   └── 是 → 模式 E（chainQuery 链式查询）
│
├── p_meta_link 中有两级链路（A→B→C）？
│   └── 是 → 模式 E（chainQuery，跨级查询）
│
└── 以上都不是 → 无法联动（或用 custom API 兜底）
```

## 十、不适合联动的元模型

以下元模型通常作为独立列表使用，不参与左右分栏联动：

| 元模型 | 分类 | 原因 |
|:-------|:-----|:-----|
| metaOperateLog | 系统运维 | 日志数据，按时间查询，不按父子关联 |
| entityOperateLog | 系统运维 | 同上 |
| excelTask | 系统运维 | 导入导出任务，独立生命周期 |
| fileMeta | 系统运维 | 文件元数据，独立管理 |
| metaLog | 系统运维 | 变更日志 |
| totpSecret | 认证安全 | 用户级配置，不参与联动 |
| passwordPolicy | 认证安全 | 全局配置，单条记录 |
| authProvider | 认证安全 | OAuth 提供商列表 |
| tenantDataRoute | 数据基础设施 | 分表路由配置 |
| tenantDataStats | 数据基础设施 | 数据统计 |
| dataShare / dataShareRoute | 数据基础设施 | 权限分片，内部使用 |
| assistantRelation | 业务扩展 | 助理关系，扁平列表 |
| teamMember | 业务扩展 | 团队成员，扁平列表 |
