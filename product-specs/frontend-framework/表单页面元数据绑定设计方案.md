# 表单页面元数据绑定设计方案

> 版本：1.0 | 日期：2026-04-20
> 目标：弹框/表单页面绑定元模型后，自动根据字段定义生成表单，字段类型、必填、帮助文本全部从元数据读取

---

## 一、当前问题

截图中的"新增/编辑选项"弹框页面，每个字段组件（标识、显示名称、排序序号、设为默认、启用状态）都是手动拖入画布并逐个配置的。问题：

1. 手动配置效率低——每个字段都要手动设置 fieldKey、fieldLabel、required、helpText
2. 与元模型定义脱节——p_meta_item 已经定义了字段类型、必填、帮助文本，但画布中没有读取
3. 元模型字段变更后画布不会自动同步——新增字段需要手动去画布加
4. 不同元模型的表单弹框需要重复创建——每个元模型一个弹框页面

## 二、设计方案

### 2.1 核心思路

弹框页面（pageType=2）绑定一个元模型后，表单字段不再手动配置，而是：

```
page.entityApiKey = "globalPickOption"  （绑定元模型）
    ↓
运行时自动加载 listMetaItems("globalPickOption")
    ↓
按 itemType 映射到字段组件（ITEM_TYPE_TO_FIELD）
    ↓
从 p_meta_item 读取：
  - label → 字段标签
  - apiKey → 字段绑定 key
  - itemType → 字段组件类型
  - requireFlg → 是否必填
  - description → 帮助文本
  - itemOrder → 排序
  - minLength/maxLength → 校验规则
    ↓
自动渲染表单
```

### 2.2 两种模式

| 模式 | 说明 | 适用场景 |
|:---|:---|:---|
| 自动模式 | 绑定元模型，字段全部从 p_meta_item 自动生成 | 标准 CRUD 弹框，快速配置 |
| 自定义模式 | 手动拖入字段组件，逐个配置 | 需要特殊布局的复杂表单 |

画布 Schema 中通过一个新的原子组件 `metaForm` 实现自动模式：

```json
{
  "version": "1.0",
  "components": [
    {
      "id": "form_auto",
      "type": "metaForm",
      "props": {
        "metamodelApiKey": "globalPickOption",
        "mode": "auto",
        "columns": 1,
        "excludeFields": ["namespace", "labelKey", "descriptionKey"],
        "fieldOverrides": {
          "apiKey": { "readOnly": "editMode", "helpText": "唯一标识，创建后不可修改" },
          "defaultFlg": { "helpText": "同一分组下只能有一个默认值" }
        }
      }
    },
    {
      "id": "form_actions",
      "type": "buttonGroup",
      "props": {
        "buttons": [
          {"label": "取消", "type": "default", "action": "cancel"},
          {"label": "保存", "type": "primary", "action": "save"}
        ],
        "align": "right"
      }
    }
  ]
}
```

### 2.3 metaForm 组件

新增一个原子组件 `metaForm`，核心属性：

| 属性 | 类型 | 说明 |
|:---|:---|:---|
| metamodelApiKey | string | 绑定的元模型（从 p_meta_item 加载字段） |
| mode | 'auto' / 'custom' | auto=自动生成 custom=手动配置 |
| columns | number | 表单列数（1/2/3） |
| excludeFields | string[] | 排除的字段（不在表单中显示） |
| includeFields | string[] | 仅包含的字段（优先级高于 exclude） |
| fieldOverrides | Record | 字段级覆盖配置（覆盖元数据中的默认值） |
| showApiKey | boolean | 是否显示 apiKey 字段（新增时显示，编辑时只读） |
| showLabel | boolean | 是否显示 label 字段 |

### 2.4 字段自动映射规则

metaForm 运行时从 p_meta_item 加载字段列表，按以下规则映射：

```
p_meta_item.itemType → ITEM_TYPE_TO_FIELD → 字段组件类型
p_meta_item.label → fieldLabel（字段标签）
p_meta_item.apiKey → fieldKey（绑定 key）
p_meta_item.requireFlg → required（是否必填）
p_meta_item.description → helpText（帮助文本）
p_meta_item.itemOrder → 排序
p_meta_item.minLength → min（数字最小值）
p_meta_item.maxLength → max（数字最大值）/ maxLength（文本最大长度）
```

### 2.5 字段级覆盖（fieldOverrides）

允许在画布中覆盖元数据的默认配置：

```json
{
  "fieldOverrides": {
    "apiKey": {
      "readOnly": "editMode",
      "helpText": "唯一标识，创建后不可修改",
      "placeholder": "camelCase 英文标识"
    },
    "defaultFlg": {
      "helpText": "同一分组下只能有一个默认值",
      "trueLabel": "是",
      "falseLabel": "否"
    },
    "enableFlg": {
      "defaultValue": "1",
      "trueLabel": "启用",
      "falseLabel": "停用"
    }
  }
}
```

`readOnly: "editMode"` 表示编辑模式下只读（apiKey 创建后不可修改）。

### 2.6 固定字段处理

每条元数据都有 `apiKey` 和 `label` 两个固定字段（不在 p_meta_item 中定义），metaForm 自动在表单顶部添加：

- apiKey：文本输入，必填，编辑模式只读
- label：文本输入，必填

通过 `showApiKey` 和 `showLabel` 控制是否显示。

## 三、运行时渲染流程

```
CanvasModal 打开弹框
    ↓ 加载画布 Schema
    ↓ 发现 metaForm 组件
    ↓
metaForm.render(props, ctx)
    ↓ listMetaItems(props.metamodelApiKey)
    ↓ 过滤系统字段 + excludeFields
    ↓ 按 itemOrder 排序
    ↓ 对每个字段：
    │   ├── itemType → ITEM_TYPE_TO_FIELD → 字段组件类型
    │   ├── 从 p_meta_item 读取 label/requireFlg/description
    │   ├── 从 fieldOverrides 读取覆盖配置
    │   ├── 从 ctx.formData 读取当前值
    │   └── 渲染字段组件
    ↓
用户填写表单
    ↓ ctx.onFieldChange 更新 formData
    ↓
点击"保存"
    ↓ ctx.dispatch({ action: 'save' })
    ↓ submitForm(formData)
```

## 四、画布编辑器中的配置

### 4.1 metaForm 的属性面板

选中 metaForm 组件后，右侧属性面板显示：

```
数据
  元模型 *        [globalPickOption ▼]  ← metamodelPicker
  模式            [自动生成 ▼]          ← auto/custom
  
字段配置
  显示 apiKey 字段  [✓]
  显示 label 字段   [✓]
  排除字段          [namespace, labelKey, descriptionKey]  ← multiSelect
  
  字段列表（从元模型自动加载，可调整）：
  ┌─────────────────────────────────────────┐
  │ ✓ apiKey      文本输入   必填  编辑只读  │
  │ ✓ label       文本输入   必填            │
  │ ✓ optionOrder 整数输入                   │
  │ ✓ defaultFlg  布尔开关         覆盖:帮助文本 │
  │ ✓ enableFlg   布尔开关         覆盖:默认值=1 │
  │ ✗ namespace   (已排除)                   │
  │ ✗ labelKey    (已排除)                   │
  └─────────────────────────────────────────┘
  
样式
  表单列数          [1 ▼]
```

### 4.2 字段覆盖编辑

点击字段列表中的某个字段，展开覆盖配置：

```
optionOrder 排序序号
  ├── 字段标签: [排序序号]     ← 默认从 p_meta_item.label 读取，可覆盖
  ├── 占位文本: [数字越小越靠前]
  ├── 帮助文本: []
  ├── 必填:     [ ]           ← 默认从 p_meta_item.requireFlg 读取
  ├── 只读:     [ ]
  └── 默认值:   []
```

## 五、与现有体系的关系

| 层级 | 现有 | 新增 | 说明 |
|:---|:---|:---|:---|
| 原子组件 | 19 个 field* 组件 | metaForm 组件 | metaForm 内部复用 field* 组件 |
| 画布 Schema | 手动拖入 field* | 放一个 metaForm | 一个组件替代多个 field* |
| 数据来源 | 画布 props 硬编码 | p_meta_item 动态读取 | 字段定义从元数据来 |
| 弹框页面 | 每个元模型一个画布 | 一个 metaForm 适配所有 | 绑定不同元模型即可 |

## 六、语言管理弹框的新画布 Schema

```json
{
  "version": "1.0",
  "components": [
    {
      "id": "form_auto",
      "type": "metaForm",
      "props": {
        "metamodelApiKey": "globalPickOption",
        "mode": "auto",
        "columns": 1,
        "showApiKey": true,
        "showLabel": true,
        "excludeFields": ["namespace", "labelKey", "descriptionKey"],
        "fieldOverrides": {
          "apiKey": { "readOnly": "editMode", "helpText": "唯一标识，创建后不可修改", "placeholder": "camelCase 英文标识，如 zhCn" },
          "label": { "placeholder": "如：简体中文" },
          "optionOrder": { "placeholder": "数字越小越靠前" },
          "defaultFlg": { "helpText": "同一分组下只能有一个默认值", "trueLabel": "是", "falseLabel": "否" },
          "enableFlg": { "defaultValue": "1", "trueLabel": "启用", "falseLabel": "停用" }
        }
      }
    },
    {
      "id": "form_divider",
      "type": "divider",
      "props": { "marginY": 12 }
    },
    {
      "id": "form_actions",
      "type": "buttonGroup",
      "props": {
        "buttons": [
          {"label": "取消", "type": "default", "action": "cancel"},
          {"label": "保存", "type": "primary", "action": "save"}
        ],
        "align": "right"
      }
    }
  ]
}
```

对比现有的 7 个组件（5 个 field* + divider + buttonGroup），新方案只需 3 个组件（metaForm + divider + buttonGroup），且字段定义全部从元数据自动读取。
