## 二、老系统同步过来的问题数据

`init_local_dev.py` 的 `sync_data()` 函数从远程老系统全量同步 `p_common_metadata`，
其中 `metamodel_api_key='entity'` 的记录包含了老系统所有实体，可能存在以下问题：

### 2.1 老系统特有实体（新系统不需要）

老系统中存在大量内部/废弃/特殊用途的实体，如：
- 老系统内部管理实体（配置类、日志类）
- 已废弃的实体（被新实体替代）
- 特定租户的自定义实体（不应出现在 Common 级）
- 老系统 ID 关联的实体（新系统用 apiKey 关联）

### 2.2 字段命名不规范

老系统的 entity 记录中：
- `api_key` 可能是数字 ID 而非 camelCase 英文
- `label` 可能是老系统的内部名称
- `entity_type` 编码可能与新系统不一致
- `dbc_*` 列的值可能是老格式（如 `is_*` 前缀的布尔字段）

### 2.3 子元数据关联问题

老系统同步的 entity 下的子元数据（item、checkRule、busiType 等）：
- `entity_api_key` 可能是数字 ID 而非 apiKey
- 字段类型编码（itemType）可能是老编码
- db_column 映射可能不正确

## 三、修复方案

### 方案 A：在 init_local_dev.py 的 fix_data() 中增加实体清洗步骤

```python
# 3.2 清洗 entity 数据：只保留标准实体，删除老系统特有实体
STANDARD_ENTITIES = {
    # CRM 核心
    'account', 'contact', 'opportunity', 'lead', 'activity', 'campaign', 'caseEntity',
    # 商业交易
    'product', 'goods', 'order', 'orderProduct', 'contract', 'quote',
    # 系统
    'user', 'note', 'task', 'event',
    # 扩展业务
    'invoice', 'payment', 'priceBook', 'priceBookEntry', 'target', 'forecast', 'territory', 'highSea',
    # 虚拟
    'approval', 'notification', 'attachment', 'comment',
}

# 删除非标准实体（Common 级）
DELETE FROM p_common_metadata
WHERE metamodel_api_key = 'entity'
  AND api_key NOT IN ({STANDARD_ENTITIES})
  AND namespace = 'system';

# 同时清理这些实体下的子元数据
DELETE FROM p_common_metadata
WHERE metamodel_api_key IN ('item', 'checkRule', 'busiType', 'pickOption', ...)
  AND entity_api_key NOT IN ({STANDARD_ENTITIES})
  AND namespace = 'system';
```

### 方案 B：不删除，通过 hiddenFlg 隐藏

```python
# 隐藏非标准实体（不删除，保留数据）
UPDATE p_common_metadata
SET dbc_smallint9 = 1  -- hiddenFlg = 1
WHERE metamodel_api_key = 'entity'
  AND api_key NOT IN ({STANDARD_ENTITIES});
```

### 推荐：方案 A + 白名单

在 `fix_data()` 中增加一个步骤，用白名单控制 Common 级 entity 数据：
1. 定义 `STANDARD_ENTITIES` 白名单（29 个）
2. 删除白名单外的 Common 级 entity 记录
3. 级联删除这些 entity 下的所有子元数据
4. 修正白名单内 entity 的字段值（label、entityType、enableFlg 等）

## 四、前端实体管理页面的适配

当前 `EntityManagementView.tsx` 和新的 `EntityPageRuntime.tsx` 都通过 `listAllEntities()` API 加载实体列表。
数据清洗后，前端无需改动，只需要后端数据正确即可。

前端可以增加的辅助过滤：
- 默认隐藏 `hiddenFlg=1` 的实体
- 默认隐藏 `entityType=3`（虚拟）的实体
- 按 `namespace` 分 Tab 展示（已实现）
