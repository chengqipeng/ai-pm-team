## 二、5 个 Tab 的完整 CRUD 能力

### Tab 1：基本信息

| 操作 | API | 说明 | 业务校验 |
|:-----|:----|:-----|:---------|
| 读取 | 左侧选中时已加载 | entity 对象直接传入 | — |
| 更新 | `POST /metadata/batch-save` updateMap.entity | 修改 label/description/启用开关 | 标准实体限制编辑范围 |
| 删除 | `POST /metadata/batch-save` deleteMap.entity | 仅 namespace='custom' 可删 | **P0：引用检查**（关联字段/自定义按钮/审批单） |
| 启用/禁用 | `POST /metadata/batch-save` updateMap.entity | enableFlg 切换 | **P0：禁用时引用检查** |
| 新建实体 | `POST /metadata/batch-save` createMap.entity + item + busiType | 弹框创建，apiKey 追加 `__c` | apiKey 唯一性、主属性同时创建、defaultBusiType 同时创建 |

启用开关（9 个）：

| 开关 | 字段 | 新建默认 | 禁用确认 | 特殊规则 |
|:-----|:-----|:---------|:---------|:---------|
| 启用操作日志 | historyLogFlg | 0 | 无 | — |
| 启用业务类型 | busiTypeFlg | 1 | "不具备业务类型功能" | — |

| 启用查重规则 | duplicateRuleFlg | 1 | "已有规则将被清空" | 编辑时提示更严重 |
| 启用业务逻辑代码 | scriptExecutorFlg | 1 | "不再执行业务逻辑代码" | 检测已安装脚本 |
| 启用团队成员 | groupMemberFlg | 0 | — | **不可逆**，后端校验 |
| 启用系统动态流 | dynamicFeedFlg | 0 | "不再自动发送动态流" | — |
| 启用共享 | sharingFlg | 0 | 通用确认 | — |
| 启用流程 | flowFlg | 0 | 通用确认 | — |

## 五、P0 业务校验（必须实现）

| # | 模块 | 校验 | 触发时机 | 实现位置 |
|:--|:-----|:-----|:---------|:---------|
| 1 | 实体 | 删除/禁用引用检查 | deleteMap.entity / updateMap.entity(enableFlg=0) | 后端 batchSaveMetadata 前置校验 |
| 2 | 实体 | 团队成员不可逆 | updateMap.entity(groupMemberFlg=0) | 后端 batchSaveMetadata 前置校验 |
| 3 | 字段 | 删除引用检查（6 种） | deleteMap.item | 后端 batchSaveMetadata 前置校验 |
| 4 | 字段 | 选项值管理 | SELECT/MULTI_SELECT 字段 | 前端 ItemFormDialog 内嵌 |
| 5 | 校验规则 | 表达式编译校验 | 保存前 | 前端调用 `/metadata/check-formula/validate` |
| 6 | 查重规则 | 字段类型过滤 | 条件子表选择字段时 | 前端过滤不支持查重的 itemType |

## 六、数据规范约束

| 约束 | 说明 |
|:-----|:-----|
| 关联方式 | 统一 apiKey 关联，禁止 ID 关联 |
| 布尔字段 | xxxFlg 后缀 + Integer(0/1) + SMALLINT |
| apiKey 命名 | camelCase，自定义追加 `__c` 后缀 |
| 多选存储 | PG 原生 VARCHAR(300)[] 数组，禁止逗号分隔 |
| ItemType 编码 | 与后端 Java ItemTypeEnum 一致（新老编码不做转换） |
| 主键 | 元数据用 apiKey 联合主键，业务数据用雪花 BIGINT |
| 软删除 | delete_flg=1，batchSaveMetadata 的 deleteMap 执行软删除 |
| 级联删除 | 后端根据 p_meta_link 自动级联 |
