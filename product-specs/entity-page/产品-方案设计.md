# 实体业务数据页面自动生成 - 方案设计

> 版本：v1.0 | 日期：2026-04-24
> 需求类型：平台能力扩展 | 特征标签：数据模型变更、前端页面

## 一句话方案

根据 entity 元数据（entity + item + entityLink）自动生成菜单和业务数据管理页面（列表、新建、编辑、详情），使用代码组件直接渲染，不走画布 UI 配置模式。每个 entity 实例（如 account、opportunity、contact）自动对应一个菜单项和一套 CRUD 页面。

### 依赖与风险

| 项目 | 类型 | 影响 | 缓解 |
|:---|:---|:---|:---|
| `/entity/data/{entityApiKey}` API 完整性 | 依赖 | 列表查询需要支持分页+过滤+排序 | 已有设计（业务数据大宽表设计.md §14），需确认实现状态 |

## 方案设计

### 扩展策略

**选择**：并行式扩展

**理由**：
1. 新能力（entity 自动生成页面）不改变现有画布 UI 配置的任何行为
2. 不使用新能力时（非 entity 类型的页面），体验完全不变
3. 两套渲染路径并行：pageType=1~4 走画布渲染，pageType=5(ENTITY) 走代码组件渲染

**对现有功能的影响**：零影响。画布 UI 配置体系完整保留，entity 页面走独立的 EntityPageRuntime 渲染路径。

### 核心流程

#### 流程一：entity→菜单自动同步

| 步骤 | 动作 | 响应 | 异常 |
|:-----|:-----|:-----|:-----|
| 1 | 管理员在实体管理中新增 entity（如 account） | 后端写入 entity 元数据 | — |
| 2 | 后端 entity 写入成功后触发同步 | 自动在 menu 元数据中插入对应菜单项 | 菜单已存在则跳过 |
| 3 | 自动在 page 元数据中插入对应页面定义 | pageType=5(ENTITY)，entityApiKey=account | 页面已存在则跳过 |
| 4 | 用户刷新页面 | 左侧菜单出现"客户"（account.label） | — |
| 5 | 管理员禁用 entity（enableFlg=0） | 对应菜单自动隐藏（visibleFlg=0） | — |
| 6 | 管理员删除 entity | 对应菜单和页面标记删除 | — |

同步写入的 menu 元数据：

```
apiKey: menuEntity_{entityApiKey}     如 menuEntity_account
label: {entity.label}                 如 "客户"
menuIcon: {entity.svgApiKey 或默认}    如 "Database"
menuGroup: "业务管理"
groupOrder: 10
menuOrder: {entity.customEntitySeq}
pageApiKey: pageEntity_{entityApiKey}  如 pageEntity_account
menuType: 1 (ITEM)
visibleFlg: 1
enableFlg: 1
namespace: system
```

同步写入的 page 元数据：

```
apiKey: pageEntity_{entityApiKey}      如 pageEntity_account
pageName: {entity.label}               如 "客户"
pageType: 5 (ENTITY)
entityApiKey: {entityApiKey}            如 "account"
enableFlg: 1
namespace: system
```

#### 流程二：entity 页面渲染

| 步骤 | 动作 | 响应 | 异常 |
|:-----|:-----|:-----|:-----|
| 1 | 用户点击菜单"客户" | menu.pageApiKey → pageEntity_account | — |
| 2 | PageRuntime 加载 page 元数据 | 检测到 pageType=5(ENTITY) | — |
| 3 | 走 EntityPageRuntime 渲染路径 | 并行加载 entity + item + entityLink 元数据 | 加载失败显示错误提示 |
| 4 | 渲染 EntityListView | 表格列从 item 元数据推导，操作栏自动生成 | — |
| 5 | 用户点击"新建" | 弹出 EntityFormDialog | — |
| 6 | EntityFormDialog 渲染表单 | 字段从 item 元数据推导，按 itemType 映射组件 | — |
| 7 | 用户填写并保存 | POST `/entity/data/{entityApiKey}` | 校验失败显示错误 |
| 8 | 用户点击行"查看" | 弹出 EntityDetailDrawer | — |
| 9 | 用户点击行"编辑" | 弹出 EntityFormDialog（编辑模式） | — |
| 10 | 用户点击行"删除" | 确认后 DELETE `/entity/data/{entityApiKey}/{id}` | — |

#### 流程三：关联对象 Tab

| 步骤 | 动作 | 响应 | 异常 |
|:-----|:-----|:-----|:-----|
| 1 | 用户在客户详情中查看关联 | EntityDetailDrawer 加载 entityLink 元数据 | — |
| 2 | 根据 entityLink 生成 Tab 页签 | 如"联系人"、"商机"、"线索" | 无关联则不显示 Tab |
| 3 | 用户切换 Tab | 加载关联对象数据 GET `/entity/data/{childEntityApiKey}?filter[accountApiKey]={id}` | — |
| 4 | 用户在 Tab 中新建关联记录 | 弹出 EntityFormDialog，自动填充关联字段 | — |

### 功能模块

| 模块 | 功能点 | 优先级 |
|:---|:---|:---|
| EntityPageRuntime | pageType=5 的渲染入口，加载 entity/item/entityLink 元数据 | P0 |
| EntityListView | 通用列表：表格（列从 item 推导）+ 筛选栏 + 操作栏（新建/删除/导出） | P0 |
| EntityFormDialog | 通用表单弹框：字段从 item 推导，按 itemType 映射组件，支持新建/编辑 | P0 |
| EntityDetailDrawer | 通用详情抽屉：字段从 item 推导，只读展示 | P0 |
| EntityRelationTabs | 关联 Tab：从 entityLink 推导，每个 Tab 是一个子 EntityListView | P1 |
| entity→menu 同步 | entity 变更时自动维护 menu + page 元数据 | P0 |
| 列表列配置 | 用户可调整列显示/隐藏/排序（存储在 localStorage 或 Tenant 元数据） | P1 |
| 高级筛选 | 支持多条件组合筛选（字段+操作符+值） | P1 |
| 批量操作 | 批量删除、批量导出 | P2 |

### 数据需求

#### entity→menu 同步需要的数据映射

| entity 字段 | menu 字段 | 映射规则 |
|:---|:---|:---|
| apiKey | apiKey | `menuEntity_{entity.apiKey}` |
| label | label | 直接映射 |
| svgApiKey | menuIcon | 有值用 svgApiKey，无值用默认 "Database" |
| customEntitySeq | menuOrder | 直接映射 |
| enableFlg | visibleFlg | 直接映射 |
| entityType | — | entityType=3(虚拟) 不生成菜单 |
| hiddenFlg | visibleFlg | hiddenFlg=1 时 visibleFlg=0 |

#### entity→page 同步需要的数据映射

| entity 字段 | page 字段 | 映射规则 |
|:---|:---|:---|
| apiKey | apiKey | `pageEntity_{entity.apiKey}` |
| label | pageName | 直接映射 |
| apiKey | entityApiKey | 直接映射 |
| — | pageType | 固定 5(ENTITY) |

#### item 元数据驱动表格列的推导规则

| item 字段 | 表格列属性 | 推导规则 |
|:---|:---|:---|
| apiKey | dataIndex | 直接映射 |
| label | title | 直接映射 |
| itemType | render | ITEM_TYPE_TO_COLUMN_RENDER 映射（TEXT→文本，SELECT→标签，BOOLEAN→开关...） |
| itemOrder | 列排序 | 按 itemOrder 升序 |
| requireFlg | — | 不影响列显示 |
| detailFlg | 是否在列表显示 | detailFlg=1 的字段默认不在列表显示（仅详情） |
| hiddenFlg | 是否隐藏 | hiddenFlg=1 不显示 |

过滤规则（哪些 item 不生成列）：
- itemType=8(LAYOUT_LINE) / 26(JOIN) / 27(FORMULA) / 99(DIMENSION)：虚拟类型，不显示
- 系统字段（id, deleteFlg, createdAt, createdBy, updatedAt, updatedBy, tenantId）：不显示
- 列表默认显示前 6~8 个非系统字段，其余可通过列配置开启

#### item 元数据驱动表单字段的推导规则

| item 字段 | 表单属性 | 推导规则 |
|:---|:---|:---|
| apiKey | name | 直接映射 |
| label | label | 直接映射 |
| itemType | 组件类型 | ITEM_TYPE_TO_FIELD 映射（TEXT→Input，SELECT→Select，DATE→DatePicker...） |
| requireFlg | required | requireFlg=1 → 必填 |
| description | tooltip | 有值时显示帮助提示 |
| maxLength | maxLength | 文本类型字段的最大长度 |
| defaultValue | initialValue | 新建时的默认值 |
| pickOption 子数据 | options | SELECT/MULTI_SELECT 类型的选项列表 |

## 架构设计

### 渲染路径分流

```
用户点击菜单
    ↓
PageRuntime 加载 page 元数据
    ↓
page.pageType = ?
    ├── 1(PAGE) / 2(DIALOG) / 3(DRAWER) / 4(EMBED)
    │   → 画布渲染路径（现有逻辑，不变）
    │   → 加载 pageSection → CanvasSchema → 逐个渲染原子组件
    │
    └── 5(ENTITY)
        → EntityPageRuntime（新增）
        → 加载 entity + item + entityLink 元数据
        → 代码组件直接渲染
```

### EntityPageRuntime 组件结构

```
EntityPageRuntime({ entityApiKey })
├── useEntityMeta(entityApiKey)        → entity 元数据
├── useItemsMeta(entityApiKey)         → item 字段列表
├── useEntityLinks(entityApiKey)       → entityLink 关联关系
│
├── EntityListView                     → 列表页面
│   ├── EntityFilterBar                → 筛选栏（从 item 推导可筛选字段）
│   ├── EntityActionBar                → 操作栏（新建/批量删除/导出）
│   └── EntityDataTable                → 数据表格（列从 item 推导）
│       └── 行操作：查看/编辑/删除
│
├── EntityFormDialog                   → 新建/编辑弹框
│   └── EntityMetaForm                 → 元数据驱动表单（字段从 item 推导）
│
└── EntityDetailDrawer                 → 详情抽屉
    ├── EntityDetailHeader             → 标题+标签
    ├── EntityDetailFields             → 字段详情（从 item 推导）
    └── EntityRelationTabs             → 关联 Tab（从 entityLink 推导）
        └── EntityListView(子)         → 每个 Tab 是一个子列表
```

### 与画布体系的边界

| 维度 | 画布 UI 配置（pageType=1~4） | entity 代码渲染（pageType=5） |
|:---|:---|:---|
| 页面结构 | 画布 JSON Schema 定义 | entity + item 元数据自动推导 |
| 组件选择 | 用户在画布中拖拽组件 | 固定组件结构（列表+表单+详情） |
| 字段配置 | 用户在画布属性面板配置 | 从 item 元数据自动推导 |
| 按钮配置 | pageAction 元数据定义 | 固定操作（新建/编辑/删除/查看） |
| 适用场景 | 管理后台页面（菜单管理、元模型管理等） | 业务数据页面（客户、商机、联系人等） |
| 定制能力 | 完全自由 | 列显示/隐藏、筛选条件（P1） |
| 数据来源 | 多种 DataSourceType | 固定 `/entity/data/{entityApiKey}` |

## 实施路径

### Phase 1：核心渲染（1.5 周）

| 任务 | 说明 |
|:-----|:-----|
| pageType=5(ENTITY) 枚举 | page 元模型增加 ENTITY 类型 |

| 任务 | 说明 |
|:-----|:-----|

| EntityRelationTabs | 从 entityLink 推导关联 Tab |

### Phase 3：菜单同步（0.5 周）

| 任务 | 说明 |

### Phase 4：增强（1 周）

| 任务 | 说明 |
|:-----|:-----|
| EntityFilterBar | 高级筛选（多条件组合） |

| 业务类型筛选 | entity.busiTypeFlg=1 时显示业务类型筛选 |

总计约 4 周。
