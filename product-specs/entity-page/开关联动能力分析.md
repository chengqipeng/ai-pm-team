# 实体开关联动能力分析

> 日期：2026-04-24
> 设计原则：开关是**运行时执行开关**，不是数据删除开关。
> 关闭开关 = 规则数据保留，但业务数据 CRUD 时跳过执行。

---

## 开关的正确语义

| 开关 | =1 时 | =0 时 | 数据影响 |
|:-----|:------|:------|:---------|
| checkRuleFlg | 业务数据创建/更新时执行校验规则 | 跳过校验 | 规则数据保留不删 |
| duplicateRuleFlg | 业务数据创建/更新时执行查重检测 | 跳过查重 | 规则数据保留不删 |
| busiTypeFlg | 业务数据创建时要求填写 busitypeApiKey | 不要求 | 业务类型数据保留 |
| historyLogFlg | 业务数据 CRUD 后写操作日志 | 不写日志 | 已有日志保留 |
| groupMemberFlg | 业务数据创建时自动创建团队成员记录 | 不创建 | 不可逆（1→0 拒绝） |

## 前端 UI 联动（已实现）

| 联动点 | 行为 |
|:-------|:-----|
| Tab 头 | 开关关闭时 Tab 名称旁显示琥珀色圆点（⚠️ 未启用） |
| Tab 内容 | 开关关闭时 Tab 仍可进入配置规则，顶部显示琥珀色提示条 |
| 提示条文案 | "当前「启用校验规则」未开启，以下规则不会在业务数据操作时执行。可在「基本信息」中开启。" |

## 后端运行时联动（P1 待办）

联动点在 `EntityDataService` 的业务数据 CRUD 方法中：

```java
// EntityDataService.create() 伪代码
public void create(String entityApiKey, Map<String, Object> data) {
    Entity entity = mergeReadService.getByApiKey("entity", entityApiKey);

    // 1. busiTypeFlg 联动
    if (MetaBooleanUtils.isTrue(entity.getBusiTypeFlg())) {
        if (data.get("busitypeApiKey") == null) {
            throw new BusinessException("请选择业务类型");
        }
    }

    // 2. duplicateRuleFlg 联动
    if (MetaBooleanUtils.isTrue(entity.getDuplicateRuleFlg())) {
        duplicateRuleEngine.check(entityApiKey, data);
    }

    // ... 执行写入 ...

    // 3. checkRuleFlg 联动
    if (MetaBooleanUtils.isTrue(entity.getCheckRuleFlg())) {
        checkRuleEngine.validate(entityApiKey, data);
    }

    // 4. historyLogFlg 联动
    if (MetaBooleanUtils.isTrue(entity.getHistoryLogFlg())) {
        operateLogService.log(entityApiKey, "CREATE", data);
    }

    // 5. groupMemberFlg 联动
    if (MetaBooleanUtils.isTrue(entity.getGroupMemberFlg())) {
        teamMemberService.autoCreate(entityApiKey, data);
    }
}
```
