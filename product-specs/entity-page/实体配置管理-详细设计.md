### 1.3 核心业务逻辑（XObjectAction.java）

#### 实体列表（entity-list）
- 检查 License（CRM-CM 模块）
- 加载自定义实体列表（`customizeEntities`）
- 加载 SVG 图标映射（`customSvgMap`）
- 加载元数据 Delta 权限（`metaDeltaPermits`）

#### 保存实体（save-entity）
- **新建**：
  1. apiKey 自动追加 `__c` 后缀（自定义标识）
  2. 设置启用开关（checkRule/busiType/duplicateRule/historyLog/scriptExecutor/groupMember/dynamicFeed）
  3. 调用 `objectManagerService.saveObject(entity, item)` 创建实体 + 主属性字段
  4. 同步国际化资源
  5. 处理多态字段关联
  6. 保存 APP 菜单和 Web 菜单
- **编辑**：
  1. 加载数据库中的实体（`xObjectService.get`）
  2. 更新 label、description、iconId、启用开关
  3. 调用 `xObjectService.updateObject` 更新
  4. 处理多态字段关联变更（增减）
  5. 更新菜单

#### 删除实体（delete-entity）
- 检查审批单引用（businessCategory=2）
- 检查字段关联引用（`checkEntityFieldRelation`）
- 调用 `xObjectService.deleteObject` 级联删除
- 删除 APP 菜单和 Canvas 菜单

#### 启用/禁用（set-entity-use-flg）
- 检查审批单 Limit
- 检查自定义按钮引用
- 检查字段关联引用（禁用时）
- 调用 `xObjectService.enableObject`

### 1.4 启用开关的交互规则

| 开关 | 新建默认 | 编辑时禁用确认 | 特殊规则 |
|:-----|:---------|:---------------|:---------|
| 启用操作日志 | ☐ 未勾选 | 无确认 | — |
| 启用业务类型 | ☑ 已勾选 | "取消后不具备业务类型功能" | — |
| 启用校验规则 | ☑ 已勾选 | "取消后已有规则将被清空" | 编辑时清空提示更严重 |
| 启用查重规则 | ☑ 已勾选 | "取消后已有规则将被清空" | 编辑时清空提示更严重 |
| 启用业务逻辑代码 | ☑ 已勾选 | "不再执行业务逻辑代码" | 检测是否有已安装脚本 |
| 启用团队成员 | ☐ 未勾选 | **不可关闭** | 一旦开启不可逆 |
| 启用系统动态流 | ☐ 未勾选 | "不再自动发送动态流" | — |

与老系统的差异：
| 维度 | 老系统 | 新系统 |

#### 保存逻辑
1. 前端校验：label 非空、apiKey 非空且 camelCase 格式
2. apiKey 自动追加 `__c` 后缀（自定义实体标识，与老系统一致）
3. 调用 `batchSaveMetadata({ createMap: { entity: [...] } })`
4. 同时创建主属性字段（item）：`batchSaveMetadata({ createMap: { item: [{ entityApiKey, apiKey: 'name', label: 主属性名称, itemType }] } })`
5. 自动创建 defaultBusiType（如果 busiTypeFlg=1）
6. 刷新左侧列表

#### 默认 apiKey 自动生成（P1，参考老系统 `getDefaultObjectApiKey`）

```typescript
// 前端自动生成默认 apiKey：customEntity + 序号
const generateDefaultApiKey = (existingEntities: XEntity[]): string => {
  const customEntities = existingEntities.filter(e => e.namespace === 'custom')
  let index = customEntities.length + 1
  let candidate = `customEntity${index}`
  const existingKeys = new Set(existingEntities.map(e => e.apiKey))
  while (existingKeys.has(candidate + '__c')) {
    index++
    candidate = `customEntity${index}`
  }
  return candidate  // 不含 __c，保存时自动追加
}
```

#### 完整保存流程（含 `__c` 后缀 + 主属性 + defaultBusiType）

```typescript
const handleCreateEntity = async (form: EntityCreateForm) => {

  // 1. 创建实体 + 主属性字段 + defaultBusiType（一个事务）
  await batchSaveMetadata({
    createMap: {
      entity: [{
        apiKey: entityApiKey,
        label: form.label,
        description: form.description,
        namespace: 'custom',
        customFlg: 1,
        enableFlg: 1,
        entityType: 1,  // 自定义
        historyLogFlg: form.historyLogFlg,
        busiTypeFlg: form.busiTypeFlg,
        checkRuleFlg: form.checkRuleFlg,
        duplicateRuleFlg: form.duplicateRuleFlg,
        scriptExecutorFlg: form.scriptExecutorFlg,
        groupMemberFlg: form.groupMemberFlg,
        dynamicFeedFlg: form.dynamicFeedFlg,
      }],
      // 主属性字段
      item: [{
        entityApiKey: entityApiKey,
        apiKey: 'name',
        label: form.mainItemLabel,
        itemType: form.mainItemType,
        requireFlg: 1,
        enableFlg: 1,
        customFlg: 0,  // 主属性是系统字段
        itemOrder: 1,
        // 自动编号类型：使用独立字段（新系统不再用 typeProperty JSON）

          format: form.mainItemFormat,          // dbc_varchar9

        } : {}),
      }],
      // 默认业务类型（如果启用了业务类型）

      } : {}),
    }
  })
}
```

#### 删除前引用检查（P0，参考老系统 `checkEntityFieldRelation`）

删除/禁用实体前，后端必须检查以下引用关系，任一存在则阻止操作：

| 检查项 | 检查方式 | 阻止提示 |
|:-------|:---------|:---------|
| 其他实体的关联字段引用 | 查 item 表中 `referEntityApiKey = 当前entity.apiKey` 的记录 | "该对象被 {引用方实体.label} 的字段 {item.label} 引用，不能删除/禁用" |
| 自定义按钮引用 | 查 entityLink 中 `childEntityApiKey = 当前entity.apiKey` 的关联 → 查父实体的 actionButton | "该对象被 {父实体.label} 的自定义按钮引用，不能删除/禁用" |
| 审批单引用 | 当 entity.businessCategory=2 时，检查是否被其他实体的按钮引用 | "该审批单被引用，不能删除/禁用" |

```java
// 后端实现（MetadataCascadeDeleteService 或 batchSaveMetadata 前置校验）
public void checkEntityDeletable(String entityApiKey) {
    // 1. 检查关联字段引用
    List<?> referItems = mergeReadService.listMergedAuto("item")
        .stream().filter(i -> entityApiKey.equals(i.getReferEntityApiKey()))
        .collect(toList());
    if (!referItems.isEmpty()) {
        throw new BusinessException("该对象被其他实体的字段引用，不能删除");
    }
    // 2. 检查 entityLink 引用
    List<?> links = mergeReadService.listMergedAuto("entityLink")
        .stream().filter(l -> entityApiKey.equals(l.getChildEntityApiKey()))
        .collect(toList());
    // ... 检查父实体的 actionButton
}
```

#### 删除前引用检查（P0，参考老系统 XItemAction.deleteXItemNew）

删除/禁用字段前，后端必须检查以下 6 种引用关系：

| # | 检查项 | 检查方式 | 错误码 | 阻止提示 |
|:--|:-------|:---------|:-------|:---------|
| 1 | 被查重规则引用 | 查 duplicateRuleCriteria 中 `itemApiKey = 当前item.apiKey` | 110004 | "该字段被查重规则引用，不能删除" |
| 2 | 被计算公式引用 | 查 formulaComputeItem 中引用了当前字段 | — | "该字段被计算公式引用，不能删除" |
| 3 | 被汇总聚合引用 | 查 aggregationComputeDetail 中引用了当前字段 | — | "该字段被汇总计算引用，不能删除" |
| 4 | 被 JOIN 关联引用 | 查 item 中 itemType=JOIN 且引用了当前字段 | 310002 | "该字段被关联字段引用，不能删除" |
| 5 | 被关联筛选引用 | 查 referenceFilter/referenceFilterDetail 中引用了当前字段 | — | "该字段被关联过滤引用，不能删除" |
| 6 | 被校验规则引用 | 查 checkRule 中 `checkErrorItemApiKey = 当前item.apiKey` | — | "该字段被校验规则引用，不能删除" |

```java
// 后端实现
public void checkItemDeletable(String entityApiKey, String itemApiKey) {
    // 1. 查重规则引用
    var criteria = mergeReadService.listMergedByEntityApiKey("duplicateRuleCriteria", entityApiKey);
    if (criteria.stream().anyMatch(c -> itemApiKey.equals(c.getItemApiKey()))) {
        throw new BusinessException(110004, "该字段被查重规则引用，不能删除");
    }
    // 2. 计算公式引用
    var formulaItems = mergeReadService.listMergedByEntityApiKey("formulaComputeItem", entityApiKey);
    if (formulaItems.stream().anyMatch(f -> itemApiKey.equals(f.getItemApiKey()))) {
        throw new BusinessException("该字段被计算公式引用，不能删除");
    }
    // 3. 汇总聚合引用
    var aggDetails = mergeReadService.listMergedByEntityApiKey("aggregationComputeDetail", entityApiKey);
    if (aggDetails.stream().anyMatch(a -> itemApiKey.equals(a.getItemApiKey()))) {
        throw new BusinessException("该字段被汇总计算引用，不能删除");
    }
    // 4~6 类似...
}
```

#### itemTypeProperty 新老对比（重要：新系统已拆分为独立字段）

> ⚠️ 老系统的 `itemTypeProperty` 是一个 JSON 字段（`typeProperty` 列），把所有类型特有属性塞在一起。

```typescript
  1,   // TEXT
  2,   // SELECT
  5,   // NUMBER

const filterItemsForDuplicateRule = (items: XEntityItem[]) => {
  return items.filter(item => {
    // 1. 不在支持类型白名单中 → 排除
```

## 三、后端接口设计

### 3.1 统一写接口：batchSaveMetadata

所有 5 个 Tab 的增删改操作统一通过一个接口完成：

```
POST /metadata/batch-save
Content-Type: application/json

{
  "create_map": { "<metamodelApiKey>": [{ ...fields }] },
  "update_map": { "<metamodelApiKey>": [{ ...fields }] },
  "delete_map": { "<metamodelApiKey>": [{ "api_key": "...", "entity_api_key": "..." }] }
}
```

支持在一个请求中同时操作多个元模型，事务内执行。

#### 各 Tab 对应的 metamodelApiKey

| Tab | 主元模型 | 子元模型 |
|:----|:---------|:---------|
| 基本信息 | `entity` | — |
| 字段管理 | `item` | `pickOption`（选项值） |
| 校验规则 | `checkRule` | — |
| 查重规则 | `duplicateRule` | `duplicateRuleCriteria`、`duplicateMatchingRule` |
| 字段组 | `fieldSet` | `fieldSetItem` |

#### 级联删除规则

`batchSaveMetadata` 的 `delete_map` 执行软删除（`delete_flg=1`）。后端 `MetadataCascadeDeleteService` 根据 `p_meta_link` 自动级联：

| 删除目标 | 自动级联删除 |
|:---------|:-------------|
| entity | item → pickOption, entityLink, checkRule, busiType → busiTypePickOption, duplicateRule → criteria/matchingRule, fieldSet → fieldSetItem, sharingRule → condition, dataPermission |
| item | pickOption, referenceFilter → detail, formulaCompute → item/factor, aggregationCompute → detail/factor |
| duplicateRule | duplicateRuleCriteria, duplicateMatchingRule |
| fieldSet | fieldSetItem |

## 六、与老系统的差异总结

| 维度 | 老系统 | 新系统 | 原因 |
|:-----|:-------|:-------|:-----|
| 实体范围 | 仅自定义 | 全部（按 namespace 分 Tab） | 管理员需要查看标准实体的配置 |
| 布局 | 列表页 + 弹框 | 左右分栏 + Tab | 减少页面跳转，信息密度更高 |
| 字段管理 | 跳转独立页面 | Tab 内嵌 | 统一工作台 |
| 子元数据 | 分散多个页面 | 5 个 Tab 统一 | 一站式配置 |
| apiKey 后缀 | 自动追加 `__c` | 保持 `__c` 约定 | 兼容老数据 |
| 主属性 | 弹框内设置 | 弹框内设置 | 保持一致 |
| 菜单同步 | 保存时同步 APP/Web 菜单 | 保存时同步 menu 元数据 | 新系统菜单元数据化 |
| 多态字段 | 弹框内勾选 | P2 阶段实现 | 复杂度高，先跳过 |
| 国际化同步 | 保存时调用聚合服务 | P2 阶段实现 | 依赖 i18n 基础设施 |
| License 检查 | 检查 CRM-CM 模块 | P2 阶段实现 | 依赖 License 服务 |
