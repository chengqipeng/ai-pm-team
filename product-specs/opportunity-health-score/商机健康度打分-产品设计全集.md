## 1. 业务背景与产品目标

### 1.1 业务痛点

**销售人员**跟进商机时，信息繁杂且分散，面对"项目正朝着签单推进还是偏离方向？""拿下项目的三大核心关键点是什么？"这类问题时，回答往往带有强烈主观色彩，或夸大其词、或过于消极、或模棱两可。这导致销售人员无法有效主导项目推进，造成商机转化延迟甚至丢单。

对于**销售经理和VP**来说，这种情况会阻碍商机复盘工作的高效开展，既无法精准预判公司销售目标达成情况，也难以科学调配销售资源、规划工作重点。

BANT、C139 等成熟的商机评估模型，可以实现商机健康度的量化评估。

### 1.2 产品目标

**面向销售人员：** 帮助销售人员清晰掌握商机当前的成果与不足，明确发力方向，及早采取改进或补救措施。

**面向销售管理者：** 帮助销售管理者精准判断销售人员对各项目的把控程度及项目走向，让管理者对所有在途商机了然于心——清晰区分稳操胜券的项目、需重点公关的项目，以及具体攻坚方向，进而结合公司整体战略，科学调配资源，实现全局战果最大化。

## 2. 商机评估方法论

| 方法论 | 说明 |
| --- | --- |
| **P6商机评估** | 销售易自创方法论。IPOV行业分析 + APOV客户分析 + P6对每一项进行评估，标记 risk off（绿色/无风险）、risk carry（黄色/有风险需了解）、risk on（红色/存在风险） |
| **C139方法论** | 多用于软件行业。从四个维度评估：内线coach、1决胜力、3个驱赢指标、9项必清事项。每项评估满足/不满足，将商机划分为赢单区、输单区、抖动区 |
| **BANT分析** | B（Budget预算）、A（Authority权限）、N（Need需求）、T（Timing时间） |
| **CCS销售方法论** | 把销售从"卖产品"转为"帮客户买"，以客户痛点、目标和决策逻辑为核心，7大核心原则 |
| **MEDDIC** | 量化指标(M)、经济买家(E)、决策流程(D)、决策标准(D)、识别痛点(I)、竞争态势(C) |

**整体结论：** ToB销售场景的商机评估维度，各方法论的评估维度基本一致，核心差异在于评估的细致程度，以及面向的场景不同（如简单的产品型销售没有解决方案维度的评估）。

## 3. 产品方案总览

### 3.1 整体思路

**产品逻辑：** 提示词模板嵌套健康度评估规则。健康度评估规则区分业务类型设置，运行时动态获取，通过变量的方式嵌入到提示词模板。

**功能界面：** 商机详情页增加「商机健康度评估」AI组件，展示商机健康度的整体及分维度评估结果；商机列表展示商机「健康度等级」、「健康度评分」信息。

## 5. 后台配置——商机健康度评估模板

### 5.1 配置入口

后台 NeoAgent → 销售智能 → 商机健康度评估，分为2个tab页签：「商机健康度评估模板」、「商机阶段目标管理」。

### 5.2 模板列表

- 字段：模板名称、模板描述、模板类型（自定义/系统标准）、创建人、最近修改人、最近修改时间
- 操作：
  - 系统标准模板：复制（不可编辑/删除）
  - 自定义模板：编辑、删除（已分配的模板不可删除）、复制
- 普通操作：新建商机健康度评估模板
- 模板分配：支持按照商机业务类型分配
- 支持按照模板名称进行模糊搜索
- 系统标准模板预置2个，使用common级数据，后续可通过更新预置模板传达最佳实践
- 模板数量上限50个

### 5.3 新建/编辑/复制模板

#### 5.3.1 基础信息

| 设置项 | 说明 |
| --- | --- |
| 模板名称 | 文本字段、必填、最长50字、同租户内不可重复 |
| 评估方法论 | 单选字段、非必填。选项：CCS(适用toB解决方案场景)、BANT(适用toB简单场景)、其他、无。无默认值。选择"其他"后增加「方法论名称」填写。选择CCS/BANT后弹窗确认是否替换当前已配置规则 |
| 评估的商机字段 | 多选字段、必填。下拉框选择商机字段，默认选中「商机名称」不可移除。最多选择50个字段，支持3级穿透 |
| 基础数据来源 | 特殊多选字段、非必填。最多选2个。选项包括：查询相关记录（预置商机明细、活动记录、联系人、联系人角色、联系人关系）、客户智能洞察（支持选择当前租户下所有已配置的洞察维度）、文档 |
| 满分制 | 单选、必填。选项：10分制（默认）/100分制 |
| 模板描述 | 文本域字段、非必填、最长200字 |

#### 5.3.2 分维度评估规则设置

- 支持设置多个评估维度，最少1个，最多6个
- 表单默认展示1个维度的录入界面

**每个维度包含：**

| 设置项 | 说明 |
| --- | --- |
| 维度名称 | 文本、必填、最多20字 |
| 维度权重 | 整数、单位：%、必填。所有维度权重之和须为100% |
| 数据来源 | 特殊多选字段、必填、最多选3个。选项同基础数据来源 |
| 评估要素 | 支持多个，最少1个，最多4个 |

**每个评估要素包含：**

| 设置项 | 说明 |
| --- | --- |
| 要素名称 | 文本、必填、最多20字 |
| 要素权重 | 整数、单位：%、必填。各要素权重和应达到100% |
| 评分规则 | 文本域、必填、最多700字 |

- 维度支持拖拽排序，顺序影响前台展示
- 删除维度/要素需二次弹窗确认

#### 5.3.3 健康度等级规则

- 预置选项集【商机健康度等级】，选项：优质、健康、预警、危机
- 用户可自定义添加其他选项（有相关数据的选项不允许删除）
- 每个模板最多配置4个等级，最少1个
- 每个等级设置：等级名称（必填）、最低分（包含，必填）、最高分（不包含，必填）
- 保存时校验等级是否覆盖全部分数区间

#### 5.3.4 功能设置

- 开关（默认打开）：商机阶段变更（推进/回退），自动触发重新评估
- 操作权限：支持按照相关用户（负责人、负责人的主管、编辑权限人员、查看权限人员）/指定用户/指定用户组设置。不设置则根据商机编辑权限控制

#### 5.3.5 评估模型输出设置

| 设置项 | 说明 |
| --- | --- |
| 评估结论输出要求 | 文本域、必填、最多700字。默认录入最佳实践：一个连贯的、不超过150字的自然段落，涵盖优势与短板、风险综述、影响分析 |
| 单维度评估结论输出要求 | 文本域、必填、最多700字。默认：不超过60字的自然段落，涵盖核心优势、关键短板、优先行动 |
| 风险说明输出要求 | 文本域、必填、默认录入最佳实践 |
| 关键进展输出要求 | 文本域、必填、默认录入最佳实践 |
| 未完成短板输出要求 | 文本域、默认录入最佳实践 |

- 支持插入变量（商机字段，最多5层穿透）
- 提供效果预览按钮

## 6. 后台配置——商机阶段目标管理

### 6.1 列表展示

按照商机业务类型，区分展示已配置的商机阶段：
- 字段：阶段名称、阶段状态、阶段目标、最近更新时间
- 操作：编辑目标（文本域，最多500字）

### 6.2 智能目标生成

- 弹窗二次确认，告知用户智能生成逻辑及将覆盖原有信息
- 确认后调用预置提示词，生成当前业务类型商机的阶段目标描述
- 同步执行，完成前当前业务类型列表不可操作
- 开关：新建/编辑阶段时，自动生成/更新阶段目标（默认不勾选）
- 处理逻辑：每天凌晨定时任务，对前一天新增或变更的阶段进行目标更新

## 7. 运行逻辑与评分算法

### 7.1 执行流程

1. 根据当前商机业务类型，匹配对应的商机健康度评估模板
2. **并发执行各分维度的评估Prompt**（最多6个）（使用推理模型）
3. 分维度评估全部返回结果后，**执行整体的评估Prompt**（使用普通模型）
4. 计算维度等级和商机总分

### 7.2 维度等级计算

为降低大模型返回结果的不稳定性，每个维度不直接展示最终分数，而是根据模型打分给出维度评估等级（放在阿波罗配置中）：

| 等级 | 分数区间（10分制） |
| --- | --- |
| 风险 | [0, 4) |
| 合格 | [4, 6) |
| 良好 | [6, 8) |
| 优秀 | [8, 10] |

### 7.3 商机总分计算

将分维度等级换算为得分（放在阿波罗配置中），再按照维度权重计算总分、按照分数区间查询等级：

| 维度等级 | 换算得分 |
| --- | --- |
| 风险 | 0（维度得分0分）/ 3（维度得分非0分） |
| 合格 | 5 |
| 良好 | 7 |
| 优秀 | 9（维度得分非10分）/ 10（维度得分10分） |

**总分公式：** Σ(各维度换算得分 × 维度权重)

### 7.4 数据存储

- 同一个商机最多存储10条评估结果，产生第11条时删除最早的一条
- 前台展示时取最新的一条
