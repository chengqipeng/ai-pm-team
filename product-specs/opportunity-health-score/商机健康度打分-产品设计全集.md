## 1. 业务背景与产品目标

### 1.1 业务痛点

**销售人员**跟进商机时，信息繁杂且分散，面对"项目正朝着签单推进还是偏离方向？""拿下项目的三大核心关键点是什么？"这类问题时，回答往往带有强烈主观色彩，或夸大其词、或过于消极、或模棱两可。这导致销售人员无法有效主导项目推进，造成商机转化延迟甚至丢单。

对于**销售经理和VP**来说，这种情况会阻碍商机复盘工作的高效开展，既无法精准预判公司销售目标达成情况，也难以科学调配销售资源、规划工作重点。

BANT、C139 等成熟的商机评估模型，可以实现商机健康度的量化评估。

### 1.2 产品目标

**面向销售人员：** 帮助销售人员清晰掌握商机当前的成果与不足，明确发力方向，及早采取改进或补救措施。

**面向销售管理者：** 帮助销售管理者精准判断销售人员对各项目的把控程度及项目走向，让管理者对所有在途商机了然于心——清晰区分稳操胜券的项目、需重点公关的项目，以及具体攻坚方向，进而结合公司整体战略，科学调配资源，实现全局战果最大化。

## 2. 商机评估方法论

| 方法论 | 说明 |
| --- | --- |
| **P6商机评估** | 销售易自创方法论。IPOV行业分析 + APOV客户分析 + P6对每一项进行评估，标记 risk off（绿色/无风险）、risk carry（黄色/有风险需了解）、risk on（红色/存在风险） |
| **C139方法论** | 多用于软件行业。从四个维度评估：内线coach、1决胜力、3个驱赢指标、9项必清事项。每项评估满足/不满足，将商机划分为赢单区、输单区、抖动区 |
| **BANT分析** | B（Budget预算）、A（Authority权限）、N（Need需求）、T（Timing时间） |
| **CCS销售方法论** | 把销售从"卖产品"转为"帮客户买"，以客户痛点、目标和决策逻辑为核心，7大核心原则 |
| **MEDDIC** | 量化指标(M)、经济买家(E)、决策流程(D)、决策标准(D)、识别痛点(I)、竞争态势(C) |

**整体结论：** ToB销售场景的商机评估维度，各方法论的评估维度基本一致，核心差异在于评估的细致程度，以及面向的场景不同（如简单的产品型销售没有解决方案维度的评估）。

## 3. 产品方案总览

### 3.1 整体思路

**产品逻辑：** 提示词模板嵌套健康度评估规则。健康度评估规则区分业务类型设置，运行时动态获取，通过变量的方式嵌入到提示词模板。

**功能界面：** 商机详情页增加「商机健康度评估」AI组件，展示商机健康度的整体及分维度评估结果；商机列表展示商机「健康度等级」、「健康度评分」信息。

## 5. 后台配置——商机健康度评估模板

### 5.1 配置入口

后台 NeoAgent → 销售智能 → 商机健康度评估，分为2个tab页签：「商机健康度评估模板」、「商机阶段目标管理」。

### 5.2 模板列表

- 字段：模板名称、模板描述、模板类型（自定义/系统标准）、创建人、最近修改人、最近修改时间
- 操作：
  - 系统标准模板：复制（不可编辑/删除）
  - 自定义模板：编辑、删除（已分配的模板不可删除）、复制
- 普通操作：新建商机健康度评估模板
- 模板分配：支持按照商机业务类型分配
- 支持按照模板名称进行模糊搜索
- 系统标准模板预置2个，使用common级数据，后续可通过更新预置模板传达最佳实践
- 模板数量上限50个

### 5.3 新建/编辑/复制模板

#### 5.3.1 基础信息

| 设置项 | 说明 |
| --- | --- |
| 模板名称 | 文本字段、必填、最长50字、同租户内不可重复 |
| 评估方法论 | 单选字段、非必填。选项：CCS(适用toB解决方案场景)、BANT(适用toB简单场景)、其他、无。无默认值。选择"其他"后增加「方法论名称」填写。选择CCS/BANT后弹窗确认是否替换当前已配置规则 |
| 评估的商机字段 | 多选字段、必填。下拉框选择商机字段，默认选中「商机名称」不可移除。最多选择50个字段，支持3级穿透 |
| 基础数据来源 | 特殊多选字段、非必填。最多选2个。选项包括：查询相关记录（预置商机明细、活动记录、联系人、联系人角色、联系人关系）、客户智能洞察（支持选择当前租户下所有已配置的洞察维度）、文档 |
| 满分制 | 单选、必填。选项：10分制（默认）/100分制 |
| 模板描述 | 文本域字段、非必填、最长200字 |

#### 5.3.2 分维度评估规则设置

- 支持设置多个评估维度，最少1个，最多6个
- 表单默认展示1个维度的录入界面

**每个维度包含：**

| 设置项 | 说明 |
| --- | --- |
| 维度名称 | 文本、必填、最多20字 |
| 维度权重 | 整数、单位：%、必填。所有维度权重之和须为100% |
| 数据来源 | 特殊多选字段、必填、最多选3个。选项同基础数据来源 |
| 评估要素 | 支持多个，最少1个，最多4个 |

**每个评估要素包含：**

| 设置项 | 说明 |
| --- | --- |
| 要素名称 | 文本、必填、最多20字 |
| 要素权重 | 整数、单位：%、必填。各要素权重和应达到100% |
| 评分规则 | 文本域、必填、最多700字 |

- 维度支持拖拽排序，顺序影响前台展示
- 删除维度/要素需二次弹窗确认

#### 5.3.3 健康度等级规则

- 预置选项集【商机健康度等级】，选项：优质、健康、预警、危机
- 用户可自定义添加其他选项（有相关数据的选项不允许删除）
- 每个模板最多配置4个等级，最少1个
- 每个等级设置：等级名称（必填）、最低分（包含，必填）、最高分（不包含，必填）
- 保存时校验等级是否覆盖全部分数区间

#### 5.3.4 功能设置

- 开关（默认打开）：商机阶段变更（推进/回退），自动触发重新评估
- 操作权限：支持按照相关用户（负责人、负责人的主管、编辑权限人员、查看权限人员）/指定用户/指定用户组设置。不设置则根据商机编辑权限控制

#### 5.3.5 评估模型输出设置

| 设置项 | 说明 |
| --- | --- |
| 评估结论输出要求 | 文本域、必填、最多700字。默认录入最佳实践：一个连贯的、不超过150字的自然段落，涵盖优势与短板、风险综述、影响分析 |
| 单维度评估结论输出要求 | 文本域、必填、最多700字。默认：不超过60字的自然段落，涵盖核心优势、关键短板、优先行动 |
| 风险说明输出要求 | 文本域、必填、默认录入最佳实践 |
| 关键进展输出要求 | 文本域、必填、默认录入最佳实践 |
| 未完成短板输出要求 | 文本域、默认录入最佳实践 |

- 支持插入变量（商机字段，最多5层穿透）
- 提供效果预览按钮

## 6. 后台配置——商机阶段目标管理

### 6.1 列表展示

按照商机业务类型，区分展示已配置的商机阶段：
- 字段：阶段名称、阶段状态、阶段目标、最近更新时间
- 操作：编辑目标（文本域，最多500字）

### 6.2 智能目标生成

- 弹窗二次确认，告知用户智能生成逻辑及将覆盖原有信息
- 确认后调用预置提示词，生成当前业务类型商机的阶段目标描述
- 同步执行，完成前当前业务类型列表不可操作
- 开关：新建/编辑阶段时，自动生成/更新阶段目标（默认不勾选）
- 处理逻辑：每天凌晨定时任务，对前一天新增或变更的阶段进行目标更新

## 7. 运行逻辑与评分算法

### 7.1 执行流程

1. 根据当前商机业务类型，匹配对应的商机健康度评估模板
2. **并发执行各分维度的评估Prompt**（最多6个）（使用推理模型）
3. 分维度评估全部返回结果后，**执行整体的评估Prompt**（使用普通模型）
4. 计算维度等级和商机总分

### 7.2 维度等级计算

为降低大模型返回结果的不稳定性，每个维度不直接展示最终分数，而是根据模型打分给出维度评估等级（放在阿波罗配置中）：

| 等级 | 分数区间（10分制） |
| --- | --- |
| 风险 | [0, 4) |
| 合格 | [4, 6) |
| 良好 | [6, 8) |
| 优秀 | [8, 10] |

### 7.3 商机总分计算

将分维度等级换算为得分（放在阿波罗配置中），再按照维度权重计算总分、按照分数区间查询等级：

| 维度等级 | 换算得分 |
| --- | --- |
| 风险 | 0（维度得分0分）/ 3（维度得分非0分） |
| 合格 | 5 |
| 良好 | 7 |
| 优秀 | 9（维度得分非10分）/ 10（维度得分10分） |

**总分公式：** Σ(各维度换算得分 × 维度权重)

### 7.4 数据存储

- 同一个商机最多存储10条评估结果，产生第11条时删除最早的一条
- 前台展示时取最新的一条

## 8. 前台展示——列表与详情页组件

### 8.1 商机实体新增字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 健康度评分 oppHealthScore | 实数字段 | 只读，只可通过健康度评估功能更新。支持列表排序、筛选、自定义视图 |
| 健康度等级 oppHealthGrade | 单选字段 | 只读，列表展示特殊样式。等级选项根据预置通用选项集【商机健康度等级】展示 |

### 8.2 详情页布局配置

- 商机详情页布局可配置「商机健康度评估」组件，受license控制
- 组件在一个布局中仅允许添加一次，预置在「AI组件」分组
- 自动初始化：租户开通function后，商机所有详情页布局自动创建Tab及组件

### 8.3 详情页头部

- 展示商机等级 & 得分
- 展示条件：用户有function且等级/得分有值
- 点击跳转到健康度评估组件

### 8.4 前台组件展示——商机整体评估

**加载逻辑：**
- 展示当前商机最新的评估结果
- 如果没有任何评估结果：自动触发首次评估（不判断当前用户权限）
- 如果模板配置了客户洞察作为数据来源但关联客户无洞察信息，则自动触发客户洞察生成

**实时提示处理进度：**
- 智能客户洞察中，预计需要几分钟，请稍候...
- 单项维度评估中，预计需要几分钟，请稍候...
- 整体健康度评估中，即将完成...

**整体评估展示信息：**
- 评估依据的方法论（模板选择"无"则不展示）
- 商机健康度等级（如优质/健康/预警/危机），hover展示等级划分规则
- 商机评分，hover展示评分计算逻辑
- 商机整体评估结论（最多展示5行，超出后点击展开）
- 分维度雷达图（最多6个维度），展示各维度名称和评估等级
  - hover展示维度权重、关键进展和未完成短板数量
  - 点击维度名称跳转定位到相关维度卡片
- 评估结果生成时间 + 刷新操作
- 点赞/点踩反馈

### 8.5 前台组件展示——单维度评估

- 最多6个维度分析卡片，默认收起
- 支持一键展开/收起，也可单个操作

**展开的维度卡片包含：**
- 维度名称、维度评估等级（风险/合格/良好/优秀）
- 维度评估结论
- **风险说明**（含风险标签）：
  - 前序阶段遗留风险（红色）：此风险应在之前阶段完成，已作为历史债务传导至本阶段
  - 本阶段重点关注风险（黄色）：此风险严重偏离当前阶段目标，需立即介入
  - 后续阶段承接风险（绿色）：此风险属后续阶段的处理范围
  - hover在风险标签上展示详细说明
- **未完成短板List**：展示0～N个未完成短板，带"未找到相关系统数据支撑"提示
- **关键进展List**：展示0～N个关键进展
  - 每个进展后展示"依据"icon，hover展示1～N个依据
  - 依据中引用原文需判断用户数据权限（无权限则80%数据加掩码）
  - 来源数据徽标：CRM数据点击跳转详情页；客户洞察数据不展示徽标；联系人关系/角色不展示徽标
- 点赞/点踩反馈

### 8.6 商机列表

- 列表可配置展示「商机得分」、「商机健康度」字段
- 等级有值时，hover展示评估卡片（评分、等级、评估结论、分维度雷达图）
- 卡片展示条件：用户有商机健康度评估function
- 浮窗高度自适应，最大高度580px

### 8.7 看板列表（Kanban）

- 商机卡片右上角显示健康度等级（特殊样式）
- 点击展示评估卡片

## 9. 触发机制

### 9.1 手动单条商机触发

- 商机详情页组件上刷新按钮，立即触发
- 权限控制：根据后台模板配置的权限校验，或走商机编辑权限

### 9.2 业务事件触发

- 商机增加触发事件"商机健康度评估"
- 受租户级"商机健康度评估"function控制，不校验用户权限
- 操作类型：业务事件，实体对象：销售机会
- **执行逻辑：** 通过业务事件触发的评估不实时执行，晚上定时统一执行。当天触发多次也仅执行一次。如果当天已手动触发执行，则晚上不再执行

### 9.3 自动触发

- 商机阶段变更（推进/回退），如果模板开关打开，自动触发重新评估（晚上24点定时执行）

## 10. Prompt模板设计

### 10.1 CCS方法论Prompt


## 11. 异常处理与日志审计

### 11.1 异常处理

**网络错误：**
- 自动重试3次，重试间隔：2秒、5秒、10秒（指数退避）
- 3次重试失败后提示"网络连接出现问题，请检查后重试"

**模型调用失败：**
- 自动重试3次，重试间隔：2秒、5秒、10秒（指数退避）
- 3次重试失败后提示"评估服务暂时不可用，请稍后重试"

**未配置评估模板：** 提示"请联系管理员为该商机类型分配健康度评估模板"

### 11.2 日志审计

记录内容：
- 评估触发时间
- 评估触发人
- 评估触发方式（手动/自动）
- 评估耗时
- 评估状态（成功/失败）
- 失败原因（如有）
- Prompt输入（裁剪后）+ 输出

### 11.3 Tokens记录

- 单独记录tokens，按照生成结果的内容进行用户tokens消耗记录

### 11.4 点赞/点踩

- 整体评估及单维度评估支持反馈准确/不准确，跟随每次评估结果记录

### 11.5 BI

- 支持在BI中配置商机健康度评估实体（受license控制）
- 最佳实践报表：以商机维度聚合，展示单个商机下各维度的评估结论

## 13. 待优化项与产品深层思考

> 来源：待优化项整理页面

### 13.1 数据源丰富

- 阶段任务的实例数据（关键事件均为布尔数据，需考虑如何应用）
- 商机阶段停留时间
- 商机阶段的基准值、保底值
- 商机阶段轨迹
- 文档（商机字段、关键事件文档、相关文档、聚合文档）
- 富文本字段

### 13.2 体验优化

- 自定义查询记录编辑不了对象
- 最佳实践缺少活动记录内容
- 阶段推进自动更新默认关闭
- 提示词调优：相似内容以最新记录为准
- 调整维度等级对应的分数（目前设定导致总分较低）
- 展示推理原因，帮助运营调整提示词
- BI看板：json数据样式处理 & BI最佳实践
- 评估字段考虑下沉到各个维度

### 13.3 要素打分规则辅助生成

- 方案一：按分数区间结构化设置（优点：结构化控制；缺点：交互不友好，嵌套3层）
- 方案二：AI辅助判断评分规则是否OK，或直接根据要素生成评分规则

### 13.4 产品深层思考

**过程管理 VS 预测分析：**
- 当前模型是一个线性的、加权平均的"进度"汇总，并非精确的"赢率预测"模型
- 理想的健康度评估不应只是线性加权平均，而应能识别关键胜负手
- 考虑引入非线性权重或一票否决/一票通过的逻辑

**评分模型权重含义：**
- 在不同场景下代表含义不同：完成各项销售动作的"进度比例" vs 各项因素对最终赢单的"贡献度"

**MEDDIC参考：**
- MEDDIC是「过程/质量管控工具」，逐维度检查商机是否具备赢单条件
- 不直接"算预测"，但每一项达标都对应真实赢率
- 输入质量决定输出质量，消除不确定性

### 13.5 评估结果修正

- 管理者在复盘中发现评估结果与实际不符时的修正机制
- 初步思路：部分角色可手动给商机打标签（锁定赢单/出局商机）及添加说明，但不影响原有评估结果

### 13.6 已知Bug与待修复项

- 新建模板选择CCS方法论不带入评估规则
- 修改预置活动记录报错
- 最佳实践等级展示为数字1234
- 保存模板按钮未防止重复点击
- 字段翻译问题
- 得0分时没有任何未完成项
- BI单维度评估结论去掉json格式
- 健康度等级红/黄标签颜色太相近

## 14. 早期方案（2025版本）对比参考

> 来源：销售经理-「商机健康度评估」（2025-05-06 v2）

### 14.1 早期方案概述

2. 获取「商机跟进情况」：商机信息、客户信息、以及相关信息
