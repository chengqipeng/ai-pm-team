## 三、测试公式设计（10 个，覆盖 30+ 函数）

### F1: 客户等级标签（CASE + TEXT + UPPER）

```
CASE(account.level,
  'A(重点客户)', 'VIP',
  'B(普通客户)', 'STANDARD',
  'C(非优先客户)', 'LOW',
  'UNKNOWN')
```

- resultType: 1 (TEXT)
- 覆盖函数: **CASE**
- 依赖: level
- 预期: level='A(重点客户)' → 'VIP', level='B(普通客户)' → 'STANDARD'

### F2: 客户名称首字母拼音（PINYINFIRSTLETTER + UPPER + LEFT）

```
UPPER(LEFT(PINYINFIRSTLETTER(account.accountName), 1))
```

- resultType: 1 (TEXT)
- 覆盖函数: **PINYINFIRSTLETTER, UPPER, LEFT**
- 依赖: accountName (name 固定列)
- 预期: 'K_甲gOVhO...' → 拼音首字母 → 取第一个 → 大写

### F3: 客户价值评分（IF + AND + OR + 嵌套 IF + 算术）

```
IF(AND(account.totalWonOpportunities > 3, account.totalOrderAmount > 100000),
  100,
  IF(OR(account.totalWonOpportunities > 0, account.totalContract > 0),
    IF(account.totalOrderAmount > 50000, 80, 60),
    IF(account.totalActiveOrders > 0, 40, 0)
  )
)
```

- resultType: 6 (REAL/NUMBER)
- 覆盖函数: **IF(3层嵌套), AND, OR**
- 依赖: totalWonOpportunities, totalOrderAmount, totalContract, totalActiveOrders
- 预期: 根据多条件组合返回 0/40/60/80/100

### F4: 应收账款健康度（IF + ISNULL + 算术 + ROUND + 百分比）

```
IF(ISNULL(account.actualInvoicedAmount),
  0,
  ROUND(account.paidAmount / account.actualInvoicedAmount * 100, 2)
)
```

- resultType: 33 (PERCENTAGE)
- 覆盖函数: **IF, ISNULL, ROUND**
- 依赖: actualInvoicedAmount, paidAmount
- 预期: paidAmount=9012, invoiced=23112 → ROUND(9012/23112*100, 2) = 38.99

### F5: 订单均价（IF + NULLVALUE + 算术）

```
IF(account.totalActiveOrders > 0,
  NULLVALUE(account.totalOrderAmount, 0) / account.totalActiveOrders,
  0
)
```

- resultType: 6 (REAL)
- 覆盖函数: **IF, NULLVALUE**
- 依赖: totalActiveOrders, totalOrderAmount
- 预期: totalOrderAmount=29438, totalActiveOrders=1 → 29438

### F6: 客户名称长度分类（LEN + IF + 嵌套）

```
IF(LEN(account.accountName) > 20, '长名称',
  IF(LEN(account.accountName) > 10, '中等名称', '短名称')
)
```

- resultType: 1 (TEXT)
- 覆盖函数: **LEN, IF(2层)**
- 依赖: accountName
- 预期: 'K_甲gOVhO辛MgrM未戊丙MYES' (22字符) → '长名称'

### F7: 赢单金额占比文本（TEXT + ROUND + IF + 算术 + 拼接）

```
IF(account.totalOrderAmount > 0,
  TEXT(ROUND(account.totalWonOpportunityAmount / account.totalOrderAmount * 100, 1)),
  '0')
```

- resultType: 1 (TEXT)
- 覆盖函数: **TEXT, ROUND, IF**
- 依赖: totalWonOpportunityAmount, totalOrderAmount
- 预期: 19294/29438*100 → ROUND(65.5, 1) → TEXT → '65.5'

### F8: 客户活跃天数（日期计算 + ABS + FLOOR）

```
FLOOR(ABS(account.updatedAt - account.createdAt) / 86400000)
```

- resultType: 6 (REAL/NUMBER)
- 覆盖函数: **FLOOR, ABS**
- 依赖: updatedAt(固定列), createdAt(固定列)
- 说明: 毫秒时间戳相减 / 86400000 = 天数
- 预期: (1638396001382 - 1522305803860) / 86400000 = FLOOR(1343.4) = 1343

> 注意：updatedAt/createdAt 是固定列，需要在 item 元数据中注册才能被公式引用。或者用已有的 dbc_bigint 列。

### F9: 综合评级（MAX + MIN + MOD + 多层嵌套）

```
IF(MAX(account.totalWonOpportunities, account.totalActiveOrders) > 5,
  'A',
  IF(MIN(account.totalWonOpportunities, account.totalActiveOrders) > 0,
    IF(MOD(account.totalWonOpportunities + account.totalActiveOrders, 2) = 0, 'B', 'C'),
    'D'
  )
)
```

- resultType: 1 (TEXT)
- 覆盖函数: **MAX, MIN, MOD, IF(3层)**
- 依赖: totalWonOpportunities, totalActiveOrders
- 预期: 根据赢单数和订单数的组合返回 A/B/C/D

### F10: 客户名称处理（TRIM + MID + CONTAINS + BEGINS + SUBSTITUTE）

```
IF(BEGINS(account.accountName, 'K_'),
  SUBSTITUTE(MID(account.accountName, 3, 5), '甲', '*'),
  TRIM(account.accountName)
)
```

- resultType: 1 (TEXT)
- 覆盖函数: **BEGINS, MID, SUBSTITUTE, TRIM, IF**
- 依赖: accountName
- 预期: 'K_甲gOVhO...' → BEGINS=true → MID(3,5)='甲gOVh' → SUBSTITUTE='*gOVh'

## 六、验证方法

### 6.1 每个公式的预期结果计算

对 100 条数据中的每一条，用 Python 独立计算预期结果，然后与公式引擎的计算结果比对。

### 6.2 验证维度

| 维度 | 验证内容 |
|:---|:---|
| 函数正确性 | 每个函数的输入输出是否符合预期 |
| 嵌套正确性 | 多层 IF 嵌套、函数嵌套的求值顺序 |
| 类型转换 | NUMBER→TEXT（TEXT函数）、TEXT→NUMBER（VALUE函数） |
| 空值处理 | ISNULL/NULLVALUE 对 NULL 参数的处理 |
| 精度 | ROUND 的四舍五入、DECIMAL 精度 |
| 边界条件 | 除零、空字符串、超长文本 |

### 6.3 成功标准

| 公式 | 匹配标准 |
|:---|:---|
| TEXT 类型 | 字符串精确匹配 |
| NUMBER/CURRENCY | 精度容差 < 0.01 |
| PERCENT | 精度容差 < 0.01 |
| 整体 | 100 条数据 100% 匹配 |
