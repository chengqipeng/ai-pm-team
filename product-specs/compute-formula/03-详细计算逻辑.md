# 计算公式引擎设计方案 · 03 详细计算逻辑

## 一、新建场景（create）

### 完整流程

```
EntityDataService.create(tenantId, entityApiKey, fixedData, extColumns)
  │
  ├─ 1. 加载元数据
  │     FormulaMetaLoader.load(tenantId, entityApiKey)
  │     → FormulaCompute 列表（公式定义）
  │     → FormulaComputeItem 列表（参数引用）
  │     → EntityItem 列表（字段类型信息）
  │
  ├─ 2. 初始化汇总字段
  │     for each aggregateItem in allItems:
  │       if aggregateComputeType in (DATE, DATETIME, TIME):
  │         data[item.apiKey] = null
  │       else:
  │         data[item.apiKey] = 0.0
  │
  ├─ 3. 分离特殊公式
  │     specialFormulas = formulas.filter(f ->
  │       f.params.any(p -> isAutoNumber(p) || isDomesticCurrency(p)))
  │     normalFormulas = formulas - specialFormulas
  │
  ├─ 4. 拓扑排序
  │     normalFormulas = DependencyAnalyzer.sort(normalFormulas)
  │
  ├─ 5. 批量取参数值
  │     paramValues = ParamValueResolver.batchResolve(dataList, normalFormulas, tenantId)
  │
  ├─ 6. 批量求值
  │     results = ExpressionEvaluator.batchEvaluate(dataList, normalFormulas, paramValues)
  │
  ├─ 7. 写回计算结果
  │     for each data, result:
  │       data.extColumns.putAll(result)
  │
  ├─ 8. 持久化（含公式字段值）
  │     routeService.executeWithRoute(tenantId, entityApiKey, () -> dataDao.save(data))
  │
  └─ 9. 计算特殊公式（入库后）
        computeSpecialFormulas(tenantId, entityApiKey, savedDataList, specialFormulas)
        → 计算结果单独 UPDATE 写库
        → 发 MQ：AGGREGATE_OBJECT_FILTER（触发父实体汇总）
```

### 特殊公式计算（入库后）

```
mapNewFormula = specialFormulas（初始）
loopCount = 20

while (!mapNewFormula.isEmpty() && loopCount-- > 0):
  for each formula in mapNewFormula:
    1. 取参数值（自增字段此时已有值，本币字段依赖已写库的汇率）
    2. 求值
    3. 写入 data[formula.itemApiKey]
    4. 记录 specialItemApiKey（该公式字段 apiKey）

  mapNewFormula.clear()

  // 检查是否有其他公式依赖了刚计算的特殊字段
  for each formula in allFormulas:
    if formula.params.any(p -> specialItemApiKey.contains(p.referItemApiKey)):
      mapNewFormula.put(formula)

  specialItemApiKey.clear()

// 批量 UPDATE 写库
updateBatchFormulaFields(tenantId, entityApiKey, savedDataList, changedFields)
```

## 二、增量更新场景（update increment）

### 完整流程

```
EntityDataService.update(tenantId, entityApiKey, id, fixedUpdates, extColumns)
  │
  ├─ 1. 查询旧数据
  │     oldData = getById(tenantId, entityApiKey, id)
  │
  ├─ 2. 合并新旧数据（构造完整数据）
  │     fullData = merge(oldData, newData)
  │
  ├─ 3. 确定变更字段集合
  │     changedItems = diffChangedItems(fullData, oldData, changedApiKeys)
  │     → 对比每个字段新旧值
  │     → 复选框（CHECK）：排序后比较数组
  │     → 文件/图片：比较 fileMetaId 集合
  │     → 公式字段/汇总字段：跳过（不允许外部更新）
  │     → 本币字段（currencyPart=2）：跳过
  │     → 货币字段变更处理：
  │         if 汇率字段（currencyRate）变更 → 所有本币字段加入 changedItems
  │         if 原币字段（multiCurrency）变更 → 对应本币字段加入 changedItems
  │
  ├─ 4. 查找受影响的公式（递归）
  │     affectedFormulas = findAffectedFormulas(changedItems, allFormulas)
  │     → 递归逻辑：
  │         changedExprs = changedItems.map(item -> entityApiKey + "." + item.apiKey)
  │         for each formula:
  │           if formula.params.any(p -> changedExprs.contains(p) or p.startsWith(changedExpr+".")):
  │             affectedFormulas.add(formula)
  │             changedItems.add(formula.item)  // 公式字段也变更了，继续递归
  │         递归直到 changedItems 不再增加
  │
  ├─ 5. 拓扑排序 affectedFormulas
  │
  ├─ 6. 批量取参数值 + 批量求值（同新建）
  │
  ├─ 7. 写回计算结果到 fullData
  │
  ├─ 8. 构建父实体汇总条件
  │     for each changedItem that is a link field (referItem):
  │       aggregateConditions.add({
  │         parentEntityApiKey: aggregateRule.entityApiKey,
  │         ruleId: aggregateRule.id,
  │         parentDataIds: [oldData[item.apiKey], newData[item.apiKey]]  // 新旧父实体都要重算
  │       })
  │
  ├─ 9. 构建字段变更消息
  │     itemFilterConditions.add({
  │       entityApiKey, dataId, changedItemIds
  │     })
  │
  ├─ 10. 持久化（含公式字段值）
  │
  ├─ 11. 计算特殊公式（同新建）
  │
  └─ 12. 发 MQ（后置异步）
         AGGREGATE_ITEM_FILTER → 父实体汇总重算
         AGGREGATE_ITEM_PARENT_FILTER → 旧父实体汇总重算
         FORMULA_FILTER → 子实体公式重算
```

## 三、批量增量更新场景（batch update increment）

### 与单条的差异

```
单条：逐条 diffChangedItems → 逐条 findAffectedFormulas → 逐条求值
批量：
  1. 按变更字段集合分组（相同变更字段的数据合并）
     key = sorted(changedApiKeys).join("")
     groups = dataList.groupBy(key)

  2. 每组内：
     a. findAffectedFormulas（一次，所有数据共用）
     b. batchResolveParams（一次批量查询，所有数据的参数）
     c. batchEvaluate（循环每条数据，但参数已批量取好）

  3. 汇总条件和字段变更消息：按组合并，避免重复
```

### 批量参数取值优化

```
// 跨实体参数批量取值
for each formula in affectedFormulas:
  for each param in formula.params where param.referEntityApiKey != null:
    // 收集所有数据的关联 ID
    referIds = dataList.map(d -> d[param.referItemApiKey]).distinct()

    // 一次查询取所有关联数据的目标字段值
    results = SELECT id, {targetDbColumn}
              FROM {shardTable}
              WHERE id IN ({referIds})
                AND entity_api_key = {param.referEntityApiKey}

    // 按 referDataId 映射
    resultMap = results.toMap(id -> value)

    // 回填到每条数据的参数值
    for each data:
      referDataId = data[param.referItemApiKey]
      paramValues[data.id][param.referItemApiKey] = resultMap[referDataId]
```

## 四、全量更新场景（batch full update）

### 与增量更新的差异

全量更新传入完整的新数据，不传 changedApiKeys，需要自行对比：

```
for each (newData, oldData) pair:
  changedItems = []
  for each item in allItems:
    if item.itemType == CHECK:
      if sorted(newCheckValues) != sorted(oldCheckValues): changedItems.add(item)
    elif item.itemType in (FORMULA, AGGREGATE):
      skip  // 不对比公式/汇总字段
    elif item.itemType in (IMAGE, FILE):
      if fileSetChanged(newFiles, oldFiles): changedItems.add(item)
    else:
      if newData[item.apiKey] != oldData[item.apiKey]: changedItems.add(item)
  handleCurrencyChanged(changedItems, allItems)
```

后续流程与批量增量更新相同。

## 五、实时计算字段（realTimeComputeFlg=1）

### 触发时机

查询时计算，不在写入时计算。适用于依赖当前时间、当前用户等动态值的公式。

### 查询时计算流程

```
EntityDataService.getById / listPage 返回数据后：
  1. 过滤出 realTimeComputeFlg=1 的公式字段
  2. 对返回的数据列表批量计算
  3. 将计算结果覆盖数据中的字段值（不写库）
  4. 返回给调用方
```

### 缓存策略

实时字段计算结果不缓存（因为依赖动态值），但公式定义本身走 FormulaMetaLoader 缓存。

## 六、默认值计算（DefaultValueEngine）

### 触发时机

新建时，在公式计算之前执行。

### 字段分类

```
1. 简单默认值（无参数表达式）
   → 直接 ComputeUtils.evaluate(defaultValue, {})
   → 示例：defaultValue = "0"、"true"、"$CurrentUser"

2. 表达式默认值（含参数，依赖其他字段）
   → 先取参数值，再 evaluate
   → 示例：defaultValue = "{account.amount} * 0.1"

3. 关联字段默认值（referEntityApiKey != null）
   → 需要先计算关联字段的默认值，再计算依赖它的字段
   → 拓扑排序后按序计算

4. 动态默认值（isDynamicDefaultValue=true，单选/复选）
   → 结果为 optionApiKey，需转换为 optionCode 存储
```

### 计算顺序

```
1. 简单默认值（非关联）
2. 表达式默认值（非关联，无依赖）
3. 关联字段默认值（拓扑排序后按序）
4. 依赖关联字段的普通字段默认值
```

## 七、货币字段特殊处理

### 多货币字段变更触发本币重算

```
if changedItems.contains(currencyRate):
  // 汇率变更 → 所有本币字段（currencyPart=2）加入变更集合
  allItems.filter(item -> item.currencyPart == 2).forEach(changedItems::add)
else:
  // 原币字段（multiCurrencyFlg=1）变更 → 对应本币字段加入变更集合
  for each changedItem where changedItem.multiCurrencyFlg == 1:
    domesticItem = allItems.find(item -> item.apiKey == changedItem.apiKey + "_domestic")
    if domesticItem != null: changedItems.add(domesticItem)
```

### 本币字段计算

本币字段（currencyPart=2）属于特殊公式，在数据入库后计算：
```
domesticValue = originalValue * currencyRate
```
其中 originalValue 和 currencyRate 从已入库的数据中读取。

## 八、主子明细实体特殊处理

子实体公式引用主实体字段时，新建/更新子实体时主实体数据可能尚未入库（或刚更新）。

### 处理方式

```
// 新建/更新子实体时，将主实体数据通过 masterContext 传入
masterContext = {
  masterEntityApiKey: "account",
  masterItemApiKey: "accountId",  // 子实体上的关联字段
  masterData: { amount: 1000, ... }  // 主实体当前数据
}

// ParamValueResolver 取跨实体参数时：
if param.referEntityApiKey == masterContext.masterEntityApiKey
   && currentData[masterContext.masterItemApiKey] == masterContext.masterData.id:
  // 直接从 masterContext.masterData 取值，不查库
  return masterContext.masterData[param.referItemApiKey]
```

## 九、错误处理策略

| 错误类型 | 处理方式 |
|---|---|
| 公式表达式语法错误 | 抛 BusinessException，阻断写入 |
| 参数字段不存在 | 抛 BusinessException（ERROR_DATA_FORMULAR_NO_ITEM），阻断写入 |
| 参数实体不存在 | 抛 BusinessException（ERROR_DATA_FORMULAR_NO_ENTITY），阻断写入 |
| 参数取值查库失败 | 抛 PaasException，阻断写入 |
| 公式求值运行时异常 | 记录 warn 日志，抛 PaasException，阻断写入 |
| 异步 MQ 发送失败 | 记录 ComputeErrorLog，不阻断主流程 |
| 汇总计算失败 | 记录错误日志，不影响其他汇总字段 |
| 批量查询数量不一致 | 降级为逐条查询（IllegalBusinessDataException 触发） |
