# 计算公式引擎设计方案 · 04 接口设计与实现要点

## 一、核心接口定义

### ComputePipeline

```java
public interface ComputePipeline {

    /**
     * 新建前置计算
     * @return specialFormulas 需要入库后单独计算的特殊公式
     */
    Map<String, FormulaCompute> computeForCreate(
        long tenantId, String entityApiKey,
        List<Map<String, Object>> dataList,
        List<EntityItem> allItems);

    /**
     * 增量更新前置计算（单条）
     */
    ComputeResult computeForIncrementUpdate(
        long tenantId, String entityApiKey,
        Map<String, Object> newData, Map<String, Object> oldData,
        Set<String> changedApiKeys,
        List<EntityItem> allItems,
        IncrementUpdateContext ctx);

    /**
     * 增量更新前置计算（批量）
     */
    Map<Long, ComputeResult> computeForIncrementUpdateBatch(
        long tenantId, String entityApiKey,
        List<Map<String, Object>> newDataList,
        Map<Long, Map<String, Object>> oldDataMap,
        List<Set<String>> changedApiKeysList,
        List<EntityItem> allItems,
        IncrementUpdateContext ctx);

    /**
     * 全量更新前置计算
     */
    List<ComputeResult> computeForFullUpdate(
        long tenantId, String entityApiKey,
        List<Map<String, Object>> newDataList,
        List<Map<String, Object>> oldDataList,
        List<EntityItem> allItems);

    /**
     * 特殊公式计算（入库后调用）
     */
    void computeSpecialFormulas(
        long tenantId, String entityApiKey,
        List<Map<String, Object>> savedDataList,
        Map<String, FormulaCompute> specialFormulas,
        List<EntityItem> allItems);

    /**
     * 查询时实时计算
     */
    void computeForQuery(
        long tenantId, String entityApiKey,
        List<Map<String, Object>> dataList,
        List<EntityItem> allItems);
}
```

## 二、EntityDataService 改造点

### create 方法改造

```java
@Transactional(rollbackFor = Exception.class)
public long create(long tenantId, String entityApiKey,
                   TenantData fixedData, Map<String, Object> extColumns) {

    // 1. 加载字段元数据
    List<EntityItem> allItems = metaLoader.loadItems(tenantId, entityApiKey);

    // 2. 默认值计算
    defaultValueEngine.compute(tenantId, entityApiKey,
        Collections.singletonList(extColumns), allItems);

    // 3. 前置公式计算
    Map<String, FormulaCompute> specialFormulas =
        computePipeline.computeForCreate(tenantId, entityApiKey,
            Collections.singletonList(extColumns), allItems);

    // 4. 持久化
    long id = routeService.executeWithRoute(tenantId, entityApiKey, () -> {
        dataDao.save(fixedData);
        if (!extColumns.isEmpty()) {
            UpdateWrapper<TenantData> uw = new UpdateWrapper<>();
            uw.eq("id", fixedData.getId());
            extColumns.forEach(uw::set);
            dataDao.update(null, uw);
        }
        routeService.incrementStats(tenantId, entityApiKey, 1);
        return fixedData.getId();
    });

    // 5. 特殊公式计算（入库后）
    if (!specialFormulas.isEmpty()) {
        Map<String, Object> savedData = new HashMap<>(extColumns);
        savedData.put("id", id);
        computePipeline.computeSpecialFormulas(tenantId, entityApiKey,
            Collections.singletonList(savedData), specialFormulas, allItems);
    }

    // 6. 发 MQ：触发父实体汇总
    asyncComputeProducer.sendAggregateFilter(tenantId, entityApiKey,
        Collections.singletonList(id));

    return id;
}
```

### update 方法改造

```java
@Transactional(rollbackFor = Exception.class)
public boolean update(long tenantId, String entityApiKey, long id,
                      TenantData fixedUpdates, Map<String, Object> extColumns) {

    // 1. 查旧数据
    Map<String, Object> oldData = getByIdWithColumns(tenantId, entityApiKey, id, null);

    // 2. 加载字段元数据
    List<EntityItem> allItems = metaLoader.loadItems(tenantId, entityApiKey);

    // 3. 确定变更字段
    Set<String> changedApiKeys = new HashSet<>(extColumns.keySet());

    // 4. 前置公式计算
    IncrementUpdateContext ctx = new IncrementUpdateContext();
    ComputeResult result = computePipeline.computeForIncrementUpdate(
        tenantId, entityApiKey,
        extColumns, oldData, changedApiKeys, allItems, ctx);

    // 5. 合并公式计算结果到 extColumns
    if (result != null && result.getFormulaResults() != null) {
        extColumns.putAll(result.getFormulaResults());
    }

    // 6. 持久化
    boolean updated = routeService.executeWithRoute(tenantId, entityApiKey, () -> {
        UpdateWrapper<TenantData> uw = new UpdateWrapper<>();
        uw.eq("id", id).eq("entity_api_key", entityApiKey);
        extColumns.forEach(uw::set);
        return dataDao.update(fixedUpdates, uw);
    });

    // 7. 特殊公式计算
    if (ctx.getSpecialFormulas() != null && !ctx.getSpecialFormulas().isEmpty()) {
        Map<String, Object> savedData = new HashMap<>(oldData);
        savedData.putAll(extColumns);
        computePipeline.computeSpecialFormulas(tenantId, entityApiKey,
            Collections.singletonList(savedData), ctx.getSpecialFormulas(), allItems);
    }

    // 8. 发 MQ（后置异步）
    if (result != null) {
        asyncComputeProducer.sendItemFilter(tenantId, entityApiKey,
            id, result.getChangedItemApiKeys());
        result.getAggregateConditions().forEach(cond ->
            asyncComputeProducer.sendAggregateParentFilter(tenantId, cond));
    }

    return updated;
}
```

## 三、缓存设计

### 公式元数据缓存

```
Key:   formula:meta:{tenantId}:{entityApiKey}
Value: { formulas: [...], params: [...] }
TTL:   30 分钟
失效:  FormulaCompute/FormulaComputeItem 写入时主动删除
```

### 字段元数据缓存

```
Key:   entity:items:{tenantId}:{entityApiKey}
Value: List<EntityItem>
TTL:   30 分钟
失效:  EntityItem 写入时主动删除
```

### 汇总规则缓存

```
Key:   aggregate:rules:{tenantId}:{entityApiKey}
Value: List<AggregationCompute>（含 AggregationComputeDetail）
TTL:   30 分钟
失效:  AggregationCompute 写入时主动删除
```

### 分片路由缓存（已有）

```
Key:   td_route:{tenantId}:{entityApiKey}
Value: tableIndex
TTL:   3600 秒
```

## 四、数据库设计要点

### 公式字段存储

公式字段的计算结果存储在分片表的 `dbc_` 扩展列中，与普通自定义字段共用同一列空间。

列分配规则（来自 EntityItem.dbColumn）：
- 数值型公式（REAL/PERCENTAGE）→ `dbc_real_{n}`
- 整数型公式（INTEGER）→ `dbc_int_{n}`
- 文本型公式（TEXT）→ `dbc_text_{n}`
- 日期型公式（DATE）→ `dbc_date_{n}`
- 布尔型公式（BOOL）→ `dbc_bool_{n}`

### 汇总字段存储

汇总字段同样存储在分片表的 `dbc_` 扩展列中，通过 `EntityItem.aggregateComputeType` 确定列类型。

```

```

## 六、监控与可观测性

### 计算日志

每次公式计算记录：
```
{
  tenantId, entityApiKey, dataId,
  formulaApiKey, expression,
  paramValues: { param → value },
  result, computeTime,
  success/error
}
```

### 关键指标

| 指标 | 说明 |
|---|---|
| formula.compute.duration | 公式计算耗时（按 entityApiKey 分组） |
| formula.param.resolve.duration | 参数取值耗时 |
| formula.batch.size | 批量计算数据量 |
| aggregate.compute.duration | 汇总计算耗时 |
| mq.send.failure | MQ 发送失败次数 |
| formula.error.count | 公式计算错误次数 |

## 七、迁移策略

### 老系统数据迁移

1. 将 XComputeFormula 迁移到 FormulaCompute（expression 字符串保持兼容）
2. 解析老系统表达式字符串，生成 FormulaComputeItem 记录（显式参数）
3. 将 AggregateRule 迁移到 AggregationCompute
4. 验证：对同一批数据，新老系统计算结果一致

### 灰度切换

1. 新系统写入时同时调用老系统计算接口（双写验证）
2. 对比结果，差异率低于阈值后切换
3. 切换后保留老系统计算接口一段时间，用于回滚
