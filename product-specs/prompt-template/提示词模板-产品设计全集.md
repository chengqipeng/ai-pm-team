## 1. 概述与业务场景

### 1.1 定位

提示词模板是 NeoAI Agent 平台的核心功能模块，负责提示词的配置管理、变量替换、查询视图数据注入、LLM调用执行，以及与Agent动作系统的集成。

### 1.2 核心业务场景

**意图识别**

模板示例：
```
我有一个文本和一个列表，帮我匹配一下哪个与这个描述更接近，找一个最接近的返回给我，匹配不到返回 null

文本:{content}
列表:{options}
```

**字段提取**
```
根据下面内容提取关键字段

关键字段:
{fields}

内容:
{content}

结果以 json 返回
```

### 1.3 引用场景

- Agent 意图识别：标准 Agent 预置调用某个提示词
- Agent Action：Action 执行逻辑可以调用一个提示词
- 平台能力调用：API、业务逻辑代码、流程事件、前端 UI 组件
- 自定义 Agent Action 执行提示词

## 2. 基础功能设计

### 2.1 导航菜单

**显示位置**：Admin → 功能设置 → NeoAI → 提示词模板设置 → 提示词模板 / 提示词结果样例

**权限控制**：
- License 控制：需授予【NeoAI】License +【提示词模板设置】Function
- 功能权限控制：需被授权【提示词模板设置】

### 2.2 列表功能

**显示字段**：模板名称、API名称、模板类型、是否自定义、模板描述、启用状态、创建人、最后修改人、最后修改时间、操作（编辑/设计/删除）

**排序字段（8.18版本）**：模板名称、API名称、是否自定义、启用状态、创建人、最后修改人、最后修改时间

**搜索字段（8.18版本）**：模板名称（模糊）、API名称（模糊）、是否自定义（下拉）、模板描述（模糊）、启用状态（下拉）、创建人/最后修改人（用户选择框）、最后修改时间（时间区间）

### 2.3 新建功能

1. 点击【新建提示词模板】按钮
2. 填写基本信息（模板名称、API名称、模板类型、模板描述、设置变量）
3. 点击下一步进入提示词模板设计器

**规则**：新建模板默认为禁用状态，调试通过后点击启用按钮启用。

### 2.4 复制功能

支持复制字段：模板名称、模板类型、模板描述、提示词指令、使用模型。复制后是否自定义=是。

### 2.5 编辑功能

- 可编辑：模板名称、模板描述
- 不可编辑：API名称、模板类型

### 2.6 删除功能

- 仅自定义提示词模板允许删除
- 启用状态不允许删除
- 被其他业务引用的模板不允许删除（包含被禁用的智能体版本引用）

### 2.7 启用/禁用功能

- 每个提示词模板最多有一个启用状态的版本
- 运行态运行该启用状态的版本，外部调用无需传版本号
- 被引用时必须存在一个启用的版本
- 启用状态不允许编辑
- 启用某个禁用版本时，原启用版本自动变为禁用
- 可启用小于当前版本号的版本

### 2.8 引用关系

- 列表提供查看引用关系按钮
- 可查看当前引用提示词模板的业务组件
- 被引用时不允许禁用、删除

## 3. 提示词模板类型体系

### 3.1 定位

提示词模版类型的本质是"基于'调用场景 + 使用主体 + 入参特性'的精细化分类体系"。

### 3.2 核心价值

- 明确适用边界：避免模板滥用
- 保障 AI 输出：标准化与精准性

### 3.3 子类型

| 类型 | 说明 |
|------|------|
| 通用类型 | 基础提示词模板 |
| 流程与代码 | 业务流程和代码调用场景 |
| 工作报告 | 工作报告生成 |
| 拜访总结 | 拜访总结生成（8.18版本新增） |

## 4. 设计器功能

### 4.1 提示词指令

- 在提示词指令区可编辑提示词内容
- 可通过插入参数框向规则中添加参数
- 参数名称规则：
  - 业务对象：`变量名.字段APIKey.字段APIKey`
  - 文本：`变量名`
  - 查询记录：`查询记录名称`
  - 当前用户：`当前用户.字段名`
- 业务对象类变量支持最多5层穿透
- 提示词指令需支持保留格式（换行、制表符等）
- 启用状态不可编辑提示词指令

### 4.2 预览区

- 显示填充测试数据后的内容
- 占位符替换的内容用颜色区分
- 可手动设置测试数据
- 测试时所有参数必填

### 4.3 生成结果

- 点击测试按钮后发送给大模型
- 显示大模型返回的结果
- 仅显示内容，不允许编辑

### 4.4 模型设置

- 支持选择模型：Deepseek R1、Deepseek V3、hunyuan
- 默认：Deepseek V3
- 含文件变量时仅显示多模态模型

## 5. 多版本管理

### 5.1 版本属性范围

**可编辑**（通过版本切换）：模板名称、模板描述、提示词指令、查询记录配置、模型

**不可编辑**：API名称、模板类型

### 5.2 创建逻辑

- 保存成功后自动生成版本1
- 系统标准模板自动初始化版本1
- 版本1不支持编辑（取代恢复默认设置功能）
- 版本1可以被禁用、复制

### 5.3 字段值逻辑

- 提示词模板名称：存在启用版本时与启用版本一致，否则与最新版本一致
- 最后修改人/时间：逻辑同上

### 5.4 复制功能

**复制新版本**：基于当前版本复制，版本号自动加1，系统记录最高版本号。

**复制新提示词模板**：复制后自动生成模板主记录和版本1，状态为禁用。

### 5.5 删除功能

- 启用状态的版本不允许删除
- 全部版本删除后，自动删除提示词模板
- 系统标准模板的版本1不允许删除

### 5.6 启用/禁用

- 每个提示词模板最多有一个启用状态的版本
- 启用某个禁用版本时，原启用版本自动变为禁用
- 禁用时需二次确认

## 6. 查询记录组件

### 6.1 支持多态字段

- 查询条件支持多态：多态字段显示格式 `{多态字段Label}.父对象`、`{多态字段Label}.父数据`
- 返回字段支持多态：返回结果格式 `{'from_object':'客户', 'from_data': '海康国际', ...}`

### 6.2 支持按业务类型筛选

- 运算符支持：等于、不等于
- 右值取值来源：值（手动设置）、变量（ID类型）
- 支持用户、部门字段

### 6.3 变量支持二级穿透

字段穿透最多穿透二级。

## 7. 插入变量支持

### 7.1 支持插入当前日期

- 插入变量窗口新增【系统函数】分类
- 点击系统函数显示 TODAY
- 插入后显示为：`${systemFunction___c.TODAY()}`
- 运行时自动转换为当前日期（yyyy-MM-dd）

### 7.2 占位符定义

| 前缀 | 含义 | 示例 |
|------|------|------|
| `variable___c` | 变量 | `${variable___c.text}` |
| `loginuser___c` | 当前用户 | `${loginuser___c.name}` |
| `searchdata___c` | 查询记录 | `${searchdata___c.apikey}` |
| `autoflow___c` | 自动流 | `${autoflow___c.apikey}` |
| `developercode___c` | 开发者代码 | `${developercode___c.apikey}` |
| `systemFunction___c` | 系统函数 | `${systemFunction___c.TODAY()}` |

## 9. SDK与业务逻辑代码集成

### 9.1 业务场景

1. 活动记录后自动提取客户意向产品和购买数量
2. 技术支持创建产品需求单后自动提取业务场景、痛点、需求
3. 工单关闭时自动判断客户满意度和延时情况

### 9.2 核心类

| 类名 | 说明 |
|------|------|
| XAIPromptTemplateService | 提示词模板服务类 |
| {TemplateAPIKey}__c | 模板模型类（系统自动生成） |
| {TemplateAPIKey}__c.{TemplateAPIKey}__cInput | 输入变量类 |
| XAIPromptTemplateResult<Output> | 调用响应类 |
| {TemplateAPIKey}__c.{TemplateAPIKey}__cOutput | 输出类 |

### 9.3 开发示例

```java
// 1. 创建提示词模板实例
LeadSummary__c template = new LeadSummary__c();

// 2. 设置输入变量
LeadSummary__c.LeadSummary__cInput variables = new LeadSummary__c.LeadSummary__cInput();
variables.setName("仁科互动");           // 文本类型变量
variables.setOwnerId(229844304534523L);  // 对象类型变量
template.setInput(variables);

// 3. 调用提示词模板
XAIPromptTemplateResult result = XAIPromptTemplateService.instance().generate(template);

// 4. 获取输出结果
LeadSummary__c.LeadSummary__cOutput output = result.getData();
String productName = output.getProductName();
Integer quantity = output.getQuantity();
```

### 9.4 适用扩展点

| 扩展点类型 | 执行方式 | 是否支持 | 说明 |
|-----------|---------|---------|------|
| Trigger | 同步 | 支持 | 15秒时限 |
| 流程脚本 | 异步 | 支持 | 可额外执行最多60秒 |
| 自定义API | 同步 | 支持 | 15秒时限 |
| 定时任务 | 异步 | 支持 | 15秒时限 |

### 9.5 Limits管控

| Limits项 | 默认值 |
|---------|--------|
| 入参大小 | 20000字符 |
| 出参大小 | 不限制 |
| 超时时长 | 与扩展点运行时长保持一致 |

## 10. 流程集成（自动流/工作流）

### 10.1 使用场景

在自动流或工作流中调用提示词模板，将AI能力嵌入业务流程。

### 10.2 前提条件

- 创建"流程与代码"类型的提示词模板，并完成调试和启用
- 创建自动流或工作流时，勾选【启用智能模式】

### 10.3 操作节点配置

| 配置项 | 说明 |
|--------|------|
| 操作类型 | 选择"提示词模板" |
| 模板名称 | 仅可选择类型=业务流程且状态=启用的模板 |
| 模板API名称 | 自动显示，不可编辑 |
| 变量映射 | 将流程数据映射到模板输入变量 |
| 输出映射 | 将模板输出结果映射到流程变量 |

### 10.4 引用关系

- 保存后自动记录流程与提示词模板引用关系
- 被引用的模板不允许禁用、删除
- 被引用的输出变量不允许删除

### 10.5 显示控制

仅当租户开启Agent平台（提示词模板管理）后显示提示词模板类型。

## 11. 多模态扩展

### 11.1 需求概述

扩展提示词模板变量系统，新增文件类型变量（FILE），支持将CRM对象图片/文件字段或用户上传文件传递给多模态大模型。

### 11.2 核心痛点

| 维度 | 现状 | 痛点 |
|------|------|------|
| 变量类型 | 仅支持文本、业务对象 | 无法将图片/文件字段传入大模型 |
| 设计器能力 | 插入变量仅支持文本/对象字段 | 无法引用对象的图片字段 |
| 运行时处理 | LLM调用仅构建纯文本消息 | 无法构建多模态请求格式 |

### 11.3 变量类型扩展

- 变量类型枚举新增 `file`，与现有 `text`、`object` 并列
- 文件类型变量传参格式为文件ID数组（`List<Long>`）
- 文件类型变量创建后不可编辑
- 计入变量总数限制

### 11.4 设计器插入文件变量

插入变量面板新增「文件」分组：
- **对象字段**：展示对象下的图片/附件字段
- **入参变量**：列出所有文件类型变量

含文件变量时，模型选择列表自动过滤为仅显示多模态模型。

### 11.5 运行时文件处理

| 步骤 | 处理逻辑 |
|------|---------|
| 1 | 识别FILE类型变量，提取文件ID列表 |
| 2 | 批量获取文件元数据（ext、fileSUrl） |
| 3a | 图片文件：直接使用签名URL |
| 3b | 文档文件：调用转PDF服务（P1） |
| 4 | 构建多模态消息（content数组） |

### 11.6 多模态消息格式

```json
{
  "role": "user",
  "content": [
    {"type": "text", "text": "<提示词指令文本>"},
    {"type": "image_url", "image_url": {"url": "<签名URL>", "detail": "high"}}
  ]
}
```

### 11.7 支持格式

- P0：图片（JPG/PNG/GIF/WEBP）
- P1：文档（Word/Excel/PPT → 自动转PDF）

### 11.8 安全设计

- 文件URL使用签名URL，有效期≥5分钟
- 加密存储的图片/附件字段不显示在可选列表中
- 文件类型白名单校验
- 单文件大小限制≤20MB，单次调用文件数量≤10个

### 11.9 SDK扩展

文件类型变量 → setter 接受 `List<Long>`（文件ID数组）

```java
variables.setPhotos(Arrays.asList(111L, 222L, 333L));
```

### 11.10 技术方案要点

- 元数据模型新增 `supportFile` 字段（ai_prompt_template_param）
- 新增 `ai_prompt_template_multimodal` 元数据模型
- 运行时消息结构支持文件数组
- 接入豆包（DouBao）多模态大模型
- 支持沙盒迁移

## 14. 自动流工作流事件支持文本域

### 14.1 需求背景

自动流调用提示词模板返回对象数组后，循环创建CRM数据时，右值字段选择器不支持文本域（TextArea）类型字段。

### 14.2 方案设计

- 右值字段选择器类型过滤白名单增加TextArea
- 适用于创建对象、更新对象、赋值三类事件节点
- 自动流和工作流行为保持一致
- 循环节点内外行为一致
- 固定值输入在左值为文本域时切换为多行文本输入框

### 14.3 核心场景

提示词模板返回对象数组 → 循环创建数据 → 文本域字段赋值（如产品描述等长文本内容）

## 15. 体验优化

### 15.1 插入变量交互优化

**痛点**：变量未插入在光标位置而是插入到设计器最底端
**方案**：插入变量时直接将变量插入到光标位置

### 15.2 提示词指令框禁止编辑变量

**痛点**：用户可在变量中间插入字符导致解析失败
**方案**：将变量设计为不可编辑的整体单元，显示背景颜色与指令内容区分

## 16. Limits管控

### 16.1 提示词模板Limits

| Limits项 | 默认值 | 版本 |
|---------|--------|------|
| 查询记录组件获取结果数量 | 50 | - |
| 单个模板可创建最大查询记录数量 | 10 | - |
| 查询记录条件数量 | 10 | 6.18 |
| 通用类型最大变量数量 | 10 | - |
| 通用类型最小变量数量 | 1 | - |
| 单个模板可创建插入占位符个数 | 20 | - |
| 预览时生成的结果字符数 | 2万 | 6.18 |
| 租户级可创建提示词模板数量 | 100 | 8.18 |
| 每个模板最多支持创建版本数 | 20 | 8.18 |

### 16.2 调用限制

| 模型 | Token限制 |

## 17. 支持字段类型

### 17.1 基础字段类型

文本、单选、多选、文本域、整数、实数、货币、日期、时间、自动编号、百分比、布尔型

### 17.2 关联字段类型

- 关联关系（支持5层穿透）
- 主子明细（支持5层穿透）
- 多态关联（可选对象、数据穿透）
- 多值关联

### 17.3 系统标准字段

ID、名称、业务类型、所属部门、所有人、创建人、创建日期、修改人、修改日期、锁定状态、审批状态、工作流阶段名称

### 17.4 查询条件类型匹配

| 字段类型 | 可匹配类型 |
|---------|-----------|
| 文本类 | 文本、单选、电话、邮箱、网址、自动编号 |
| 数值类 | 整数、实数、货币、百分比（互相匹配） |
| 日期时间类 | 只能匹配同类型 |
| 关联类 | 数值类（ID比较）、关联关系、主子明细、多态关联 |
