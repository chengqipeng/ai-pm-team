### 1.2 问题总结

- 19 项能力中，**9 项有前端管理入口，7 项缺失，3 项是内部能力不需要 UI**
- 缺失的 7 项分散在不同的业务场景中，没有统一的管理入口
- 现有的管理入口分散在 3 个不同的菜单页面中（SharingRulesView、PublicGroupsView、RoleAuthorizationView）
- 用户需要在多个页面间跳转才能完成一个完整的权限配置

## 二、统一配置管理方案

### 2.1 设计原则

1. **一个入口管理所有权限** — 不再分散在多个页面
2. **按实体组织** — 每个业务对象的权限配置集中在一起
3. **分层展示** — 从全局概览到单条数据溯源，逐层深入
4. **实时验证** — 配置变更后自动验证一致性

### 2.2 新菜单结构

```
数据安全（重构）
  ├── 权限总览          ← 新增：全局权限仪表盘
  ├── 对象权限配置       ← 合并：OWD + 共享规则 + 验证
  ├── 共享组管理         ← 保留
  ├── 区域管理          ← 新增：替代占位页面
  ├── 助理代理          ← 新增
  └── 权限诊断          ← 新增：溯源 + 验证 + 修复
```

## 五、权限配置流程（用户视角）

### 场景：为"商机"对象配置完整权限

```
1. 进入「权限总览」→ 看到 opportunity 状态为 ⚠
2. 点击 opportunity → 跳转「对象权限配置」
3. Tab 1 基础配置：
   - 默认访问 → 私有
   - 层级访问 → 只读（上级看下级）
   - 负责人 → 读写
   - 团队成员 → 只读
   - 共享规则 → 启用
4. Tab 2 共享规则：
   - 新建规则「大额商机共享给管理层」
   - 类型：基于条件
   - 条件：金额 > 100万
   - 目标：共享组「管理层」
   - 权限：只读
5. Tab 3 团队成员配置：
   - 启用团队成员 ✓
   - 默认权限：只读
6. Tab 4 验证：
   - 自动验证 → ✅ 全部通过
7. 回到「权限总览」→ opportunity 状态变为 ✅
```

### 场景：排查"为什么用户看不到某条数据"

```
1. 进入「权限诊断」
2. 选择用户：王磊
3. 选择对象：opportunity
4. 输入数据 ID：12345
5. 点击「查询」
6. 结果：❌ 无权访问
7. 原因链：空（无任何权限来源）
8. 建议：
   - 该数据负责人是张伟，王磊不是负责人
   - 王磊在华东销售组，数据在销售部，不是同部门
   - 无共享规则匹配
   - 不是团队成员
   → 建议：将王磊添加为该数据的团队成员，或创建共享规则
```
