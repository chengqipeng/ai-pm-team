# 权限全场景验证数据集设计

---

## 一、组织架构

```
鸿阳科技 (dept=1000, level=0)
├── 销售部 (dept=1001, level=1)
│   ├── 华东销售组 (dept=NEW_EAST, level=2)
│   └── 华南销售组 (dept=NEW_SOUTH, level=2)
├── 客服部 (dept=1002, level=1)
└── 研发部 (dept=1003, level=1)
```

## 二、用户分配

|:---|:---|:---|:---|:---|
| 张伟 | ...06 | 销售部(1001) | 管理员 | 李娜的经理（助理代理） |
| 李娜 | ...07 | 销售部(1001) | 普通 | 张伟的助理 |
| 王磊 | ...08 | 华东销售组 | 普通 | 区域「华东区」成员 |

## 三、权限配置

### 3.1 区域
- 华东区 (terr_east) — 成员：王磊

### 3.2 共享组
- pg_vip — 成员：张伟、王磊

### 3.3 助理关系
- 李娜 → 张伟（李娜是张伟的助理）

### 3.4 商机(opportunity) 数据权限
- defaultAccess = 0（私有）
- hierarchyAccess = 1（下级只读）
- ownerAccess = 2（读写）

### 3.5 共享规则
- 规则 A：基于负责人，来源=共享组 pg_vip，目标=共享组 pg_vip，只读
  → pg_vip 成员创建的数据共享给 pg_vip 全组

## 四、商机数据

| # | 名称 | 负责人 | 部门 | 预期可见用户 |
|:---:|:---|:---|:---|:---|
| D1 | 张伟-大客户A | 张伟 | 销售部 | 张伟(owner), 李娜(助理) |
| D2 | 张伟-大客户B | 张伟 | 销售部 | 张伟(owner), 李娜(助理), 王磊(共享规则→pg_vip) |
| D3 | 李娜-中型客户 | 李娜 | 销售部 | 李娜(owner), 张伟(hierarchy→下级无,同部门无因为private) |
| D4 | 王磊-华东客户 | 王磊 | 华东销售组 | 王磊(owner), 张伟(hierarchy), 李娜(助理继承张伟的hierarchy), 张伟+王磊(共享规则→pg_vip) |

注意：D2 中张伟是 pg_vip 成员，创建的数据触发共享规则 A → 共享给 pg_vip → 王磊也是 pg_vip 成员 → 王磊可见
