# 公共组（Public Group）权限详细设计方案

> 基于现有数据权限架构，补全公共组权限的读路径（查询过滤）和写路径（share 记录生成）

---

## 一、现状分析

### 1.1 已完成

| 组件 | 状态 | 说明 |
|---|---|---|
| `UserSubject.ofPublicGroup(pgApiKey)` | ✅ 已有 | 工厂方法，创建 type=2 的主体 |
| `DataShareQueryService.applySubjectCondition` | ✅ 已有 | 按 subject_type 分组查询，天然支持 type=2 |
| `PermissionCondition.applyTo` | ✅ 已有 | share 表查出的 visibleDataIds 已包含公共组授权的数据 |
| `p_data_share` 表结构 | ✅ 已有 | subject_type=2 对应公共组，share_cause=4 对应公共组来源 |
| `sharingRule` 元模型 | ✅ 已有 | toSubjectType=1(publicGroup) 支持共享给公共组 |

### 1.2 未完成

| 组件 | 状态 | 缺失内容 |
|---|---|---|
| `UserSubjectService.buildSubjects` | ❌ TODO | 公共组展开逻辑未实现 |
| 公共组成员关系查询 | ❌ 无 | 无服务可查"用户属于哪些公共组" |
| 共享规则引擎 | ❌ 无 | 无代码执行 sharingRule 匹配和 share 写入 |
| 权限写入（create/update/delete 联动） | ❌ 无 | EntityDataService CRUD 未触发 share 写入 |
| MQ 事件 | ❌ 无 | 数据变更/组织变更未发送权限事件 |

### 1.3 核心结论

**读路径（查询过滤）只差一步**：`UserSubjectService.buildSubjects` 中补上公共组展开，查询链路就能自动生效。

**写路径（share 记录生成）完全缺失**：需要实现共享规则引擎 + CRUD 联动 + MQ 事件。

## 三、读路径实现（查询过滤）

### 3.1 公共组成员关系存储

公共组成员关系存储在元数据体系中。公共组本身是一个 entity（`publicGroup`），成员关系通过关联表维护。

两种实现方案：

| 方案 | 存储 | 查询方式 | 优缺点 |
|---|---|---|---|
| A. 查业务数据表 | `p_tenant_data` 中 entity_api_key='publicGroupMember' | SQL 查询 | 与现有架构一致，但需要跨分片表查询 |
| B. 独立成员表 | `p_public_group_member` 独立表 | 简单 SQL | 查询快，但需要新建表和同步机制 |

**推荐方案 B**：独立成员表，查询性能好，且公共组成员关系是高频查询（每次权限判定都要用）。

```sql
-- 公共组成员表（在 public schema，不分片）
CREATE TABLE IF NOT EXISTS p_public_group_member (
    id                  BIGINT       NOT NULL,
    tenant_id           BIGINT       NOT NULL,
    group_api_key       VARCHAR(128) NOT NULL,   -- 公共组 apiKey
    user_id             BIGINT       NOT NULL,   -- 成员用户 ID
    user_api_key        VARCHAR(128),            -- 成员用户 apiKey
    role_type           SMALLINT     DEFAULT 0,  -- 0=普通成员, 1=管理员
    delete_flg          SMALLINT     NOT NULL DEFAULT 0,
    created_at          BIGINT,
    created_by          BIGINT,
    updated_at          BIGINT,
    updated_by          BIGINT,
    PRIMARY KEY (id)
);
CREATE INDEX idx_pgm_tid_uid ON p_public_group_member (tenant_id, user_id, delete_flg);
CREATE INDEX idx_pgm_tid_gak ON p_public_group_member (tenant_id, group_api_key, delete_flg);
CREATE UNIQUE INDEX uk_pgm_tid_gak_uid ON p_public_group_member (tenant_id, group_api_key, user_id);
```

## 四、写路径实现（share 记录生成）

公共组的 share 记录有两种来源：

| 来源 | share_cause | 触发时机 | subject_type |
|---|---|---|---|
| 共享规则自动匹配 | 5 (sharing_rule) | 数据创建/修改/规则变更 | 2 (public_group) |
| 手动共享给公共组 | 6 (manual_share) | 用户手动操作 | 2 (public_group) |

### 4.1 DataShareWriteService（新增）

```java
/**
 * share 记录写入服务。
 * 负责在数据 CRUD 时同步写入/更新/删除 share 记录。
 *
 * ⚠️ 必须在 DynamicTableNameHolder 上下文之外调用（与 buildCondition 同理），
 * 内部使用独立的 shareRouteService.executeWithRoute 路由到 share 分片表。
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class DataShareWriteService {

    private final ShareTableRouteService      shareRouteService;
    private final DataShareServiceImpl         shareDao;
    private final DataPermissionConfigService  configService;
    private final IdGenerator                  idGenerator;

    /**
     * 数据创建时初始化权限。
     * 同步写入 owner share + dept share。
     */
    public void initPermission(long tenantId, String entityApiKey,
                                long dataId, Long ownerId, Long departId, Long operatorId) {
        DataPermission config = configService.get(tenantId, entityApiKey);
        if (config == null) return; // 无权限配置，不写 share

        shareRouteService.executeWithRoute(tenantId, entityApiKey, () -> {
            long now = System.currentTimeMillis();

            // 1. 负责人 share (cause=0)
            if (ownerId != null) {
                DataShare ownerShare = new DataShare();
                ownerShare.setId(idGenerator.nextId());
                ownerShare.setTenantId(tenantId);
                ownerShare.setEntityApiKey(entityApiKey);
                ownerShare.setDataId(dataId);
                ownerShare.setSubjectApiKey(String.valueOf(ownerId));
                ownerShare.setSubjectType(UserSubject.SubjectType.USER);
                ownerShare.setAccessLevel(config.getOwnerAccess() != null ? config.getOwnerAccess() : 2);
                ownerShare.setShareCause(0); // OWNER
                ownerShare.setDeleteFlg(0);
                ownerShare.setCreatedAt(now);
                ownerShare.setCreatedBy(operatorId);
                shareDao.save(ownerShare);
            }

            // 2. 部门 share (cause=2)，仅 defaultAccess > 0 时
            if (departId != null && config.getDefaultAccess() != null && config.getDefaultAccess() > 0) {
                DataShare deptShare = new DataShare();
                deptShare.setId(idGenerator.nextId());
                deptShare.setTenantId(tenantId);
                deptShare.setEntityApiKey(entityApiKey);
                deptShare.setDataId(dataId);
                deptShare.setSubjectApiKey(String.valueOf(departId));
                deptShare.setSubjectType(UserSubject.SubjectType.DEPART);
                deptShare.setAccessLevel(config.getDefaultAccess());
                deptShare.setShareCause(2); // DEPARTMENT
                deptShare.setDeleteFlg(0);
                deptShare.setCreatedAt(now);
                deptShare.setCreatedBy(operatorId);
                shareDao.save(deptShare);
            }

            return null;
        });
    }

    /**
     * 手动共享给公共组。
     */
    public void shareToPublicGroup(long tenantId, String entityApiKey,
                                    long dataId, String groupApiKey,
                                    int accessLevel, Long operatorId) {
        shareRouteService.executeWithRoute(tenantId, entityApiKey, () -> {
            DataShare share = new DataShare();
            share.setId(idGenerator.nextId());
            share.setTenantId(tenantId);
            share.setEntityApiKey(entityApiKey);
            share.setDataId(dataId);
            share.setSubjectApiKey(groupApiKey);
            share.setSubjectType(UserSubject.SubjectType.PUBLIC_GROUP);
            share.setAccessLevel(accessLevel);
            share.setShareCause(6); // MANUAL_SHARE
            share.setDeleteFlg(0);
            share.setCreatedAt(System.currentTimeMillis());
            share.setCreatedBy(operatorId);
            shareDao.save(share);
            return null;
        });
    }

    /**
     * 共享规则匹配后写入 share（公共组作为目标主体）。
     */
    public void writeRuleShare(long tenantId, String entityApiKey,
                                long dataId, String targetGroupApiKey,
                                int accessLevel, String ruleApiKey, Long operatorId) {
        shareRouteService.executeWithRoute(tenantId, entityApiKey, () -> {
            // 先检查是否已存在（同一规则+同一数据+同一目标，不重复写入）
            QueryWrapper<DataShare> check = new QueryWrapper<>();
            check.eq("tenant_id", tenantId)
                 .eq("entity_api_key", entityApiKey)
                 .eq("data_id", dataId)
                 .eq("subject_api_key", targetGroupApiKey)
                 .eq("subject_type", UserSubject.SubjectType.PUBLIC_GROUP)
                 .eq("rule_api_key", ruleApiKey)
                 .eq("delete_flg", 0);
            if (shareDao.count(check) > 0) return null; // 已存在

            DataShare share = new DataShare();
            share.setId(idGenerator.nextId());
            share.setTenantId(tenantId);
            share.setEntityApiKey(entityApiKey);
            share.setDataId(dataId);
            share.setSubjectApiKey(targetGroupApiKey);
            share.setSubjectType(UserSubject.SubjectType.PUBLIC_GROUP);
            share.setAccessLevel(accessLevel);
            share.setShareCause(5); // SHARING_RULE
            share.setRuleApiKey(ruleApiKey);
            share.setDeleteFlg(0);
            share.setCreatedAt(System.currentTimeMillis());
            share.setCreatedBy(operatorId);
            shareDao.save(share);
            return null;
        });
    }

    /**
     * 数据删除时清理所有 share。
     */
    public void cleanOnDelete(long tenantId, String entityApiKey, long dataId) {
        shareRouteService.executeWithRoute(tenantId, entityApiKey, () -> {
            UpdateWrapper<DataShare> uw = new UpdateWrapper<>();
            uw.eq("tenant_id", tenantId)
              .eq("entity_api_key", entityApiKey)
              .eq("data_id", dataId)
              .set("delete_flg", 1)
              .set("updated_at", System.currentTimeMillis());
            shareDao.update(null, uw);
            return null;
        });
    }
}
```

## 五、共享规则引擎（公共组场景）

### 5.1 SharingRuleEngine（新增）

共享规则引擎负责：加载 sharingRule 元数据 → 匹配数据 → 写入 share 记录。

公共组相关的典型规则：
- "将华东区负责人的客户共享给华东销售公共组"（基于负责人 + 目标=公共组）
- "将 VIP 客户共享给 VIP 服务公共组"（基于条件 + 目标=公共组）

```java
/**
 * 共享规则引擎。
 * 加载 sharingRule 元数据，匹配数据，写入 share 记录。
 * 通过 MQ 异步触发，不阻塞业务数据 CRUD。
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class SharingRuleEngine {

    private final IMetadataMergeReadService mergeReadService;
    private final DataShareWriteService     shareWriteService;
    private final PublicGroupMemberService  pgMemberService;

    /**
     * 对单条数据执行所有激活的共享规则。
     */
    public void matchAndApply(long tenantId, String entityApiKey,
                               long dataId, Long ownerId, Long departId,
                               Map<String, Object> dataFields) {
        // 1. 加载该 entity 的所有激活共享规则
        List<SharingRule> rules = mergeReadService.listMergedByEntityApiKey(
                "sharingRule", entityApiKey);
        if (rules == null || rules.isEmpty()) return;

        for (SharingRule rule : rules) {
            if (rule.getActiveFlg() == null || rule.getActiveFlg() != 1) continue;
            if (rule.getEnableFlg() == null || rule.getEnableFlg() != 1) continue;

            try {
                boolean matched = matchRule(rule, tenantId, ownerId, departId, dataFields);
                if (!matched) continue;

                // 解析目标主体并写入 share
                applyRule(rule, tenantId, entityApiKey, dataId);

            } catch (Exception e) {
                log.error("[SharingRuleEngine] 规则执行失败: rule={}, data={}", rule.getApiKey(), dataId, e);
            }
        }
    }

    private boolean matchRule(SharingRule rule, long tenantId,
                               Long ownerId, Long departId,
                               Map<String, Object> dataFields) {
        if (rule.getShareType() == 0) {
            // 基于负责人：判断 ownerId 是否属于 fromSubject
            return matchOwnerSubject(rule, tenantId, ownerId);
        } else {
            // 基于条件：判断数据字段是否满足条件
            // TODO: 加载 sharingRuleCondition，按 criteriaLogic 组合判断
            return false;
        }
    }

    private boolean matchOwnerSubject(SharingRule rule, long tenantId, Long ownerId) {
        Integer fromType = rule.getFromSubjectType();
        String fromApiKey = rule.getFromSubjectApiKey();

        if (fromType == null || fromApiKey == null) return false;

        switch (fromType) {
            case 0: // 用户：ownerId 直接匹配
                return String.valueOf(ownerId).equals(fromApiKey);
            case 1: // 公共组：ownerId 是否是该公共组的成员
                List<Long> members = pgMemberService.getGroupMemberUserIds(tenantId, fromApiKey);
                return members.contains(ownerId);
            case 2: // 部门：ownerId 所在部门是否匹配
                // TODO: 需要用户服务查 ownerId 的部门
                return false;
            default:
                return false;
        }
    }

    private void applyRule(SharingRule rule, long tenantId, String entityApiKey, long dataId) {
        Integer toType = rule.getToSubjectType();
        String toApiKey = rule.getToSubjectApiKey();
        int accessLevel = rule.getAccessLevel() != null ? rule.getAccessLevel() : 1;

        if (toType == null || toApiKey == null) return;

        switch (toType) {
            case 0: // 共享给用户
                shareWriteService.writeRuleShare(tenantId, entityApiKey, dataId,
                        toApiKey, accessLevel, rule.getApiKey(), null);
                break;
            case 1: // 共享给公共组
                shareWriteService.writeRuleShare(tenantId, entityApiKey, dataId,
                        toApiKey, accessLevel, rule.getApiKey(), null);
                break;
            case 2: // 共享给部门
                shareWriteService.writeRuleShare(tenantId, entityApiKey, dataId,
                        toApiKey, accessLevel, rule.getApiKey(), null);
                break;
            // case 3: 部门及下级 → 展开部门树，为每个子部门写入
            // case 5: 区域
        }
    }
}
```

## 六、组织变更联动

### 6.1 公共组成员变更

| 变更类型 | 影响 | 处理 |
|---|---|---|
| 添加成员 | 新成员可看到公共组被共享的数据 | 清除该用户的 `user_subjects` 缓存 |
| 移除成员 | 该成员不再能看到公共组数据 | 清除该用户的 `user_subjects` 缓存 |
| 删除公共组 | 所有成员不再能看到该组数据 | 软删除 share 表中 subject_api_key=groupApiKey 的记录 + 清除所有成员缓存 |

```java
// 公共组成员变更时：
publicGroupMemberService.evictUserCache(tenantId, userId);
userSubjectService.evictCache(tenantId, userId);

// 公共组删除时：
shareRouteService.executeWithRoute(tenantId, entityApiKey, () -> {
    UpdateWrapper<DataShare> uw = new UpdateWrapper<>();
    uw.eq("tenant_id", tenantId)
      .eq("subject_api_key", groupApiKey)
      .eq("subject_type", UserSubject.SubjectType.PUBLIC_GROUP)
      .set("delete_flg", 1);
    shareDao.update(null, uw);
    return null;
});
// 清除所有成员的缓存
List<Long> memberIds = pgMemberService.getGroupMemberUserIds(tenantId, groupApiKey);
memberIds.forEach(uid -> userSubjectService.evictCache(tenantId, uid));
```

## 八、与部门/区域权限的关系

本方案聚焦公共组，但架构设计是通用的。部门和区域权限的实现模式完全一致：

| 权限类型 | subject_type | 成员查询服务 | UserSubject 工厂方法 | 差异点 |
|---|---|---|---|---|
| 公共组 | 2 | PublicGroupMemberService | ofPublicGroup(apiKey) | 成员关系扁平 |
| 部门 | 1 | DepartmentService（待接入） | ofDepart(id, apiKey) | 有层级（hierarchyAccess） |
| 区域 | 3 | TerritoryService（待接入） | ofTerritory(apiKey) | 有层级 |

部门和区域的实现只需：
1. 在 `UserSubjectService.buildSubjects` 中调用对应服务展开主体
2. 共享规则引擎中处理对应的 fromSubjectType/toSubjectType

查询链路（`DataShareQueryService`、`PermissionCondition`）已经天然支持所有 subject_type，无需修改。
