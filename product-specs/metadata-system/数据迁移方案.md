## 全局迁移：表名标准化

```sql
-- p_meta_model.db_table 更新
UPDATE p_meta_model SET db_table = 'p_meta_metamodel_data'
WHERE api_key IN ('entity', 'item', 'entityLink', 'checkRule', 'pickOption', 'referenceFilter');

-- 全局选项集表重命名
RENAME TABLE x_global_pickitem TO p_tenant_global_pickitem;
```

## 执行顺序

```
Step 1: 备份数据
Step 2: 表名标准化（上方 SQL）
Step 3: 按元模型执行迁移（顺序无关，可并行）
  - entity:          migration/entity.md
  - item:            migration/item.md
  - entityLink:      migration/entity-link.md
  - checkRule:       migration/check-rule.md
  - pickOption:      migration/pick-option.md
  - referenceFilter: migration/reference-filter.md
Step 4: 全量验证
```

## 全量验证清单

| 检查项 | SQL |
|:---|:---|
| 无 enable*/is* 前缀残留 | `SELECT * FROM p_meta_item WHERE api_key LIKE 'enable%' OR api_key LIKE 'is%'` |
| 所有 *Flg 字段 item_type=31 | `SELECT * FROM p_meta_item WHERE api_key LIKE '%Flg' AND item_type != 31` |
| entity api_key 无 snake_case | `SELECT * FROM p_meta_item WHERE metamodel_api_key='entity' AND api_key LIKE '%\_%'` |
| item_type 无老编码残留 | `SELECT DISTINCT dbc_int1 FROM p_common_metadata WHERE metamodel_api_key='item' AND dbc_int1 IN (23,24,29,32,34,38,39,40,41)` |
| db_column 格式统一 dbc_xxxN | `SELECT * FROM p_meta_item WHERE db_column LIKE 'dbc\_%\_%' AND metamodel_api_key != 'entityLink'` |
| globalPickItem 无纯数字 ID | `SELECT * FROM p_common_metadata WHERE metamodel_api_key='item' AND dbc_varchar10 REGEXP '^[0-9]+$'` |
