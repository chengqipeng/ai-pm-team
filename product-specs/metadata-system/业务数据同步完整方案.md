## 二、核心挑战与解决策略

| # | 挑战 | 影响 | 解决策略 |
|:---:|:---|:---|:---|
| 1 | 列前缀合并：老 9 种 → 新 6 种 | 列号全部需要重新映射 | 通过 item 元数据的 dbColumn 字段驱动，老列→新列一一对应 |
| 2 | 列名格式：`dbc_varchar_1` → `dbc_varchar1` | 所有列名去下划线 | 元数据层已统一，同步时按新 dbColumn 写入 |
| 3 | SELECT 值转换：数字编码 → apiKey 字符串 | 老存 `3`，新存 `it_industry` | 构建 pickOption code→apiKey 映射表 |
| 4 | MULTISELECT 格式：逗号分隔 → VARCHAR(300)[] 数组 | 存储格式变更 | 拆分逗号字符串，逐个转换后组装数组 |
| 5 | 固定列变更：`entity_type`(Long) → `busitype_api_key`(String) | 业务类型标识方式变更 | 构建 busiType ID→apiKey 映射表 |
| 6 | 分表路由 | 新系统 2000 张分表 | 同步前为每个 (tenant_id, entity_api_key) 分配路由 |
| 7 | 数据量巨大 | 单租户核心对象百万级 | 按租户分批，每批 5000 行，断点续传 |

## 三、Phase 0 — 元数据验证（同步前必做）

业务数据的列映射完全依赖 item 元数据的 `dbColumn` 字段，必须先验证元数据描述的正确性。

### 3.1 验证清单（9 项）

#### V1: entity 存在性

```sql
-- 预期：7 条全部存在
SELECT api_key FROM p_common_metadata
WHERE metamodel_api_key = 'entity'
  AND api_key IN ('account','lead','opportunity','product','salesOrder','contact','salesOrderItem')
  AND delete_flg = 0;
```

**判定**：缺失任何一个 → 阻断，必须先补充 entity 元数据。

#### V2: item 完整性（新老数量比对）

对每个核心 entity，比对新库 item 数量与老库表的实际 dbc 列数：

```sql
-- 新库 item 数量
SELECT entity_api_key, COUNT(*) AS item_count
FROM p_common_metadata
WHERE metamodel_api_key = 'item'
  AND entity_api_key IN ('account','lead','opportunity','product','salesOrder','contact','salesOrderItem')
  AND delete_flg = 0
GROUP BY entity_api_key;

-- 老库 dbc 列数（对每个 entity 执行）
SELECT COUNT(DISTINCT column_name)
FROM information_schema.columns
WHERE table_name = 'a_account'
  AND column_name LIKE 'dbc\_%';
```

**判定**：差异超过 20% → 警告，需逐字段排查是否有遗漏或多余。

#### V3: item.dbColumn 非空（非虚拟字段）

```sql
-- 预期：0 条
SELECT entity_api_key, api_key, dbc_int1 AS itemType
FROM p_common_metadata
WHERE metamodel_api_key = 'item'
  AND dbc_int1 NOT IN (6, 7, 21, 27)  -- 排除 FORMULA/ROLLUP/JOIN/COMPUTED
  AND (dbc_varchar3 IS NULL OR dbc_varchar3 = '')
  AND delete_flg = 0
  AND entity_api_key IN ('account','lead','opportunity','product','salesOrder','contact','salesOrderItem');
```

**判定**：存在记录 → 阻断，dbColumn 为空的非虚拟字段无法写入业务数据。

#### V4: item.dbColumn 格式正确

```sql
-- 预期：0 条（新系统不应有下划线分隔数字的格式）
SELECT entity_api_key, api_key, dbc_varchar3 AS dbColumn
FROM p_common_metadata
WHERE metamodel_api_key = 'item'
  AND dbc_varchar3 REGEXP 'dbc_[a-z]+_[0-9]'
  AND delete_flg = 0
  AND entity_api_key IN ('account','lead','opportunity','product','salesOrder','contact','salesOrderItem');
```

**判定**：存在记录 → 警告，需确认是否已在 Converter 中兼容处理。

#### V5: itemType 全部为新编码

```sql
-- 预期：无老编码（23,24,29,32,34,38,39,40,41,99）
```

**判定**：存在重复 → 阻断，同一 entity 内 dbColumn 重复会导致数据覆盖。

**判定**：SELECT 字段无 pickOption → 警告（可能是全局选项集或前端硬编码）。

### 3.2 元数据问题发现与修复矩阵

基于验证结果，按优先级分类处理：

| 优先级 | 问题类型 | 影响范围 | 修复方式 | 阻断同步 |
|:---:|:---|:---|:---|:---:|
| P0 | entity 缺失 | 整个对象无法同步 | 补充 entity 元数据到 p_common_metadata | 是 |
| P0 | item.dbColumn 为空（非虚拟字段） | 该字段数据丢失 | 按 itemType→dbColumnPrefix 规则分配列号 | 是 |
| P0 | itemType 含老编码 | 列前缀映射错误 | 执行编码转换 SQL（见数据迁移方案 §item） | 是 |
| P0 | dbColumn 重复 | 数据覆盖 | 重新分配冲突的列号 | 是 |
| P0 | dbColumn 前缀与 itemType 不一致 | 数据写入错误列类型 | 修正 dbColumn 或 itemType | 是 |
| P1 | SELECT 字段无 pickOption | 值转换失败，存原始数字 | 从老库补充 pickOption 或确认全局选项集 | 否（降级处理） |
| P1 | busiType 缺失 | busitype_api_key 为空 | 从老库补充 busiType 元数据 | 否（降级处理） |
| P2 | item 数量差异 >20% | 部分字段数据丢失 | 逐字段排查，补充缺失 item | 否 |
| P2 | dbColumn 含下划线格式 | Converter 需兼容 | 确认代码已处理或修正 dbColumn | 否 |

### 3.3 验证脚本

```python
#!/usr/bin/env python3
"""
verify_metadata.py — 业务数据同步前元数据验证
运行方式: python verify_metadata.py
输出: 验证报告（PASS/WARN/FAIL）
"""

import pymysql
import psycopg2
import sys

CORE_ENTITIES = ['account', 'lead', 'opportunity', 'product',
                 'salesOrder', 'contact', 'salesOrderItem']

OLD_ITEM_TYPE_CODES = {23, 24, 29, 32, 34, 38, 39, 40, 41, 99}

# 新系统 itemType → 业务数据大宽表 dbColumnPrefix
BIZ_DATA_PREFIX = {
    1: 'dbc_varchar',   2: 'dbc_bigint',    3: 'dbc_bigint',
    4: 'dbc_varchar',   5: 'dbc_bigint',    8: 'dbc_textarea',
    9: 'dbc_smallint',  10: 'dbc_decimal',  11: 'dbc_decimal',
    12: 'dbc_varchar',  13: 'dbc_varchar',  14: 'dbc_varchar',
    15: 'dbc_bigint',   16: 'dbc_array',    17: 'dbc_bigint',
    18: 'dbc_varchar',  19: 'dbc_varchar',  20: 'dbc_varchar',
    22: 'dbc_varchar',
}

VIRTUAL_TYPES = {6, 7, 21, 27}  # FORMULA/ROLLUP/JOIN/COMPUTED

class Verifier:
    def __init__(self, mysql_cfg, pg_cfg):
        self.new_db = pymysql.connect(**mysql_cfg)
        self.old_db = psycopg2.connect(**pg_cfg) if pg_cfg else None
        self.errors, self.warnings = [], []

    def run(self):
        checks = [
            self._v1_entity_existence,
            self._v2_item_completeness,
            self._v3_dbcolumn_not_null,
            self._v4_dbcolumn_format,
            self._v5_itemtype_codes,
            self._v6_dbcolumn_unique,
            self._v7_pickoption,
            self._v8_busitype,
            self._v9_prefix_consistency,
        ]
        for fn in checks:
            fn()

        print(f"\n{'='*60}")
        print(f"验证完成: {len(self.errors)} FAIL, {len(self.warnings)} WARN")
        if self.errors:
            print("\n❌ 阻断项（必须修复）:")
            for e in self.errors: print(f"  - {e}")
        if self.warnings:
            print("\n⚠️ 警告项（建议修复）:")
            for w in self.warnings: print(f"  - {w}")
        return len(self.errors) == 0

    def _q(self, sql, params=None):
        cur = self.new_db.cursor()
        cur.execute(sql, params)
        return cur.fetchall()

    def _v1_entity_existence(self):
        rows = self._q("""
            SELECT api_key FROM p_common_metadata
            WHERE metamodel_api_key='entity' AND api_key IN %s AND delete_flg=0
        """, (tuple(CORE_ENTITIES),))
        found = {r[0] for r in rows}
        missing = set(CORE_ENTITIES) - found
        if missing:
            self.errors.append(f"[V1] entity 缺失: {missing}")
        else:
            print(f"[V1] ✅ {len(found)} 个核心 entity 全部存在")

    def _v2_item_completeness(self):
        for ent in CORE_ENTITIES:
            rows = self._q("""
                SELECT COUNT(*) FROM p_common_metadata
                WHERE metamodel_api_key='item' AND entity_api_key=%s AND delete_flg=0
            """, (ent,))
            cnt = rows[0][0]
            print(f"[V2] {ent}: {cnt} 个 item")
            if cnt == 0:
                self.errors.append(f"[V2] {ent} 无任何 item 定义")

    def _v3_dbcolumn_not_null(self):
        rows = self._q("""
            SELECT entity_api_key, api_key, dbc_int1
            FROM p_common_metadata
            WHERE metamodel_api_key='item'
              AND dbc_int1 NOT IN (6,7,21,27)
              AND (dbc_varchar3 IS NULL OR dbc_varchar3='')
              AND delete_flg=0
              AND entity_api_key IN %s
        """, (tuple(CORE_ENTITIES),))
        if rows:
            self.errors.append(f"[V3] {len(rows)} 个非虚拟字段 dbColumn 为空: {rows[:3]}")
        else:
            print(f"[V3] ✅ 非虚拟字段 dbColumn 全部非空")

    def _v4_dbcolumn_format(self):
        rows = self._q("""
            SELECT entity_api_key, api_key, dbc_varchar3
            FROM p_common_metadata
            WHERE metamodel_api_key='item'
              AND dbc_varchar3 REGEXP 'dbc_[a-z]+_[0-9]'
              AND delete_flg=0 AND entity_api_key IN %s
        """, (tuple(CORE_ENTITIES),))
        if rows:
            self.warnings.append(f"[V4] {len(rows)} 个 dbColumn 含下划线格式: {rows[:3]}")
        else:
            print(f"[V4] ✅ dbColumn 格式全部正确")

    def _v5_itemtype_codes(self):
        rows = self._q("""
            SELECT DISTINCT dbc_int1, COUNT(*) FROM p_common_metadata
            WHERE metamodel_api_key='item' AND entity_api_key IN %s AND delete_flg=0
            GROUP BY dbc_int1
        """, (tuple(CORE_ENTITIES),))
        old = [(t, c) for t, c in rows if t in OLD_ITEM_TYPE_CODES]
        if old:
            self.errors.append(f"[V5] itemType 含老编码: {old}")
        else:
            print(f"[V5] ✅ 所有 itemType 均为新编码")

    def _v6_dbcolumn_unique(self):
        rows = self._q("""
            SELECT entity_api_key, dbc_varchar3, COUNT(*) AS cnt
            FROM p_common_metadata
            WHERE metamodel_api_key='item' AND dbc_varchar3 IS NOT NULL
              AND delete_flg=0 AND entity_api_key IN %s
            GROUP BY entity_api_key, dbc_varchar3 HAVING cnt>1
        """, (tuple(CORE_ENTITIES),))
        if rows:
            self.errors.append(f"[V6] dbColumn 重复: {rows[:5]}")
        else:
            print(f"[V6] ✅ dbColumn 唯一性通过")

    def _v7_pickoption(self):
        for ent in CORE_ENTITIES:
            sel = self._q("""
                SELECT api_key FROM p_common_metadata
                WHERE metamodel_api_key='item' AND entity_api_key=%s
                  AND dbc_int1 IN (4,16) AND delete_flg=0
            """, (ent,))
            if not sel: continue
            opts = self._q("""
                SELECT DISTINCT parent_metadata_api_key FROM p_common_metadata
                WHERE metamodel_api_key='pickOption' AND entity_api_key=%s AND delete_flg=0
            """, (ent,))
            opt_set = {r[0] for r in opts}
            missing = [r[0] for r in sel if r[0] not in opt_set]
            if missing:
                self.warnings.append(f"[V7] {ent}: {len(missing)} 个 SELECT 字段无 pickOption: {missing[:3]}")
            else:
                print(f"[V7] ✅ {ent}: {len(sel)} 个 SELECT 字段全部有 pickOption")

    def _v8_busitype(self):
        rows = self._q("""
            SELECT entity_api_key, COUNT(*) FROM p_common_metadata
            WHERE metamodel_api_key='busiType' AND entity_api_key IN %s AND delete_flg=0
            GROUP BY entity_api_key
        """, (tuple(CORE_ENTITIES),))
        bt = {r[0]: r[1] for r in rows}
        for ent in CORE_ENTITIES:
            if ent not in bt:
                self.warnings.append(f"[V8] {ent}: 无 busiType")
            else:
                print(f"[V8] ✅ {ent}: {bt[ent]} 个 busiType")

    def _v9_prefix_consistency(self):
        rows = self._q("""
            SELECT entity_api_key, api_key, dbc_int1, dbc_varchar3
            FROM p_common_metadata
            WHERE metamodel_api_key='item' AND dbc_varchar3 IS NOT NULL
              AND delete_flg=0 AND entity_api_key IN %s
        """, (tuple(CORE_ENTITIES),))
        bad = []
        for ent, item, itype, dcol in rows:
            exp = BIZ_DATA_PREFIX.get(itype)
            if exp is None: continue
            prefix = ''.join(c for c in dcol if not c.isdigit()).rstrip('_')
            if prefix != exp:
                bad.append((ent, item, itype, dcol, exp))
        if bad:
            self.errors.append(f"[V9] dbColumn 前缀不一致: {len(bad)} 条, 示例: {bad[:3]}")
        else:
            print(f"[V9] ✅ dbColumn 前缀与 itemType 全部一致")
```

## 四、Phase 1 — 映射表构建

验证通过后，构建同步所需的 4 张映射表。

### 4.1 列映射表（老列→新列）

**构建逻辑**：对每个 entity，从 item 元数据中提取老列→新列的对应关系。

关键点：item 元数据中的 `dbColumn`（dbc_varchar3）存的是**业务数据大宽表**中的新列名，而老列名需要从老系统的 item 定义中反查。

```
构建列映射(entityApiKey):
    │
    ├─ 1. 从新库查询该 entity 的所有 item
    │     SELECT api_key, dbc_varchar3 AS newDbColumn, dbc_int1 AS itemType
    │     FROM p_common_metadata
    │     WHERE metamodel_api_key='item' AND entity_api_key=? AND delete_flg=0
    │
    ├─ 2. 从老库查询该 entity 的 item 定义（老 metadata_json 中的 dbColumn）
    │     老系统 item 的 dbColumn 格式: dbc_varchar_1, dbc_select_3, dbc_integer_5 等
    │
    ├─ 3. 按 item.apiKey 关联，生成映射:
    │     { item.apiKey → (oldDbColumn, newDbColumn, itemType) }
    │
    └─ 4. 缓存到 column_mapping 字典
```

**老→新列前缀映射规则**（业务数据大宽表层面）：

| 老前缀 | 老列号范围 | 新前缀 | 新列号范围 | 值转换 |
|:---|:---|:---|:---|:---|
| dbc_varchar_ | 1~145 | dbc_varchar | 1~145 | 无 |
| dbc_svarchar_ | 1~25 | dbc_varchar | 146~170 | 无 |
| dbc_select_ | 1~80 | dbc_varchar | 171~250 | 数字编码→apiKey |

### 5.3 行转换逻辑（核心）

```python
def transform_row(entity_api_key, old_row):
    """将老库一行数据转换为新库格式"""
    new_row = {}

    # ---- 固定列转换 ----
    new_row['id'] = old_row['id']
    new_row['tenant_id'] = old_row['tenant_id']
    new_row['entity_api_key'] = entity_api_key
    new_row['name'] = old_row.get(f'{entity_api_key}_name') or old_row.get('name')
    new_row['owner_id'] = old_row.get('owner_id')
    new_row['depart_id'] = old_row.get('dim_depart')
    new_row['delete_flg'] = old_row.get('del_flg', 0)
    new_row['created_at'] = old_row.get('created_at')
    new_row['created_by'] = old_row.get('created_by')
    new_row['updated_at'] = old_row.get('updated_at')
    new_row['updated_by'] = old_row.get('updated_by')
    new_row['lock_status'] = old_row.get('lock_status', 1)
    new_row['approval_status'] = old_row.get('approval_status')
    new_row['applicant_id'] = old_row.get('applicant_id')

    # ---- busitype 转换: entity_type(Long) → busitype_api_key(String) ----
    old_entity_type = old_row.get('entity_type')
    if old_entity_type and old_entity_type in busitype_map[entity_api_key]:
        new_row['busitype_api_key'] = busitype_map[entity_api_key][old_entity_type]
    else:
        new_row['busitype_api_key'] = get_default_busitype(entity_api_key)
        log_unmapped('busitype', entity_api_key, old_row['id'], old_entity_type)

    # ---- 扩展列转换 ----
    for item_apikey, (old_col, new_col, item_type) in column_map[entity_api_key].items():
        old_value = old_row.get(old_col)
        if old_value is None:
            continue

        new_value = convert_value(entity_api_key, item_apikey, item_type, old_value)
        new_row[new_col] = new_value

    return new_row


def convert_value(entity_api_key, item_apikey, item_type, old_value):
    """根据 itemType 执行值转换"""

    # SELECT(4): 数字编码 → apiKey 字符串
    if item_type == 4:
        code = int(old_value) if old_value else None
        mapping = select_value_map.get(entity_api_key, {}).get(item_apikey, {})
        if code in mapping:
            return mapping[code]
        else:
            log_unmapped('select', entity_api_key, item_apikey, old_value)
            return str(old_value)  # fallback: 保留原始值的字符串形式

    # MULTI_SELECT(16): 逗号分隔数字 → VARCHAR(300)[] 数组
    if item_type == 16:
        codes = str(old_value).split(',')
        mapping = select_value_map.get(entity_api_key, {}).get(item_apikey, {})
        api_keys = []
        for code_str in codes:
            code = int(code_str.strip()) if code_str.strip() else None
            if code and code in mapping:
                api_keys.append(mapping[code])
            elif code:
                api_keys.append(str(code))
                log_unmapped('multiselect', entity_api_key, item_apikey, code)
        return json.dumps(api_keys)  # '["opt1","opt2"]'

    # BOOLEAN(9): 统一为 0/1
    if item_type == 9:
        if old_value in (True, 1, '1', 'true'): return 1
        return 0

    # CURRENCY(10) / PERCENT(11): 保留 4 位小数
    if item_type in (10, 11):
        from decimal import Decimal
        return Decimal(str(old_value)).quantize(Decimal('0.0001'))

    # DATE(3) / DATETIME(15): 确保毫秒时间戳
    if item_type in (3, 15):
        ts = int(old_value) if old_value else None
        if ts and ts < 10000000000:  # 秒级时间戳
            ts = ts * 1000
        return ts

    # TEXT(1)/EMAIL(12)/PHONE(13)/URL(14)/IMAGE(19)/GEO(18)/AUDIO(22)/AUTONUMBER(20):
    # 直接字符串存储
    if item_type in (1, 12, 13, 14, 18, 19, 20, 22):
        return str(old_value)[:300]  # VARCHAR(300) 截断

    # TEXTAREA(8): 直接存储
    if item_type == 8:
        return str(old_value)

    # NUMBER(2) / RELATION_SHIP(5) / MASTER_DETAIL(17): BIGINT
    if item_type in (2, 5, 17):
        return int(old_value) if old_value else None

    # 其他: 原样存储
    return old_value
```

```python
def get_or_create_route(tenant_id, entity_api_key):
    """获取或创建分表路由"""
    # 1. 查缓存
    route = route_cache.get((tenant_id, entity_api_key))
    if route: return route

    # 2. 查配置表
    cur.execute("""
        SELECT table_index FROM p_tenant_data_route
        WHERE tenant_id=%s AND entity_api_key=%s
    """, (tenant_id, entity_api_key))
    row = cur.fetchone()
    if row:
        route_cache[(tenant_id, entity_api_key)] = row[0]
        return row[0]

    # 3. Least-Loaded 分配
    cur.execute("""
        SELECT table_index FROM p_tenant_data_stats
        ORDER BY row_count ASC LIMIT 10
    """)
    candidates = [r[0] for r in cur.fetchall()]
    table_index = random.choice(candidates)

    cur.execute("""
        INSERT INTO p_tenant_data_route (id, tenant_id, entity_api_key, table_index, created_at)
        VALUES (%s, %s, %s, %s, %s)
    """, (snowflake_id(), tenant_id, entity_api_key, table_index, now_ms()))

    route_cache[(tenant_id, entity_api_key)] = table_index
    return table_index
```

### 5.6 批量写入

```python
def batch_insert(target_table, rows, batch_size=500):
    """批量 INSERT，每 500 行一批"""
    if not rows: return

    columns = list(rows[0].keys())
    placeholders = ', '.join(['%s'] * len(columns))
    col_str = ', '.join(columns)

    sql = f"INSERT INTO {target_table} ({col_str}) VALUES ({placeholders})"

    for i in range(0, len(rows), batch_size):
        batch = rows[i:i+batch_size]
        values = [tuple(row[c] for c in columns) for row in batch]
        cur.executemany(sql, values)
        conn.commit()

        # 更新统计
        cur.execute("""
            UPDATE p_tenant_data_stats
            SET row_count = row_count + %s, updated_at = %s
            WHERE table_index = %s
        """, (len(batch), now_ms(), get_table_index(target_table)))
```

```python
def values_equal(old_val, new_val, item_type):
    """考虑类型转换后的值比较"""
    if old_val is None and new_val is None: return True
    if old_val is None or new_val is None: return False

    # SELECT: 老数字 vs 新 apiKey
    if item_type == 4:
        expected = select_value_map.get(entity, {}).get(item, {}).get(int(old_val))
        return str(new_val) == str(expected) if expected else str(new_val) == str(old_val)

    # BOOLEAN: 统一比较
    if item_type == 9:
        return (old_val in (1, True, '1')) == (new_val in (1, True, '1'))

    # CURRENCY/PERCENT: 精度容差
    if item_type in (10, 11):
        return abs(float(old_val) - float(new_val)) < 0.001

    # 其他: 字符串比较
    return str(old_val) == str(new_val)
```

## 十一、新老数据转换详细逻辑

本章逐类型说明每种数据从老库到新库的完整转换规则，确保转换后数据与老系统完全一致。

### 11.1 转换总览

```
老库一行业务数据（a_account 等）
  │
  ├─ 固定列转换（15 列，规则固定）
  │     id → id（不变）
  │     entity_type(Long) → busitype_api_key(String)  ← 查映射表
  │     dim_depart → depart_id（改名）
  │     {entity}_name → name（统一列名）
  │     del_flg → delete_flg（改名）
  │     + 新增 entity_api_key 列
  │
  ├─ 扩展列转换（按 itemType 分类处理）
  │     ├─ TEXT/EMAIL/PHONE/URL/IMAGE/GEO/AUDIO/AUTONUMBER → 字符串直接迁移
  │     ├─ NUMBER/DATE/DATETIME/RELATION_SHIP/MASTER_DETAIL → BIGINT 直接迁移
  │     ├─ CURRENCY/PERCENT → DECIMAL 直接迁移
  │     ├─ BOOLEAN → SMALLINT 直接迁移
  │     ├─ TEXTAREA → TEXT 直接迁移
  │     ├─ SELECT → 数字编码→apiKey 字符串（查 pickOption 映射表）
  │     ├─ MULTI_SELECT → 逗号分隔→VARCHAR(300)[] 数组（逐个转换后组装）
  │     └─ FORMULA/ROLLUP/COMPUTED/JOIN → 不迁移值，同步后重算
  │
  └─ 列号重映射
        老 dbc_varchar_3 → 新 dbc_varchar{N}（N 由 item 元数据 dbColumn 决定）
        老 dbc_select_5 → 新 dbc_varchar{M}（前缀变更 + 列号变更 + 值转换）
        老 dbc_integer_2 → 新 dbc_bigint{K}（前缀变更 + 列号变更）
```

#### 11.2.2 SELECT(4) — 核心转换

| 维度 | 老系统 | 新系统 | 转换 |
|:---|:---|:---|:---|
| 存储类型 | INTEGER (dbc_select_) | VARCHAR(300) (dbc_varchar) | 类型变更 |
| 值格式 | 数字编码（如 `3`） | apiKey 字符串（如 `it_industry`） | 值转换 |
| 空值 | NULL 或 0 | NULL | 0→NULL |

```python
def convert_select(entity_api_key, item_api_key, old_value):
    """SELECT 字段值转换：数字编码 → pickOption.apiKey"""
    if old_value is None or old_value == 0:
        return None

    code = int(old_value)
    mapping = pickoption_code_map.get((entity_api_key, item_api_key), {})

    if code in mapping:
        return mapping[code]  # 返回 apiKey 字符串
    else:
        # 降级：检查是否为全局选项集
        global_mapping = global_pickoption_map.get(item_api_key, {})
        if code in global_mapping:
            return global_mapping[code]
        else:
            log_unmapped('SELECT', entity_api_key, item_api_key, code)
            return str(code)  # fallback: 保留数字的字符串形式
```

**pickOption 映射表构建**：

```sql
-- 从新库构建 (entity_api_key, item_api_key, option_code) → option_api_key
SELECT
    po.entity_api_key,
    po.parent_metadata_api_key AS item_api_key,
    po.dbc_int1 AS option_code,       -- pickOption 的 optionCode
    po.api_key AS option_api_key       -- pickOption 的 apiKey
FROM p_common_metadata po
WHERE po.metamodel_api_key = 'pickOption'
  AND po.delete_flg = 0;
```

**验证方法**：

```python
def verify_select_conversion(tenant_id, entity_api_key, item_api_key, sample_size=100):
    """验证 SELECT 字段转换正确性"""
    # 1. 从老库取样
    old_col = get_old_dbcolumn(entity_api_key, item_api_key)  # 如 dbc_select_5
    old_rows = query_old(f"""
        SELECT id, {old_col} FROM a_{entity_api_key}
        WHERE tenant_id = %s AND {old_col} IS NOT NULL AND del_flg = 0
        ORDER BY random() LIMIT %s
    """, (tenant_id, sample_size))

    # 2. 从新库取对应记录
    new_col = get_new_dbcolumn(entity_api_key, item_api_key)  # 如 dbc_varchar171
    errors = []
    for old_id, old_code in old_rows:
        new_row = query_new(f"""
            SELECT {new_col} FROM p_tenant_data_{route}
            WHERE id = %s AND tenant_id = %s
        """, (old_id, tenant_id))

        if not new_row:
            errors.append(('MISSING', old_id))
            continue

        new_value = new_row[0]
        expected = pickoption_code_map.get((entity_api_key, item_api_key), {}).get(old_code)

        if expected and new_value != expected:
            errors.append(('MISMATCH', old_id, old_code, expected, new_value))
        elif not expected and new_value != str(old_code):
            errors.append(('FALLBACK_MISMATCH', old_id, old_code, new_value))

    return errors
```

#### 11.2.3 MULTI_SELECT(16) — 格式变更

| 维度 | 老系统 | 新系统 | 转换 |
|:---|:---|:---|:---|
| 存储类型 | VARCHAR(300) (dbc_varchar_) | VARCHAR(300)[] (dbc_array) | 类型变更 |
| 值格式 | 逗号分隔数字（如 `"1,3,5"`） | PG 数组（如 `'{opt_a,opt_c,opt_e}'`） | 格式+值转换 |

```python
def convert_multiselect(entity_api_key, item_api_key, old_value):
    """MULTI_SELECT 转换：逗号分隔数字 → VARCHAR(300)[] 数组 apiKey"""
    if not old_value:
        return None

    codes = [c.strip() for c in str(old_value).split(',') if c.strip()]
    mapping = pickoption_code_map.get((entity_api_key, item_api_key), {})

    api_keys = []
    for code_str in codes:
        try:
            code = int(code_str)
        except ValueError:
            api_keys.append(code_str)  # 已经是字符串，直接保留
            continue

        if code in mapping:
            api_keys.append(mapping[code])
        else:
            api_keys.append(str(code))
            log_unmapped('MULTISELECT', entity_api_key, item_api_key, code)

    return json.dumps(api_keys)  # '["opt_a","opt_c","opt_e"]'
```

### 11.3 计算公式字段的转换与验证（核心）

计算公式字段（FORMULA/ROLLUP/COMPUTED）是最关键的验证对象——这些字段的值不是直接迁移，而是在同步后由新系统公式引擎重算，必须确保重算结果与老系统完全一致。

#### 11.3.1 计算字段处理策略

| 字段类型 | 同步时处理 | 同步后处理 | 验证方式 |
|:---|:---|:---|:---|
| FORMULA(6) — 持久化公式 | 从老库复制原始值（作为基准） | 新引擎重算，与基准比对 | 重算值 vs 老库值 |
| FORMULA(6) — 实时公式 | 不迁移（每次查询时计算） | 无需处理 | 查询时验证 |
| ROLLUP(7) — 汇总字段 | 从老库复制原始值（作为基准） | 新引擎重算，与基准比对 | 重算值 vs 老库值 |
| COMPUTED(27) | 从老库复制原始值（作为基准） | 新引擎重算，与基准比对 | 重算值 vs 老库值 |
| JOIN(21) — 虚拟引用 | 不迁移（无物理列） | 查询时动态取值 | 查询时验证 |

**关键决策**：同步时先把老库的计算结果值也复制过来，作为验证基准。新引擎重算后与基准比对，确认一致后再清除基准值。

#### 11.3.2 FORMULA 字段完整验证流程

```
Step 1: 同步时保留老值
  ├─ 从老库读取 FORMULA 字段的当前值（老系统已计算好的结果）
  ├─ 写入新库对应的 dbc 列（作为 baseline）
  └─ 同时写入 baseline 对照表: sync_formula_baseline

Step 2: 验证元数据完整性
  ├─ 每个 FORMULA 字段必须有 formulaCompute 定义
  │     SELECT fc.* FROM p_common_metadata fc
  │     WHERE fc.metamodel_api_key = 'formulaCompute'
  │       AND fc.entity_api_key = '{entity}'
  │       AND fc.dbc_varchar1 = '{itemApiKey}'  -- itemApiKey
  │
  ├─ 每个 formulaCompute 必须有 formulaComputeItem（参数列表）
  │     SELECT fci.* FROM p_common_metadata fci
  │     WHERE fci.metamodel_api_key = 'formulaComputeItem'
  │       AND fci.entity_api_key = '{entity}'
  │       -- 注意：computeApiKey 可能是负数 ID，需要通过 entity_api_key 关联
  │
  ├─ 每个参数引用的字段必须存在且有 dbColumn
  │     对 formulaComputeItem 中的每个 referItemApiKey:
  │       检查 item 元数据中该字段存在且 dbColumn 非空
  │
  └─ 公式表达式语法可解析
        尝试解析 computeExpression，提取所有参数引用
        验证参数引用与 formulaComputeItem 一致

Step 3: 新引擎重算
  ├─ 加载公式定义: FormulaMetaLoader.loadFormulas(tenantId, entityApiKey)
  ├─ 拓扑排序: DependencyAnalyzer.sort(formulas)
  ├─ 对每条业务数据:
  │     a. 取参数值（从已同步的业务数据中读取依赖字段的值）
  │     b. 执行公式求值
  │     c. 类型转换（ResultTypeConverter）
  │     d. 记录计算结果
  └─ 批量写回新库

Step 4: 新老值比对
  ├─ 对每条记录的每个 FORMULA 字段:
  │     baseline_value = sync_formula_baseline[record_id][item_apikey]
  │     recalc_value = 新引擎计算结果
  │     比对规则（见 §11.3.3）
  │
  ├─ 统计比对结果:
  │     exact_match: 完全一致
  │     precision_match: 精度容差内一致（CURRENCY/PERCENT）
  │     null_match: 新老都为 NULL
  │     mismatch: 不一致（需排查）
  │
  └─ 生成验证报告
```

#### 11.3.3 公式值比对规则

```python
def compare_formula_values(old_value, new_value, result_type):
    """
    比对公式计算结果
    result_type: formulaCompute.resultType（老编码）
      1=TEXT, 3=SELECT, 6=REAL, 21=DATETIME, 31=BOOL, 33=PERCENTAGE
    """
    # 双 NULL
    if old_value is None and new_value is None:
        return 'EXACT_MATCH'

    # 单侧 NULL
    if old_value is None or new_value is None:
        return 'NULL_MISMATCH'

    # TEXT(1) / SELECT(3): 字符串精确比较
    if result_type in (1, 3):
        return 'EXACT_MATCH' if str(old_value) == str(new_value) else 'MISMATCH'

    # BOOL(31): 统一为 0/1 比较
    if result_type == 31:
        old_bool = 1 if old_value in (1, True, '1', 'true') else 0
        new_bool = 1 if new_value in (1, True, '1', 'true') else 0
        return 'EXACT_MATCH' if old_bool == new_bool else 'MISMATCH'

    # REAL(6) / PERCENTAGE(33): 精度容差比较
    if result_type in (6, 33):
        try:
            diff = abs(float(old_value) - float(new_value))
            if diff == 0:
                return 'EXACT_MATCH'
            elif diff < 0.01:  # 1 分钱容差
                return 'PRECISION_MATCH'
            else:
                return 'MISMATCH'
        except (ValueError, TypeError):
            return 'TYPE_ERROR'

    # DATETIME(21): 毫秒时间戳比较
    if result_type == 21:
        try:
            old_ts = int(old_value)
            new_ts = int(new_value)
            if old_ts == new_ts:
                return 'EXACT_MATCH'
            elif abs(old_ts - new_ts) < 1000:  # 1 秒容差
                return 'PRECISION_MATCH'
            else:
                return 'MISMATCH'
        except (ValueError, TypeError):
            return 'TYPE_ERROR'

    # 其他: 字符串比较
    return 'EXACT_MATCH' if str(old_value) == str(new_value) else 'MISMATCH'
```

#### 11.3.5 ROLLUP 字段验证流程

ROLLUP（汇总）字段的验证更复杂，因为它依赖子对象的数据。

```
Step 1: 确保子对象数据已同步
  ROLLUP 字段依赖子对象数据，必须在子对象同步完成后才能验证。
  例: account.totalWonOpportunities 依赖 opportunity 数据

Step 2: 验证 aggregationCompute 元数据
  对每个 ROLLUP 字段:
    a. 查询 aggregationCompute 定义
       SELECT * FROM p_common_metadata
       WHERE metamodel_api_key = 'aggregationCompute'
         AND entity_api_key = '{parentEntity}'
         AND dbc_varchar2 = '{rollupItemApiKey}'  -- 父实体汇总字段（修正后语义）

    b. 确认字段语义（基于验证报告修正）:
       dbc_varchar1 = 子实体 apiKey（如 'opportunity'）
       dbc_varchar2 = 父实体汇总字段 apiKey（如 'totalWonOpportunities'）
       dbc_varchar3 = 子实体关联父实体的字段 apiKey（如 'accountId'）
       dbc_int1 = 汇总类型（0=COUNT, 1=SUM）

    c. 查询 aggregationComputeDetail（过滤条件）
       SELECT * FROM p_common_metadata
       WHERE metamodel_api_key = 'aggregationComputeDetail'
         AND dbc_varchar1 = '{aggregateApiKey}'

Step 3: 在新库中执行汇总 SQL
  -- 以 account.totalWonOpportunities 为例
  -- aggregateType=0 (COUNT), 子实体=opportunity, 关联字段=accountId
  SELECT
    {accountId_dbcol} AS parent_id,
    COUNT(*) AS agg_value
  FROM p_tenant_data_{opp_route}
  WHERE tenant_id = ?
    AND entity_api_key = 'opportunity'
    AND {accountId_dbcol} IN ({parent_ids})
    AND delete_flg = 0
    AND {filter_conditions}  -- 如 wonFlg = 1
  GROUP BY {accountId_dbcol}

Step 4: 与老库值比对
  对每条父记录:
    new_rollup = 汇总 SQL 结果
    old_rollup = baseline_table[parent_id]['totalWonOpportunities']
    比对（允许精度容差）
```

#### 11.3.6 跨实体公式验证

部分公式引用了其他实体的字段（跨实体参数），验证时需要确保关联数据已同步。

```
公式参数格式: {entityA}.{referItemApiKey}.{entityB}.{targetItemApiKey}
例: campaignInfluence.opportunityId.opportunity.money

验证步骤:
  1. 确认 entityA（campaignInfluence）数据已同步
  2. 确认 entityB（opportunity）数据已同步
  3. 通过 referItemApiKey（opportunityId）找到关联记录
  4. 从关联记录中取 targetItemApiKey（money）的值
  5. 代入公式计算
  6. 与老库值比对
```

```python
def verify_values(tenant_id, entity_api_key, sample_size=200):
    """逐字段类型抽样比对"""

            'sample': sample_size,
```

### 12.4 L4 计算验证（FORMULA + ROLLUP 重算比对）

```python
def verify_formulas(tenant_id, entity_api_key, sample_size=500):
    """FORMULA 字段重算验证"""
    # 1. 加载公式定义
    formulas = load_formula_computes(entity_api_key)
    if not formulas:
        return {'status': 'SKIP', 'reason': 'no formulas'}

    # 2. 抽样业务数据
    route = get_route(tenant_id, entity_api_key)
    records = query_new(f"""
        SELECT * FROM p_tenant_data_{route}
        WHERE tenant_id = %s AND entity_api_key = %s AND delete_flg = 0
        ORDER BY random() LIMIT %s
    """, (tenant_id, entity_api_key, sample_size))

    # 3. 对每个公式逐条验证
    results = {}
    for formula in formulas:
        item_api_key = formula['itemApiKey']
        expression = formula['computeExpression']
        result_type = formula['resultType']
        formula_dbcol = get_dbcolumn(entity_api_key, item_api_key)

        exact, precision, null_ok, mismatch, error = 0, 0, 0, 0, 0

        for record in records:
            # 取老值（baseline，同步时从老库复制过来的）
            baseline = record.get(formula_dbcol)

            # 取参数值
            params = {}
            for param in formula['params']:
                param_dbcol = get_dbcolumn(
                    param.get('referEntityApiKey') or entity_api_key,
                    param['referItemApiKey']
                )
                if param.get('referEntityApiKey'):
                    # 跨实体参数：需要查关联记录
                    ref_id = record.get(get_link_dbcol(entity_api_key, param))
                    if ref_id:
                        ref_val = query_remote_field(
                            tenant_id, param['referEntityApiKey'],
                            ref_id, param_dbcol
                        )
                        params[f"{param['referEntityApiKey']}.{param['referItemApiKey']}"] = ref_val
                else:
                    # 本实体参数
                    params[f"{entity_api_key}.{param['referItemApiKey']}"] = record.get(param_dbcol)

            # 执行公式计算
            try:
                computed = evaluate_formula(expression, params)
                computed = convert_result(computed, result_type)
            except Exception as e:
                error += 1
                continue

            # 比对
            match_result = compare_formula_values(baseline, computed, result_type)
            if match_result == 'EXACT_MATCH':
                exact += 1
            elif match_result == 'PRECISION_MATCH':
                precision += 1
            elif match_result in ('NULL_MISMATCH', 'MISMATCH'):
                mismatch += 1
                if mismatch <= 5:
                    log_formula_mismatch(
                        entity_api_key, item_api_key,
                        record['id'], baseline, computed, params
                    )
            else:
                null_ok += 1

        total = exact + precision + null_ok + mismatch + error
        results[item_api_key] = {
            'expression': expression[:80],
            'resultType': result_type,
            'total': total,
            'exact_match': exact,
            'precision_match': precision,
            'null_match': null_ok,
            'mismatch': mismatch,
            'error': error,
            'match_rate': f"{(exact+precision+null_ok)/max(total,1)*100:.1f}%"
        }

    return results


def verify_rollups(tenant_id, entity_api_key, sample_size=100):
    """ROLLUP 字段重算验证"""
    aggregations = load_aggregation_computes(entity_api_key)
    if not aggregations:
        return {'status': 'SKIP', 'reason': 'no rollups'}

    route = get_route(tenant_id, entity_api_key)
    parent_records = query_new(f"""
        SELECT id FROM p_tenant_data_{route}
        WHERE tenant_id = %s AND entity_api_key = %s AND delete_flg = 0
        ORDER BY random() LIMIT %s
    """, (tenant_id, entity_api_key, sample_size))
    parent_ids = [r[0] for r in parent_records]

    results = {}
    for agg in aggregations:
        # 修正后的字段语义
        child_entity = agg['dbc_varchar1']       # 子实体
        rollup_item = agg['dbc_varchar2']         # 父实体汇总字段
        link_item = agg['dbc_varchar3']           # 子实体关联字段
        agg_type = agg['dbc_int1']                # 0=COUNT, 1=SUM

        rollup_dbcol = get_dbcolumn(entity_api_key, rollup_item)
        link_dbcol = get_dbcolumn(child_entity, link_item)

        child_route = get_route(tenant_id, child_entity)

        # 执行汇总 SQL
        agg_func = 'COUNT(*)' if agg_type == 0 else f'SUM({get_agg_target_dbcol(agg)})'
        agg_results = query_new(f"""
            SELECT {link_dbcol} AS parent_id, {agg_func} AS agg_value
            FROM p_tenant_data_{child_route}
            WHERE tenant_id = %s AND entity_api_key = %s
              AND {link_dbcol} IN %s AND delete_flg = 0
            GROUP BY {link_dbcol}
        """, (tenant_id, child_entity, tuple(parent_ids)))

        agg_map = {r[0]: r[1] for r in agg_results}

        # 比对
        match, mismatch = 0, 0
        for pid in parent_ids:
            baseline = get_field_value(tenant_id, entity_api_key, pid, rollup_dbcol)
            computed = agg_map.get(pid, 0)

            if compare_formula_values(baseline, computed, 6) in ('EXACT_MATCH', 'PRECISION_MATCH'):
                match += 1
            else:
                mismatch += 1

        results[rollup_item] = {
            'child_entity': child_entity,
            'agg_type': 'COUNT' if agg_type == 0 else 'SUM',
            'total': len(parent_ids),
            'match': match, 'mismatch': mismatch,
            'match_rate': f"{match/max(len(parent_ids),1)*100:.1f}%"
        }

    return results
```

## 十三、公式引擎新老一致性保障措施

### 13.1 表达式兼容性

老系统公式表达式格式在新系统中完全复用，无需修改：

| 表达式特性 | 老系统 | 新系统 | 兼容性 |
|:---|:---|:---|:---|
| 参数格式 | `entity.item` | `entity.item` | ✅ 完全一致 |
| 多级引用 | `a.b.c.d` | `a.b.c.d` | ✅ 完全一致 |
| IF/OR/AND | 支持 | 支持 | ✅ 复用同一引擎 |
| 数学运算 | +, -, *, / | +, -, *, / | ✅ |
| 字符串比较 | `=` | `=`（引擎内部处理） | ✅ |
| 内置函数 | OPTIONAPINAME, NUMBERSTRING 等 | 复用 | ✅ |

### 13.2 计算触发一致性

| 触发场景 | 老系统 | 新系统 | 一致性 |
|:---|:---|:---|:---|
| 新建记录 | 前置计算所有公式 | ComputePipeline.computeForCreate | ✅ |
| 更新记录 | 检测变更字段→递归找受影响公式→重算 | DependencyAnalyzer.findAffected→重算 | ✅ |
| 特殊公式（自增/本币） | 入库后循环计算（最多 20 层） | computeSpecialFormulas（同逻辑） | ✅ |
| 汇总触发 | MQ 异步→paas-compute-service | MQ 异步→AggregationEngine | ✅ |
| 实时公式 | 查询时计算 | computeForQuery | ✅ |

### 13.3 数值精度一致性

| 类型 | 老系统 | 新系统 | 处理 |
|:---|:---|:---|:---|
| REAL(6) | DOUBLE PRECISION | DECIMAL(20,4) | ROUND_HALF_UP，4 位小数 |
| PERCENTAGE(33) | DOUBLE PRECISION | DECIMAL(20,4) | 同上 |
| INTEGER(5) | BIGINT | BIGINT | 无精度问题 |
| BOOL(31) | '1'/'0' | '1'/'0' | 字符串比较 |

**精度风险**：老系统用 DOUBLE（浮点），新系统用 DECIMAL（定点），极端情况下可能有微小差异（如 0.1+0.2 在浮点下为 0.30000000000000004）。新系统精度更高，以新系统为准。

### 13.4 null 处理一致性

| nullTreatment | 含义 | 老系统 | 新系统 |
|:---|:---|:---|:---|
| 0 | null 视为 0 | 参数为 null 时替换为 0 | 同 |
| 1 | null 视为空 | 参数为 null 时结果为 null | 同 |
| 2 | （实际数据中的值） | 推测同 0 或同 1 | 按"null 参与计算结果为 null"处理，验证后确认 |

**验证方法**：找到参数含 NULL 的记录，分别用 nullTreatment=0 和 nullTreatment=1 计算，与老库值比对，确认 nullTreatment=2 的实际语义。
