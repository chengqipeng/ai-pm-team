# 元数据引用注册设计方案

> 版本：1.0 | 日期：2026-04-22
> 定位：追踪元数据之间的引用关系，实现删除保护和影响分析
> 参考：老系统 `p_metadata_use_info` 表 + `XMetadataUseInfoService`

---

## 一、问题

当前系统已有以下功能在运行时引用元数据：

| 功能 | 引用方 | 被引用方 | 当前状态 |
|---|---|---|---|
| 计算公式 | formulaCompute 表达式 | item 字段 | ✅ 运行中，DependencyAnalyzer 分析依赖 |
| 汇总累计 | aggregationCompute | item 字段 + 子实体 | ✅ 运行中 |
| 画布页面 | pageSection(CanvasSchema) | item 字段（columns 引用） | ✅ 运行中 |
| 字段默认值 | item.defaultValue 公式 | 其他 item 字段 | ✅ 运行中 |
| 关联筛选器 | referenceFilter | item 字段 | ✅ 数据已有 |
| 校验规则 | checkRule.checkFormula | item 字段 | ✅ 数据已有 |
| 查重规则 | duplicateRuleCriteria | item 字段 | ✅ 数据已有 |
| 字段集 | fieldSetItem | item 字段 | ✅ 数据已有 |
| 字段依赖 | itemDependency | item 字段（级联选项） | ✅ 数据已有 |

**但删除操作没有任何引用检查**：前端 `ActionContext` 的 `delete` 动作直接调用 `batchSaveMetadata({ deleteMap })` 删除元数据，不检查是否被其他元数据引用。删除一个被公式引用的字段会导致计算引擎运行时报错。

## 二、设计目标

1. 删除元数据前，检查是否被其他元数据引用，阻止或提示用户
2. 引用关系由系统自动维护（元数据写入时注册，删除时清理），不需要用户手动管理
3. 支持查询"谁引用了我"（删除保护）和"我引用了谁"（影响分析）
4. 遵循 apiKey 关联规范，不使用 ID

## 三、数据模型

### 3.1 新增元模型：`metadataRef`（元数据引用注册）

在 `p_meta_model` 中注册为独立元模型，数据存储在 Tenant 级快捷表 `p_tenant_metadata_ref`。

| 属性 | 值 |
|---|---|
| api_key | `metadataRef` |
| label | 元数据引用注册 |
| metamodel_type | 5（系统运维） |
| enable_common | 0 |
| enable_tenant | 1 |
| enable_tenant_intercept | 1 |
| enable_ui_config | 0（不在画布数据源中显示） |
| db_table | `p_tenant_metadata_ref` |

### 3.2 字段定义（p_meta_item）


| api_key | db_column | label | 类型 | 说明 |
|---|---|---|---|---|
| targetMetamodelApiKey | dbc_varchar1 | 被引用元模型 | String | 被引用的元数据所属元模型（如 `item`） |
| targetApiKey | dbc_varchar2 | 被引用元数据 | String | 被引用的元数据 apiKey（如 `phone`） |
| targetEntityApiKey | dbc_varchar3 | 被引用所属实体 | String | 被引用元数据的 entityApiKey（如 `account`），独立元模型为空 |
| sourceMetamodelApiKey | dbc_varchar4 | 引用来源元模型 | String | 引用方所属元模型（如 `formulaCompute`） |
| sourceApiKey | dbc_varchar5 | 引用来源元数据 | String | 引用方的 apiKey |
| sourceEntityApiKey | dbc_varchar6 | 引用来源所属实体 | String | 引用方的 entityApiKey |
| refType | dbc_smallint1 | 引用类型 | Integer | 见下方枚举 |

### 3.3 引用类型枚举（refType）

| 值 | 名称 | 说明 | 引用方 → 被引用方 |
|---|---|---|---|
| 1 | FORMULA_PARAM | 计算公式引用字段 | formulaCompute → item |
| 2 | AGGREGATION_SOURCE | 汇总累计引用字段 | aggregationCompute → item |
| 3 | DEFAULT_VALUE | 默认值引用字段 | item(defaultValue) → item |
| 4 | CHECK_RULE_EXPR | 校验规则引用字段 | checkRule → item |
| 5 | DUPLICATE_CRITERIA | 查重条件引用字段 | duplicateRuleCriteria → item |
| 6 | FIELD_SET_MEMBER | 字段集包含字段 | fieldSetItem → item |
| 7 | ITEM_DEPENDENCY | 字段级联依赖 | itemDependency → item |
| 8 | REFER_FILTER | 关联筛选器引用字段 | referenceFilterDetail → item |
| 9 | CANVAS_COLUMN | 画布列引用字段 | pageSection(CanvasSchema) → item |
| 10 | AGGREGATION_FILTER | 汇总过滤条件引用字段 | aggregationComputeDetail → item |

### 3.4 与老系统 `p_metadata_use_info` 的对比

| 维度 | 老系统 | 新系统 |
|---|---|---|
| 表名 | `p_metadata_use_info` | `p_tenant_metadata_ref` |
| 关联方式 | ID（metaModelId, metadataDataId） | apiKey（targetMetamodelApiKey, targetApiKey） |
| 存储方式 | 独立表 + 直接 DAO | 元模型化，走 Common/Tenant 标准链路 |
| 跨服务注册 | microServiceCode + Dubbo RPC | 不需要（新系统单体服务，所有引用在同一个 JVM 内） |
| 引用类型 | metadataType（整数，含非元数据化类型） | refType（纯枚举，只覆盖元数据引用） |
| 字段定位 | objectId + itemId（多字段组合定位） | targetEntityApiKey + targetApiKey（apiKey 二元组） |

## 四、引用注册时机

### 4.1 自动注册（元数据写入时）

在 `MetadataBatchSaveApiService.batchSave` 的 create/update 流程中，按元模型类型触发引用分析：

| 元模型 | 触发条件 | 分析逻辑 | 注册的引用 |
|---|---|---|---|
| `formulaCompute` | create/update | 解析 `computeExpression` 中的 `entityApiKey.itemApiKey` 参数 | formulaCompute → 被引用的 item |
| `aggregationCompute` | create/update | 读取 `sourceItemApiKey`、`targetItemApiKey` | aggregationCompute → 被引用的 item |
| `aggregationComputeDetail` | create/update | 读取过滤条件中的字段引用 | aggregationComputeDetail → 被引用的 item |
| `item`（defaultValue 变更） | update | 解析 `defaultValue` 公式中的字段引用 | item → 被引用的 item |
| `checkRule` | create/update | 解析 `checkFormula` 中的字段引用 | checkRule → 被引用的 item |
| `duplicateRuleCriteria` | create/update | 读取 `itemApiKey` | duplicateRuleCriteria → 被引用的 item |
| `fieldSetItem` | create/update | 读取 `itemApiKey` | fieldSetItem → 被引用的 item |
| `itemDependency` | create/update | 读取 `controlItemApiKey` | itemDependency → 被引用的 item |
| `referenceFilterDetail` | create/update | 读取过滤条件中的字段引用 | referenceFilterDetail → 被引用的 item |

### 4.2 自动清理（元数据删除时）

删除引用方元数据时，同步删除其注册的引用记录：

```
删除 formulaCompute X
  → DELETE FROM p_tenant_metadata_ref WHERE sourceMetamodelApiKey='formulaCompute' AND sourceApiKey='X'
```

### 4.3 画布引用注册

画布 Schema 保存时（`ensureCommonPages` 或画布编辑器保存），扫描 `CanvasSchema.components` 中所有 `dataTable` 的 `columns` 配置，注册对字段的引用：

```
pageSection 'pageMetamodel_canvas' 的 CanvasSchema 中：
  dataTable.columns = ['apiKey', 'label', 'dbColumn', 'itemType', ...]
  → 注册 pageSection → item(apiKey), item(label), item(dbColumn), ...
```

## 五、删除保护流程

### 5.1 后端校验

在 `MetadataBatchSaveApiService.executeDelete` 中，删除前查询引用：

```java
// 伪代码
for (metamodelApiKey, deleteList) in deleteMap:
    for record in deleteList:
        refs = metadataRefService.findByTarget(metamodelApiKey, record.apiKey, record.entityApiKey)
        if refs is not empty:
            throw BusinessException("无法删除「{record.label}」，被以下元数据引用：{refs}")
```

### 5.2 前端提示

后端返回引用列表后，前端展示：

```
无法删除字段「phone」，被以下元数据引用：
  · 计算公式「fullName」（实体 account）
  · 校验规则「phoneRequired」（实体 account）
  · 画布页面「pageUserList」列配置
```

### 5.3 强制删除（级联清理）

管理员可选择强制删除，此时：
1. 先删除所有引用记录（`metadataRef`）
2. 再删除被引用的元数据本身
3. 引用方的公式/规则变为无效状态（运行时跳过或报错）

## 六、查询接口

### 6.1 后端 API

```
GET /meta/metadata-refs?targetMetamodelApiKey=item&targetApiKey=phone&targetEntityApiKey=account
→ 返回所有引用 account.phone 字段的元数据列表

GET /meta/metadata-refs?sourceMetamodelApiKey=formulaCompute&sourceApiKey=fullNameFormula
→ 返回 fullNameFormula 公式引用的所有字段列表
```

### 6.2 通用接口复用

由于 `metadataRef` 是标准元模型，可以直接用现有的通用接口：

```
GET /meta/metadata?metamodelApiKey=metadataRef
→ 返回所有引用记录（全量，前端过滤）
```

但建议新增专用接口，支持按 target/source 维度高效查询。

## 七、实施计划

| 阶段 | 内容 | 优先级 |
|---|---|---|
| 1 | 注册 `metadataRef` 元模型 + 建表 + 字段定义 | P0 |
| 2 | `batchSave` 删除前引用检查（阻止删除） | P0 |
| 3 | `formulaCompute` 写入时自动注册引用 | P1 |
| 4 | `checkRule`/`duplicateRuleCriteria` 写入时注册 | P1 |
| 5 | `item.defaultValue` 变更时注册 | P1 |
| 6 | 画布 Schema 保存时注册 | P2 |
| 7 | 前端删除确认弹框展示引用列表 | P2 |
| 8 | 存量数据引用关系补录（扫描现有公式/规则，批量注册） | P2 |

## 八、存储估算

| 维度 | 估算 |
|---|---|
| 每个公式平均引用 3 个字段 | 公式数 × 3 |
| 每个校验规则平均引用 2 个字段 | 规则数 × 2 |
| 每个画布页面平均引用 10 个字段 | 页面数 × 10 |
| 单租户总量 | 约 500-2000 条 |
| 查询频率 | 仅删除时查询，低频 |

不需要缓存，直接查表即可满足性能要求。
