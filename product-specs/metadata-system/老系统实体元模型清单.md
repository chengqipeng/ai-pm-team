# 老系统实体域元模型完整清单

> 来源：`repos/apass_old_projects/paas-metarepo-service/`
> 分析依据：`sysmeta/*.schema.json`（Schema 定义）、`MetaModelKit.java`（域分类 + 分层注册）、`dao.xml`（PO 类映射）、`MetaContextKit.java`（父子关系）
> 整理日期：2026-04-02

## 概述

老系统通过 `MetaConstants.METAMODEL_ID_*` 常量定义元模型 ID（负数 Long），每个元模型对应：
- Schema 定义文件（`sysmeta/*.schema.json`）— 定义字段结构和校验规则
- DTO 类（X 前缀，facade 层）— 服务间传输对象
- PO 类（repository 层）— 数据库持久化对象

实体域元模型共 **31 个**（含 3 个扩展能力元模型），按层级关系分为核心元模型和子元模型。

父子关系通过 `MetaContextKit.childMetaModelIdMap` 定义，数据查询时自动级联加载子模型。

## PO 类动态选择逻辑

老系统部分元模型的 PO 类会根据运行时条件动态选择（`MetaModelKit.getPoClazz`）：

| 元模型 | 条件 | PO 类 |
|:---|:---|:---|
| item | objectId < 系统自定义对象阈值 | `Item`（系统标准对象） |
| item | objectId >= 系统自定义对象阈值 | `CustomItem`（自定义对象） |
| busiType | objectId < 系统自定义对象阈值 | `EntityBelongType`（系统标准对象） |
| busiType | objectId >= 系统自定义对象阈值 | `BusiType`（自定义对象） |
| pickOption | objectId == null | `Pickoption`（通用选项） |
| pickOption | objectId < 阈值 且 optionType=SELECT | `SelectItem`（系统标准对象单选） |
| pickOption | objectId < 阈值 且 optionType=CHECK | `CheckItem`（系统标准对象多选） |
| pickOption | objectId >= 阈值 | `Pickoption`（自定义对象） |
