# 元模型分类体系设计

> 版本：1.0 | 日期：2026-04-20
> 目标：通过 metamodel_type 字段将 59+ 个元模型分为清晰的类别

---

## 一、当前问题

p_meta_model 表有 `metamodel_type` 字段，但只有两个值：
- `1`：26 个 entity 相关子元模型（item/checkRule/pickOption 等）
- `9`：1 个（globalPickOption）
- `NULL`：34 个（role/department/page/menu/uiAction 等全部未分类）

导致：
- 前端元模型选择器 59 个混在一起
- 无法按类别管理和授权
- 实体元数据和 UI 配置元数据混淆

## 二、分类定义

| metamodel_type | 分类名 | 说明 | 示例 |
|:---:|:---|:---|:---|
| 1 | 实体元数据 | entity 及其子元模型，描述业务对象的结构 | entity, item, entityLink, checkRule, pickOption, busiType, duplicateRule, formulaCompute... |
| 2 | 组织权限 | 组织架构、角色、权限相关 | role, department, sharingRule, sharingRuleCondition, dataPermission, publicGroupMember |
| 3 | UI 页面 | 页面定义、画布、菜单 | page, pageSection, pageAction, menu |
| 4 | UI 组件 | 画布组件、按钮样式、事件动作 | uiComponent, uiComposite, uiButtonStyle, uiAction |
| 5 | 系统运维 | 日志、任务、文件、认证 | metaOperateLog, entityOperateLog, excelTask, fileMeta, metaLog |
| 6 | 认证安全 | 用户认证、密码策略、OAuth | totpSecret, passwordPolicy, authProvider, oauthClient, loginPolicy, ipWhitelist, thirdBinding |
| 7 | 数据基础设施 | 分表路由、数据统计 | tenantDataRoute, tenantDataStats, dataShare, dataShareRoute |
| 8 | 业务扩展 | 岗位、助理、团队、区域 | position, positionUser, assistantRelation, teamMember, territory, territoryMember |
| 9 | 全局配置 | 全局选项集、字段依赖 | globalPickOption, globalPickDependency, globalPickDependencyDetail, itemDependency, itemDependencyDetail |

## 四、前端使用

### 4.1 元模型选择器分组

画布编辑器中的"元模型"下拉选择器，按 metamodel_type 分组显示：

```
▸ 实体元数据 (26)
    entity - 自定义业务对象
    item - 自定义字段
    ...
▸ 组织权限 (5)
    role - 角色
    department - 部门
    ...
▸ 全局配置 (5)
    globalPickOption - 全局选项集
    ...
```

### 4.2 权限控制

- 实体元数据（type=1）：业务管理员可配置
- UI 页面/组件（type=3/4）：系统管理员可配置
- 认证安全（type=6）：超级管理员可配置
- 数据基础设施（type=7）：仅平台运维可见

## 五、实施

直接更新 p_meta_model.metamodel_type：

```sql
-- type=2 组织权限
UPDATE p_meta_model SET metamodel_type = 2 WHERE api_key IN ('role','department','sharingRule','sharingRuleCondition','dataPermission');

-- type=3 UI 页面
UPDATE p_meta_model SET metamodel_type = 3 WHERE api_key IN ('menu','page','pageSection','pageAction');

-- type=4 UI 组件
UPDATE p_meta_model SET metamodel_type = 4 WHERE api_key IN ('uiComponent','uiComposite','uiButtonStyle','uiAction');

-- type=5 系统运维
UPDATE p_meta_model SET metamodel_type = 5 WHERE api_key IN ('metaOperateLog','entityOperateLog','excelTask','fileMeta','metaLog');

-- type=6 认证安全
UPDATE p_meta_model SET metamodel_type = 6 WHERE api_key IN ('totpSecret','passwordPolicy','authProvider','oauthClient','loginPolicy','ipWhitelist','thirdBinding');

-- type=7 数据基础设施
UPDATE p_meta_model SET metamodel_type = 7 WHERE api_key IN ('tenantDataRoute','tenantDataStats','dataShare','dataShareRoute');

-- type=8 业务扩展
UPDATE p_meta_model SET metamodel_type = 8 WHERE api_key IN ('position','positionUser','assistantRelation','teamMember','territory','territoryMember');

-- type=9 全局配置（globalPickOption 已经是 9，补充其他）
UPDATE p_meta_model SET metamodel_type = 9 WHERE api_key IN ('globalPickDependency','globalPickDependencyDetail','itemDependency','itemDependencyDetail');

-- 确保 publicGroupMember 归入组织权限
UPDATE p_meta_model SET metamodel_type = 2 WHERE api_key = 'publicGroupMember';
```
