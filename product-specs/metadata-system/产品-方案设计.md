# aPaaS 元数据体系 — 方案设计

## 1. 现状分析

### 1.1 已实现能力（基于 paas-metarepo-service 代码分析）

**元模型定义层（4 张核心表，已实现）：**
- p_meta_model：元模型注册表，当前 7 种元模型（entity/item/entityLink/checkRule/pickOption/referenceFilter/globalPickOption）
- p_meta_item：元模型字段定义，176 个字段，通过 db_column 映射到大宽表 dbc_xxxN 列
- p_meta_link：元模型间关联关系，5 条层级关系（entity→item→pickOption/referenceFilter, entity→entityLink/checkRule）
- p_meta_option：元模型字段取值范围，62 个选项值覆盖 17 个枚举字段

**元数据实例层（大宽表机制，已实现）：**
- p_common_metadata：Common 级大宽表（固定列 + 130 个 dbc_xxxN 扩展列）
- p_tenant_*：7 张 Tenant 级独立快捷表（结构与 p_common_metadata 一致 + tenant_id）
- DynamicTableNameHolder 路由：写入时自动路由到 db_table 指向的 Tenant 表
- CommonMetadataConverter：读取时通过 p_meta_item.db_column 映射还原字段语义

**合并读取机制（已实现）：**
- IMetadataMergeReadService：Common + Tenant 合并读取
- 合并规则：Common 有 Tenant 无→用 Common，同 apiKey→Tenant 覆盖，Tenant delete_flg=1→隐藏

**API 层（已实现）：**
- MetaRepoReadApi：listEntities/getEntity/listItems/listPickOptions/listCheckRules/listEntityLinks
- MetaRepoWriteApi：createEntity/updateEntity/deleteEntity/createItem/updateItem/deleteItem
- MetamodelBrowseApiService：listMetaModels/listMetaItems/getColumnMapping

**前端（paas-metarepo-web，已实现）：**
- React 19 + Antd 6 + Vite 8 + TypeScript 5.9
- 元模型浏览、元数据列表、字段映射可视化

## 2. 架构设计

### 2.1 三层架构总览

```
┌─────────────────────────────────────────────────────────────┐
│ 应用层（CRM 业务 / AI 能力 / 工作流）                         │
│   通过 MetaRepoReadApi 获取元数据，驱动业务逻辑               │
├─────────────────────────────────────────────────────────────┤
│ 平台服务层                                                   │
│   entity-service │ layout-service │ rule-service │ gateway  │
│   基于元数据驱动实体 CRUD、布局渲染、规则执行                   │
├─────────────────────────────────────────────────────────────┤
│ 元数据仓库层（paas-metarepo-service）                        │
│   ┌─────────────────────────────────────────────────────┐   │
│   │ 元模型定义（Schema）                                  │   │
│   │   p_meta_model → p_meta_item → p_meta_option        │   │
│   │                → p_meta_link                         │   │
│   ├─────────────────────────────────────────────────────┤   │
│   │ 元数据实例（Data）                                    │   │
│   │   Common: p_common_metadata（大宽表）                 │   │
│   │   Tenant: p_tenant_entity/item/...（独立快捷表）      │   │
│   ├─────────────────────────────────────────────────────┤   │
│   │ 合并引擎                                              │   │
│   │   CommonMetadataConverter + MergeReadService          │   │
│   │   namespace 过滤 + license 控制                       │   │
│   └─────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│ 存储层                                                       │
│   paas_metarepo_common（元模型 + Common 元数据）              │
│   paas_metarepo（Tenant 元数据 + 日志）                      │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 数据流

**读取流程：**
```
业务请求(tenantId, metamodelApiKey, entityApiKey)
  → MetaModel 查询(enable_common, enable_tenant, db_table)
  → [if enable_common] 查 p_common_metadata + CommonMetadataConverter 转换
  → [if enable_tenant] 查 db_table 指向的 Tenant 表
  → MergeReadService 合并(Common有Tenant无→用Common, 同apiKey→Tenant覆盖, delete_flg=1→隐藏)
  → namespace 过滤(product 检查 license)
  → 返回业务 Entity 列表
```

**写入流程：**
```
业务请求(tenantId, 业务Entity)
  → Schema 校验(p_meta_option 取值范围 + p_meta_item 必填/唯一)
  → CommonMetadataConverter.toTenantMetadata()
  → DynamicTableNameHolder.executeWith(dbTable)
  → INSERT/UPDATE Tenant 表
  → [if enable_log] 写入 p_tenant_meta_log
```

### 2.3 关键设计决策

| 决策 | 选择 | 理由 |
|:---|:---|:---|
| 存储模型 | Schema-on-Read 大宽表 | 动态扩展无需 DDL，新增元模型零成本 |
| 关联方式 | api_key 关联（禁止 ID） | 跨环境迁移一致性，人类可读 |
| 租户隔离 | Common/Tenant 双库 + 独立快捷表 | 出厂数据独立升级，租户数据互不影响 |
| 列名格式 | dbc_xxxN（无下划线分隔数字） | 统一规范，避免历史遗留格式混乱 |
| 主键策略 | 雪花算法 BIGINT（禁止自增） | 分布式环境唯一性 |
| DDL 兼容 | MySQL + PostgreSQL | 多数据库部署支持 |

## 3. 模块拆分

### 3.1 paas-metarepo-service（核心）

| 子模块 | 职责 |
|:---|:---|
| paas-metarepo-service-core | API 模型（XEntity/XEntityItem/XLink 等）+ Feign 接口定义 |
| paas-metarepo-service-feign | Feign Client 实现 |
| paas-metarepo-service-server | 服务实现：元模型管理 + 元数据 CRUD + 合并读取 + 变更日志 |

**核心 Service：**

| Service | 职责 |
|:---|:---|
| IMetaModelService | 元模型定义查询与管理 |
| IMetaItemService | 元模型字段定义查询 |
| ICommonMetadataService | Common 大宽表 CRUD |
| ITenantMetadataService | Tenant 大宽表 CRUD（通过 DynamicTableNameHolder 路由） |
| IMetadataMergeReadService | Common + Tenant 合并读取 |
| IMetadataMergeWriteService | Tenant 级写入 + Schema 校验 + 变更日志 |
| CommonMetadataConverter | 大宽表行 ↔ 业务 Entity 双向转换 |

### 3.2 paas-metarepo-web（管理前端）

| 页面 | 功能 |
|:---|:---|
| 元模型列表 | 展示所有已注册元模型，字段数、关联关系 |
| 元模型详情 | 字段定义列表、db_column 映射、取值约束 |
| 元数据浏览 | 按 entity 分组浏览元数据实例，Common/Tenant 标识 |
| 元数据编辑 | Tenant 级元数据的创建/更新/删除 |
| 字段映射可视化 | db_column → 大宽表物理列的映射关系图 |

## 4. 接口规格

### 4.1 读接口（MetaRepoReadApi）

| 方法 | 路径 | 参数 | 返回 |
|:---|:---|:---|:---|
| listEntities | GET /metarepo/read/entities | tenantId(header) | List\<XEntity\> |
| getEntity | GET /metarepo/read/entity | apiKey | XEntity |
| listItems | GET /metarepo/read/items | entityApiKey | List\<XEntityItem\> |
| listPickOptions | GET /metarepo/read/pick-options | entityApiKey, itemApiKey | List\<XPickOption\> |
| listCheckRules | GET /metarepo/read/check-rules | entityApiKey | List\<XCheckRule\> |
| listEntityLinks | GET /metarepo/read/entity-links | entityApiKey | List\<XLink\> |

### 4.2 写接口（MetaRepoWriteApi）

| 方法 | 路径 | 参数 | 说明 |
|:---|:---|:---|:---|
| createEntity | POST /metarepo/write/entity | XEntity body | 创建 Tenant 级对象 |
| updateEntity | PUT /metarepo/write/entity | XEntity body | 更新（Tenant 覆盖 Common） |
| deleteEntity | DELETE /metarepo/write/entity | apiKey | 软删除（遮蔽删除） |
| createItem | POST /metarepo/write/item | XEntityItem body | 创建字段 |
| updateItem | PUT /metarepo/write/item | XEntityItem body | 更新字段 |
| deleteItem | DELETE /metarepo/write/item | apiKey, entityApiKey | 级联删除 pickOption/referenceFilter |

### 4.3 内部接口（MetamodelBrowseApiService）

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| listMetaModels | GET /metarepo/internal/metamodels | 元模型列表 |
| listMetaItems | GET /metarepo/internal/meta-items | 元模型字段映射 |
| getColumnMapping | GET /metarepo/internal/column-mapping | 列映射摘要 |

## 5. 数据迁移方案

详见 repos/apass_ai_platform/apass_new_projects/paas-metarepo-service/METAREPO-DATA-MIGRATION.md

**执行顺序：**
1. 备份数据
2. 表名标准化（x_global_pickitem → p_tenant_global_pickitem）
3. 更新 p_meta_item 字段定义（删除/重命名老字段）
4. 转换 item_type 编码（剩余 3,333 条）
5. 重新分配 db_column（按 entity 分组 + itemType 前缀递增）
6. globalPickItem 字段值迁移（ID → apiKey）
7. 验证

## 6. 性能设计

| 场景 | 策略 |
|:---|:---|
| 元模型定义读取 | 本地缓存（启动时加载，变更时刷新） |
| 元数据列表查询 | Redis 缓存 + 按 entity_api_key 分片 |
| Common/Tenant 合并 | 并行查询 + 内存合并 |
| 大宽表索引 | (metamodel_api_key, entity_api_key, api_key) 联合索引 |
| Tenant 表索引 | (tenant_id, metamodel_api_key, entity_api_key) 联合索引 |

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|:---|:---|:---|
| 老数据迁移失败 | 业务中断 | 分批迁移 + 回滚脚本 + 灰度验证 |
| 大宽表列数不足 | 新元模型无法注册 | 当前 130 列，预留充足；极端情况可扩展 |
| 合并读取性能 | 高并发下延迟 | 缓存 + 并行查询 + 索引优化 |
| 前后端字段映射不一致 | 渲染异常 | ItemTypeEnum 统一管理，API 优先加载 |
