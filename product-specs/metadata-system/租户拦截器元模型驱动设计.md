# 租户拦截器元模型驱动设计（enable_tenant_intercept）

> 版本：v3.0 | 日期：2026-04-16
> 关联文档：[元模型设计体系](元模型设计体系.md)、[操作日志下沉设计](操作日志下沉framework-dao详细设计.md)、[DAO层使用设计规范](DAO层使用设计规范.md)

## 一、问题背景

`EntityPendingTransactionRecovery` 在应用启动时通过 `@PostConstruct` 调用 `OperateLogBaseDao.findPendingProcessIds()` 查询操作日志表。该方法继承自 `DataBaseServiceImpl`，走 MyBatis-Plus 标准链路，`TenantLineInnerInterceptor` 自动追加 `tenant_id` 条件，但启动阶段 `GlobalContext` 中无 `tenantId`，导致 `IllegalStateException`。

根因：操作日志表从 `JdbcTemplate` 迁移到 `DataBaseServiceImpl` 后，启动恢复和跨租户扫描场景不适合被拦截器管控。

## 二、设计目标

将"跳过租户拦截"实现为 **framework-dao 框架级能力**：
- 业务 DAO 不需要手动包装任何拦截器控制代码
- 通过 Entity 注解声明式配置，框架自动识别并处理
- 与 `@DaoCacheConfig`（缓存配置）、`@TableName`（表名路由）同级的框架注解

## 三、技术基础

### 3.1 MyBatis-Plus 官方机制

`InterceptorIgnoreHelper`（3.5.3+）是 MyBatis-Plus 官方提供的编程式拦截器控制 API（[官方文档](https://baomidou.com/en/plugins/)）：

```java
try {
    InterceptorIgnoreHelper.handle(IgnoreStrategy.builder().tenantLine(true).build());
    // 此范围内的 SQL 不走租户拦截
} finally {
    InterceptorIgnoreHelper.clearIgnoreStrategy();
}
```

原理：`TenantLineInnerInterceptor.beforeQuery()` / `beforePrepare()` 入口处第一行调用 `willIgnoreTenantLine(ms.getId())`，为 true 时直接跳过整个 SQL 解析和条件追加（[源码分析](https://www.cnblogs.com/changelzj/p/18893845)）。

### 3.2 框架注解先例

framework-dao 已有通过注解在 Entity 级别配置框架行为的模式：
- `@TableName("p_xxx")` — 表名路由
- `@DaoCacheConfig` — 缓存策略

新增 `@IgnoreTenantLine` 遵循同一模式。

## 四、p_meta_model 新增字段

```sql
ALTER TABLE paas_metarepo_common.p_meta_model
    ADD COLUMN enable_tenant_intercept SMALLINT NOT NULL DEFAULT 1;

UPDATE paas_metarepo_common.p_meta_model
SET enable_tenant_intercept = 0
WHERE api_key IN ('metaOperateLog', 'entityOperateLog');
```

| enable_tenant | enable_tenant_intercept | 含义 |
|:---:|:---:|:---|
| 1 | 1 | 有 tenant_id，拦截器自动追加条件（默认） |
| 1 | 0 | 有 tenant_id，框架自动跳过拦截，业务代码自行控制 |
| 0 | — | 无 Tenant 级存储，不涉及 |

## 五、Java 实现

### 5.1 新增 `@IgnoreTenantLine` 注解（framework-dao）

```java
package com.hongyang.framework.dao.annotation;

import java.lang.annotation.*;

/**
 * 标记 Entity 类跳过 TenantLineInnerInterceptor 租户拦截。
 *
 * 加在 Entity 类上，DataBaseServiceImpl 初始化时自动识别，
 * 该 Entity 的所有数据库操作自动通过 InterceptorIgnoreHelper 跳过租户拦截。
 *
 * 对应 p_meta_model.enable_tenant_intercept=0 的配置语义。
 *
 * 适用场景：
 * - 系统补偿表（操作日志），启动恢复需跨租户扫描
 * - 系统任务表，调度器需跨租户查询
 * - 审计表，管理员需跨租户查看
 *
 * 使用示例：
 * <pre>
 * {@literal @}Data
 * {@literal @}IgnoreTenantLine
 * {@literal @}TableName("p_entity_operate_log")
 * public class EntityOperateLog extends OperateLog {}
 * </pre>
 */
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface IgnoreTenantLine {
}
```

### 5.2 `DataBaseServiceImpl` 框架基类改造

```java
package com.hongyang.framework.dao.service;

import com.baomidou.mybatisplus.extension.plugins.inner.InterceptorIgnoreHelper;
import com.baomidou.mybatisplus.extension.plugins.handler.IgnoreStrategy;
import com.hongyang.framework.dao.annotation.IgnoreTenantLine;
// ... 其他 import

/**
 * 业务数据 Service 基类。
 * 提供缓存、租户隔离、双阶段缓存失效、软删除等核心能力。
 *
 * 新增：@IgnoreTenantLine 支持。
 * 如果泛型参数 T 的 Entity 类标注了 @IgnoreTenantLine，
 * 则该 Service 的所有数据库操作自动跳过 TenantLineInnerInterceptor。
 */
public class DataBaseServiceImpl<T> extends ... {

    private static final IgnoreStrategy IGNORE_TENANT =
            IgnoreStrategy.builder().tenantLine(true).build();

    /** 当前 Entity 是否标注了 @IgnoreTenantLine */
    private volatile Boolean ignoreTenantLine;

    /**
     * 延迟初始化：判断当前 Entity 类是否标注了 @IgnoreTenantLine。
     * 使用 MyBatis-Plus 的 entityClass 获取泛型参数的实际类型。
     */
    private boolean isIgnoreTenantLine() {
        if (ignoreTenantLine == null) {
            synchronized (this) {
                if (ignoreTenantLine == null) {
                    Class<T> clazz = entityClass;
                    ignoreTenantLine = (clazz != null
                            && clazz.isAnnotationPresent(IgnoreTenantLine.class));
                }
            }
        }
        return ignoreTenantLine;
    }

    /**
     * 框架级租户拦截控制：在执行数据库操作前后自动设置/清除 IgnoreStrategy。
     * 如果 Entity 未标注 @IgnoreTenantLine，直接执行，零开销。
     */
    protected <R> R executeWithTenantControl(Supplier<R> action) {
        if (!isIgnoreTenantLine()) {
            return action.get();
        }
        try {
            InterceptorIgnoreHelper.handle(IGNORE_TENANT);
            return action.get();
        } finally {
            InterceptorIgnoreHelper.clearIgnoreStrategy();
        }
    }

    // ===== 以下为现有方法的改造，在入口处包装 executeWithTenantControl =====

    @Override
    public T getById(Serializable id) {
        return executeWithTenantControl(() -> super.getById(id));
    }

    @Override
    public List<T> listByIds(Collection<? extends Serializable> ids) {
        return executeWithTenantControl(() -> super.listByIds(ids));
    }

    @Override
    public List<T> list() {
        return executeWithTenantControl(() -> super.list());
    }

    @Override
    public List<T> list(Wrapper<T> queryWrapper) {
        return executeWithTenantControl(() -> super.list(queryWrapper));
    }

    @Override
    public boolean save(T entity) {
        return executeWithTenantControl(() -> super.save(entity));
    }

    @Override
    public boolean updateById(T entity) {
        return executeWithTenantControl(() -> super.updateById(entity));
    }

    @Override
    public boolean removeById(Serializable id) {
        return executeWithTenantControl(() -> super.removeById(id));
    }

    @Override
    public boolean remove(Wrapper<T> queryWrapper) {
        return executeWithTenantControl(() -> super.remove(queryWrapper));
    }

    @Override
    public boolean update(Wrapper<T> updateWrapper) {
        return executeWithTenantControl(() -> super.update(updateWrapper));
    }

    @Override
    public boolean saveBatch(Collection<T> entityList) {
        return executeWithTenantControl(() -> super.saveBatch(entityList));
    }

    // ... 其他需要覆盖的方法同理
}
```

// paas-platform-service: 子类无需任何特殊处理

### 5.4 OperateLogBaseDao — 不再需要

`@IgnoreTenantLine` 下沉到 `DataBaseServiceImpl` 后，framework-dao 不再需要 `OperateLogBaseDao` 抽象类。
操作日志的业务方法（`commitByProcessId`、`findPendingProcessIds`、`rollbackByProcessId` 等）
直接在 paas-platform-service 的 DAO 子类中实现，DAO 子类直接继承 `DataBaseServiceImpl`：

```java
@Slf4j
@Service
public class EntityOperateLogDao extends DataBaseServiceImpl<EntityOperateLog> {

    public void commitByProcessId(Long processId) {
        // 直接调用 remove()，框架自动跳过租户拦截
        remove(new QueryWrapper<EntityOperateLog>()
                .eq("process_id", processId));
    }

    public List<Long> findPendingProcessIds(long beforeTime) {
        // 直接调用 list()，框架自动跳过租户拦截
        List<EntityOperateLog> logs = list(new QueryWrapper<EntityOperateLog>()
                .le("created_at", beforeTime)
                .eq("rollback_status", OperateLog.ROLLBACK_PENDING));
        if (logs.isEmpty()) return Collections.emptyList();
        return logs.stream()
                .map(OperateLog::getProcessId)
                .distinct()
                .collect(Collectors.toList());
    }

    // ... 其他业务方法
}
```

### 5.5 设计要点

1. **注解继承**：`@IgnoreTenantLine` 加在 `OperateLog` 基类上，`MetaOperateLog` / `EntityOperateLog` 自动继承（Java 注解默认不继承，但 `isAnnotationPresent` 检查的是 `entityClass` 即子类，需要在子类或基类上标注。如果加在抽象基类上，需使用 `@Inherited` 元注解或在 `isIgnoreTenantLine()` 中遍历父类链）。

    推荐做法：`@IgnoreTenantLine` 加 `@Inherited` 元注解：
    ```java
    @Target(ElementType.TYPE)
    @Retention(RetentionPolicy.RUNTIME)
    @Inherited   // ← 子类自动继承
    @Documented
    public @interface IgnoreTenantLine {}
    ```

2. **零开销**：`isIgnoreTenantLine()` 使用 double-check locking 延迟初始化，结果缓存在 `Boolean` 字段中。未标注 `@IgnoreTenantLine` 的 Entity（绝大多数），`executeWithTenantControl` 直接执行 `action.get()`，无 ThreadLocal 操作，无性能损耗。

3. **与 `@DaoCacheConfig` 同级**：`@DaoCacheConfig` 控制缓存行为，`@IgnoreTenantLine` 控制租户拦截行为，都是 Entity 级别的框架注解，由 `DataBaseServiceImpl` 统一消费。

## 六、启动时序

```
Spring Boot 启动
  │
  ├─ 1. DataSource + MyBatis-Plus 初始化
  │     └─ TenantLineInnerInterceptor 注册（标准配置，无特殊处理）
  │
  ├─ 2. Bean 初始化（@PostConstruct）
  │     └─ EntityPendingTransactionRecovery.recoverPendingTransactions()
  │         → operateLogService.findPendingProcessIds(threshold)
  │           → EntityOperateLogDao.findPendingProcessIds()  // 继承 DataBaseServiceImpl
  │             → list(queryWrapper)  // 继承 DataBaseServiceImpl
  │               → DataBaseServiceImpl.list(wrapper)
  │                 → executeWithTenantControl(() -> super.list(wrapper))
  │                   → isIgnoreTenantLine()
  │                     → EntityOperateLog.class.isAnnotationPresent(@IgnoreTenantLine) = true
  │                   → InterceptorIgnoreHelper.handle(tenantLine=true)
  │                   → super.list(wrapper)
  │                     → TenantLineInnerInterceptor.beforeQuery()
  │                       → willIgnoreTenantLine() = true ✅ 跳过
  │                   → 查询成功
  │                   → InterceptorIgnoreHelper.clearIgnoreStrategy()
  │
  └─ 3. 应用启动完成
```

## 七、向后兼容性

| 维度 | 影响 |
|:---|:---|
| 未标注 `@IgnoreTenantLine` 的 Entity | `isIgnoreTenantLine()=false`，`executeWithTenantControl` 直接执行，行为不变 |
| 现有 `DataBaseServiceImpl` 子类 | 无感知，所有方法行为不变 |
| TenantInterceptor | 无需改动 |
| 调用方 | 无需改动 |
| 性能 | 未标注的 Entity 零开销；标注的 Entity 仅多一次 ThreadLocal set/clear |

## 九、使用指南

未来新增需要跳过租户拦截的表时，只需两步：

1. `p_meta_model` 注册时设置 `enable_tenant_intercept=0`
2. Entity 类加 `@IgnoreTenantLine` 注解

```java
@Data
@IgnoreTenantLine
@TableName("p_system_task")
public class SystemTask extends BaseEntity {
    private Long tenantId;
    // ...
}

@Service
public class SystemTaskDao extends DataBaseServiceImpl<SystemTask> {
    // 所有方法自动跳过租户拦截，无需任何额外代码
}
```
