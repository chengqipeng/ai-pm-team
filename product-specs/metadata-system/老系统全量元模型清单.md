# 老系统 paas-metarepo-service 全量元模型清单

> 来源：`repos/apass_old_projects/paas-metarepo-service/`
> 分析依据：`MetaModelKit.java`（域分类 + 分层注册）、`dao.xml`（PO 类映射）、`sysmeta/*.schema.json`（Schema 定义）
> 整理日期：2026-04-01

## 概述

老系统通过 `MetaConstants.METAMODEL_ID_*` 常量定义元模型 ID（负数 Long），每个元模型对应：
- 一个 Schema 定义文件（`sysmeta/*.schema.json`）
- 一个 DTO 类（X 前缀，facade 层）
- 一个 PO 类（数据库实体，repository 层）

元模型按业务域分为 **9 大域**，共计 **130+ 个元模型**。

---

## 一、元模型域（MetaModel Domain）— 3 个

元模型的元模型，定义元模型自身的结构。

| Schema 文件 | 说明 | PO 类 | 数据表前缀 |
|:---|:---|:---|:---|
| metamodel.schema.json | 元模型定义 | MetaModel | p_meta_model |
| metaitem.schema.json | 元属性定义 | MetaItem | p_meta_item |
| metalink.schema.json | 元关联定义 | MetaLink | p_meta_link |

辅助 PO：MetaOption（元选项）、MetaSandboxItem（沙盒元属性）、MetaSandboxRelation（沙盒元关联）

## 数据存储模式分类

老系统元模型的数据存储有 4 种模式：

| 存储模式 | 说明 | 典型元模型 |
|:---|:---|:---|
| 独立 PO 表 | 每个元模型有独立的数据库表和 PO 类 | entity、item、pickOption、checkRule 等 |
| CommonMetaModel 分层表 | 使用通用分层表 `p_meta_metamodel_data_*` | fieldSet、entityTypeRelFieldSet、territoryRecycleTemplate 等 |
| CommonJSON 统一表 | 使用 `p_meta_tenant_metadata` JSON 存储 | tenantMetadata |
| 系统级只读 | 只有系统级数据，不支持租户定制 | metaModel、metaItem、metaLink、各种 Config 等 |
