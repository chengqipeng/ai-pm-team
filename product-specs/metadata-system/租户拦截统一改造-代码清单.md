# 租户拦截统一改造 — 完整逻辑清单与代码修改清单

> 日期：2026-04-16
> 目标：租户拦截行为统一由两个来源驱动，消除硬编码忽略列表

## 一、统一后的逻辑模型

### 1.1 两个来源，两个层级

```
来源 1（框架硬编码）：p_meta_* / p_common_* → TenantInterceptor.ignoreTable() 跳过
来源 2（元模型配置）：enable_tenant_intercept=0 → Entity @IgnoreTenantLine → DataBaseServiceImpl 跳过
```

### 1.2 SQL 执行时的判断流程

```
SQL 进入 MyBatis-Plus 拦截器链
  │
  ├─ 层级 1：DataBaseServiceImpl.executeWithTenantControl()
  │   Entity 标注 @IgnoreTenantLine?
  │   ├─ 是 → InterceptorIgnoreHelper.handle(tenantLine=true)
  │   │       → beforeQuery() 第一行 willIgnoreTenantLine()=true → 直接 return
  │   └─ 否 → 不设置，正常进入拦截器
  │
  └─ 层级 2：TenantInterceptor.ignoreTable(tableName)
      表名以 p_meta_ 或 p_common_ 开头?
      ├─ 是 → true（跳过，元数据体系基石）
      └─ 否 → false → getTenantId() → 追加 AND tenant_id = ?
```

### 1.3 消除的内容

| 消除项 | 原位置 | 替代方案 |
|:---|:---|:---|
| `PLATFORM_TABLES` 硬编码集合 | MetaRepoDataConfig | Entity 加 @IgnoreTenantLine |
| `TenantInterceptor.addIgnorePredicate()` 中的 PLATFORM_TABLES 判断 | MetaRepoDataConfig.init() | 删除 |
| `TenantInterceptor.IGNORE_TABLES` 静态集合 | TenantInterceptor | 删除 |
| `TenantInterceptor.ignorePredicate` 静态字段 | TenantInterceptor | 删除 |
| `TenantInterceptor.addIgnoreTable()` 静态方法 | TenantInterceptor | 删除 |
| `TenantInterceptor.addIgnorePredicate()` 静态方法 | TenantInterceptor | 删除 |

## 二、代码修改清单

### 2.1 framework-dao（2 个文件）

#### ① TenantInterceptor.java — 简化

当前代码：
```java
private static final Set<String> IGNORE_TABLES = new CopyOnWriteArraySet<>();
private static volatile Predicate<String> ignorePredicate;

public static void addIgnoreTable(String tableName) { ... }
public static void addIgnorePredicate(Predicate<String> predicate) { ... }

@Override
public boolean ignoreTable(String tableName) {
    String simple = tableName.contains(".")
            ? tableName.substring(tableName.lastIndexOf('.') + 1) : tableName;
    if (IGNORE_TABLES.contains(simple)) return true;
    if (ignorePredicate != null && ignorePredicate.test(simple)) return true;
    return false;
}
```

改为：
```java
// 删除 IGNORE_TABLES、ignorePredicate、addIgnoreTable()、addIgnorePredicate()

@Override
public boolean ignoreTable(String tableName) {
    String simple = tableName.contains(".")
            ? tableName.substring(tableName.lastIndexOf('.') + 1) : tableName;
    // 元数据体系基石：元模型定义表 + Common 级存储表，固定无 tenant_id
    return simple.startsWith("p_meta_") || simple.startsWith("p_common_");
}
```

删除的 import：`java.util.Set`、`java.util.concurrent.CopyOnWriteArraySet`、`java.util.function.Predicate`

#### ② DataBaseServiceImpl.java — 已完成（无需再改）

`@IgnoreTenantLine` 识别 + `executeWithTenantControl()` 已在前序步骤实现。

### 2.2 paas-platform-service Entity 层（4 个文件加注解）

以下 Entity 当前在 `PLATFORM_TABLES` 中被硬编码忽略，改为加 `@IgnoreTenantLine` 注解：

#### ③ Passport.java — 加 @IgnoreTenantLine

```java
// 新增 import
import com.hongyang.framework.dao.annotation.IgnoreTenantLine;

@Data
@EqualsAndHashCode(callSuper = true)
@IgnoreTenantLine    // ← 新增：全局身份凭证，无 tenant_id，不走租户拦截
@TableName("p_passport")
public class Passport extends BaseEntity { ... }
```

原因：`p_passport` 无 tenant_id 列，当前靠 `PLATFORM_TABLES` 硬编码跳过。

#### ④ PassportLog.java — 加 @IgnoreTenantLine

```java
import com.hongyang.framework.dao.annotation.IgnoreTenantLine;

@Data
@EqualsAndHashCode(callSuper = true)
@IgnoreTenantLine    // ← 新增：登录日志，无 tenant_id
@TableName("p_passport_log")
public class PassportLog extends BaseEntity { ... }
```

#### ⑤ PlatformUser.java — 加 @IgnoreTenantLine

```java
import com.hongyang.framework.dao.annotation.IgnoreTenantLine;

@Data
@EqualsAndHashCode(callSuper = true)
@IgnoreTenantLine    // ← 新增：用户表，业务代码手动控制 tenant_id
@TableName("p_user")
@DaoCacheConfig(prefix = "dao:PlatformUser", ttl = 1800)
public class PlatformUser extends BaseTenantDataEntity { ... }
```

原因：`p_user` 有 tenant_id 列，但业务代码在 QueryWrapper 中手动加 `eq("tenant_id", tenantId)`，不依赖拦截器自动追加。当前靠 `PLATFORM_TABLES` 硬编码跳过。

#### ⑥ UserRole.java — 加 @IgnoreTenantLine

```java
import com.hongyang.framework.dao.annotation.IgnoreTenantLine;

@Data
@EqualsAndHashCode(callSuper = true)
@IgnoreTenantLine    // ← 新增：用户角色关联，业务代码手动控制 tenant_id
@TableName("p_user_role")
public class UserRole extends BaseEntity { ... }
```

原因：同 `PlatformUser`，`UserRoleServiceImpl` 中所有查询都手动加了 `eq("tenant_id", tenantId)`。

### 2.3 paas-platform-service 配置层（1 个文件）

#### ⑦ MetaRepoDataConfig.java — 删除租户忽略规则

当前代码：
```java
private static final Set<String> PLATFORM_TABLES = Set.of(
        "p_passport", "p_passport_log", "p_user", "p_user_role"
);

@PostConstruct
public void init() {
    // ---- 租户忽略规则 ----
    TenantInterceptor.addIgnorePredicate(tableName ->
            tableName.startsWith("p_meta_")
            || tableName.startsWith("p_common_")
            || PLATFORM_TABLES.contains(tableName));

    // ---- 统一表名路由 ----
    DynamicTableNameInterceptor.registerGlobalMapper(tableName -> { ... });
}
```

改为：
```java
// 删除 PLATFORM_TABLES 常量
// 删除 import com.hongyang.framework.dao.tenant.TenantInterceptor;

@PostConstruct
public void init() {
    // 租户忽略规则已统一：
    //   p_meta_* / p_common_* → TenantInterceptor.ignoreTable() 框架硬编码
    //   其他表 → Entity @IgnoreTenantLine + DataBaseServiceImpl 自动处理
    // 此处不再注册任何忽略规则

    // ---- 统一表名路由（不变） ----
    DynamicTableNameInterceptor.registerGlobalMapper(tableName -> { ... });
}
```

## 五、执行顺序

```
Step 1: 执行 DML — 新增元模型注册 + user 补充 enable_tenant_intercept=0
         （纯数据，不改代码，可先验证元模型注册正确性）

Step 2: Entity 加注解 — Passport、PassportLog、PlatformUser、UserRole 加 @IgnoreTenantLine
         （此时两套机制并存，PLATFORM_TABLES 和 @IgnoreTenantLine 同时生效，安全）

Step 3: 验证 — 启动 + 登录 + 用户/角色/部门 CRUD
         （确认 @IgnoreTenantLine 生效）

Step 4: 清理 TenantInterceptor — 删除 IGNORE_TABLES/ignorePredicate/addIgnoreTable/addIgnorePredicate
         简化 ignoreTable() 为只判断 p_meta_*/p_common_* 前缀

Step 5: 清理 MetaRepoDataConfig — 删除 PLATFORM_TABLES + addIgnorePredicate 调用

Step 6: 最终验证 — 全链路回归
```

Step 2→3→4→5 的顺序保证了任何一步出问题都可以安全回退。
