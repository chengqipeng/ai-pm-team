# 产品设计工作流 — 能力目录

当用户询问"你能做什么"、"有哪些工具"、"帮我做XX"等问题时，根据以下目录推荐合适的工具。
用户在聊天框输入 `#` 可以看到所有可用工具的列表。

## 核心流程（三阶段）

默认走分阶段流程，从需求澄清开始，每个阶段完成后 Kiro 自动提醒进入下一阶段。

| 工具 | 引用方式 | 说明 | 触发方式 |
|:---|:---|:---|:---|
| 需求澄清 | `#pm-1-clarify` | 痛点量化→四维评估→决策建议 | 编辑 产品-需求澄清.md 时自动 |
| 方案设计 | `#pm-2-design` | 现状调研+竞品分析+方案设计 | 编辑 产品-方案设计.md 时自动 |
| PRD 撰写 | `#pm-3-prd` | 研发可执行的产品需求文档 | 编辑 产品-PRD.md 时自动 |
| 全流程一键生成 | `#pm-0-full-pipeline` | 一次性跑完三个阶段（仅限简单场景） | 手动 |

## 自动注入的上下文

以下内容会自动加载到每次对话中，无需手动引用：
- `pm-context-product` — 产品定位、Agent 矩阵、平台能力、数据模型、全局约束
- `pm-capabilities`（本文件）— 能力目录

## 推荐引导

当用户要做一个新项目/需求时，推荐先用 `#pm-project-init` 建立项目结构（context/ + product-specs/），再进入分阶段流程。

**默认推荐分阶段流程**（从 `#pm-1-clarify` 开始）。仅在以下情况推荐 `#pm-0-full-pipeline`：
- 用户明确要求使用 pm-0 / 一键生成 / 全流程
- 需求非常简单（如：改个文案、加个配置项、小交互优化）
- 只是快速出个草稿，后续再细化

其他场景推荐：
- 新项目立项/建结构 → `#pm-project-init`
- 想探索方向/发散 → `#pm-tool-brainstorm`
- 想做战略规划 → `#pm-tool-strategy`
- 想验证设计效果 → `#pm-tool-prototype`
- 想了解已有功能 → `#pm-context-harvest`
- 想分析代码实现 → `#source-code-analyzer`
